The .mochi file is just a zip archive with a plain text description of the deck&#x2F;cards (in extensible data notation[0]) and any media files used in the deck(s) exported. You can find documentation on the format of edn file on the FAQ[1].<p>2. At the moment you can also use cloze deletions. I do have plans for things like typing in an answer, or drawing with a touch screen, but I have no plans for something like multiple choice, but I could be convinced otherwise. I&#x27;m also open to suggestions.<p>3. Yeah, this is probably the best part of Anki IMO. I decided not to include it in the initial version of Mochi because I thought plain markdown documents would be easier for new users to &quot;grok&quot;. I still plan to add this kind of templating thing in the future, but I still need some time to let the idea &quot;bake&quot;.<p>4. You can actually get this behavior in Anki, but it&#x27;s not the default. The initial inspiration for this change came from this blog post[2], but it is roughly equivalent to the Leitner System[3].
