Item(by='tailspin2019', descendants=None, kids=None, score=None, time=1611347309, title=None, item_type='comment', url=None, parent=25870623, text='I agree with the concern of the server code being out-of-date.<p>I know the consensus seems to be that it’s E2E so you “don’t need to trust the server”, but still as a principle it seems strange to be so out of date.<p>Perhaps more importantly, I believe Signal doesn’t have reproducible builds for their iOS client app[0] * whereas, interestingly, it seems Telegram does [1].<p>For me, the combination of out of date server code and non-reproducible iOS client binaries mean that for all my support of Signal (I donated during their recent outage) the benefits of this being a fully “open source” message solution are somewhat devalued.<p>Not to be too down on them though, I don’t touch WhatsApp with a barge pole, so they’re still my favoured messaging platform.<p>* Not necessarily entirely Signal’s fault because, of course, Apple doesn’t make this easy.<p>[0]: <a href="https:&#x2F;&#x2F;github.com&#x2F;signalapp&#x2F;Signal-iOS&#x2F;issues&#x2F;641" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;signalapp&#x2F;Signal-iOS&#x2F;issues&#x2F;641</a><p>[1]: <a href="https:&#x2F;&#x2F;core.telegram.org&#x2F;reproducible-builds" rel="nofollow">https:&#x2F;&#x2F;core.telegram.org&#x2F;reproducible-builds</a>')
