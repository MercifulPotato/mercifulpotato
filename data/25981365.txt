You could use &quot;reflection&quot; or some dynamic feature to do this, but lose type-safety</i><p>By using reflection you only lose type-safety for this particular function. With a dynamic language, you lose it everywhere.<p>And it&#x27;s still an one-liner or close in a language without much ceremony (e.g. not Java 1.5).<p>Example implementation (pseudo-code):<p><pre><code>  fun capitalize (o: object) -&gt; [f.toUpper() for f in \n  reflection.fields(o) if reflection.type(f) == &quot;string&quot;]</code></pre>')
