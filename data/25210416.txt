Item(by='hunter2_', descendants=None, kids=[25211002], score=None, time=1606319194, title=None, item_type='comment', url=None, parent=25208511, text='I wonder if the disconnect is that the phrase &quot;open source&quot; refers to &quot;source code,&quot; and UI designers who would otherwise feel able to contribute may think &quot;I don&#x27;t produce source code, I produce [whatever artifact of their process]; too bad the pull request workflow only collects source code and a readme.&quot;<p>If I&#x27;m correct to suspect that this train of thought might exist (and not only keeping UI designers away, but all sorts of related professions like product managers, marketing, etc.) then maybe the whole notion of &quot;[product] is open source, contribute to the source code or issue queue&quot; needs to be rebranded as &quot;[product] is an open collective work, contribute in any possible way&quot; with a collaborationware workflow that doesn&#x27;t consider changing the source and filing issues (bugs, feature requests) to be the only first class citizens.<p>Many projects do have communities with robust organization of leadership, management, branding, etc. but with real or perceived barriers to entry that don&#x27;t give the same sense of openness and &quot;everyone can contribute to this well-oiled contribution intake machine&quot; in non-code aspects akin to what developers enjoy.')
