You can read and write database files without using any SQLite source code by conforming to the expected format: <a href="https:&#x2F;&#x2F;www.sqlite.org&#x2F;fileformat.html" rel="nofollow">https:&#x2F;&#x2F;www.sqlite.org&#x2F;fileformat.html</a><p>It&#x27;s not even that exotic: a bunch of rows in a simple binary format stored in a btree. The rollback journal &#x2F; WAL are trickier, but aren&#x27;t <i>strictly</i> necessary.<p>Implementing arbitrary SQL queries on top of the file format is obviously _much_ harder.<p>Here&#x27;s an example project doing this-- a pure Go sqlite reader: <a href="https:&#x2F;&#x2F;github.com&#x2F;alicebob&#x2F;sqlittle" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;alicebob&#x2F;sqlittle</a>')
