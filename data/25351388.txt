Item(by='some_furry', descendants=None, kids=[25351647], score=None, time=1607461810, title=None, item_type='comment', url=None, parent=25347915, text='I took a quick look at the source code. It&#x27;s providing end-to-end encryption with libsodium, using crypto_box [1], crypto_secretbox[2], and crypto_pwhash for password-based key derivation [3].<p>The public key model appears to be TOFU [4]. It&#x27;s doing a distinct crypto_box per notification [5]. It doesn&#x27;t use an authenticated key exchange or offer key rotation or forward secrecy, but that&#x27;s probably fine for this use case. Not too long ago, I wrote a guide to end-to-end encryption [6], and I would classify the &quot;end-to-end encryption&quot; here as meeting the minimum definition (data is encrypted between devices, rather than in a client-server architecture where the server has access to plaintext), even if it&#x27;s not suitable for more sensitive threat models.<p>One thing I didn&#x27;t see was message padding of location data prior to encryption, to prevent side-channel attacks via ciphertext length.
