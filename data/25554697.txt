Item(by='cle', descendants=None, kids=[25555183], score=None, time=1609106915, title=None, item_type='comment', url=None, parent=25553583, text='Bash runs on a lot more than Linux.<p>I ran a Docker container and copied a statically-linked curl and bash onto a scratch image and ran it, it does not work:<p><pre><code>    $ docker run -it --rm  $(docker build -q .)\n    &#x2F;srvmon.sh: line 162: date: command not found\n\n    $ cat Dockerfile\n    FROM scratch\n    COPY bash &#x2F;usr&#x2F;bin&#x2F;bash\n    COPY curl &#x2F;usr&#x2F;bin&#x2F;curl\n    COPY srvmon.sh &#x2F;srvmon.sh\n    CMD [&quot;&#x2F;usr&#x2F;bin&#x2F;bash&quot;, &quot;&#x2F;srvmon.sh&quot;]\n\n</code></pre>\nI&#x27;m not pointing this out to be pedantic. I&#x27;m pointing it out because there is a common misconception that all these things like &quot;date&quot; and &quot;cat&quot; and &quot;mkdir&quot; are part of Bash, but they&#x27;re not, they&#x27;re part of coreutils and there are dramatically different versions of coreutils on different installations of Linux, macOS, BSD, etc., and some environments (like barebones Docker containers) don&#x27;t have coreutils at all.')
