Item(by='pjmlp', descendants=None, kids=[25801323], score=None, time=1610788010, title=None, item_type='comment', url=None, parent=25800348, text='While I am on the Rust side on this, I am not sold out on mixing crates with different editions, because I believe it won&#x27;t work at scale in typical enterprise contexts.<p>Meaning, having binary crates compiled with different epochs, and then passing data and closures across crate public functions, specially using as input data from modern ones into the older ones.<p>With the language evolution it won&#x27;t be possible to keep semantic changes to work as expected, or if the runtime requirements have changed, that linking everything together will still work as expected.<p>To me epochs seem to only work when using a single compiler, compiling all crates from scratch, with is nice, but not what enterprise C, C++, Java, .NET code looks like, and any new language being adopted is expected to play the same game.<p>Hence why Apple took such a big effort to make Swift ABI being able to deal with language updates.')
