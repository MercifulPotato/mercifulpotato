Item(by='phillipcarter', descendants=None, kids=None, score=None, time=1609524247, title=None, item_type='comment', url=None, parent=25601346, text='&quot;the complete API&quot; is a bit of a misnomer, since there have been new APIs and runtime capabilities that <i>aren&#x27;t</i> available to the Windows-only, older runtime (The .NET Framework). This has been the case since at least .NET Core 2.1 but has continued ever since.<p>There are several APIs in the older runtime that are intentionally not brought forward, and what I believe you&#x27;re referring to is this announcement: <a href="https:&#x2F;&#x2F;github.com&#x2F;dotnet&#x2F;announcements&#x2F;issues&#x2F;130" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;dotnet&#x2F;announcements&#x2F;issues&#x2F;130</a><p>The remaining APIs are (mostly) AppDomains, Remoting, Web Forms, WCF server, and Windows Workflow, most of which is either an acknowledged &quot;this was the wrong way to do it so we won&#x27;t bring it forward&quot; (e.g., Remoting) or tied to Windows anyways (e.g., WCF).')
