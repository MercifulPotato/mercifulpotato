Item(by='acemarke', descendants=None, kids=[25776569, 25773762], score=None, time=1610606074, title=None, item_type='comment', url=None, parent=25771790, text='Hi, I&#x27;m a Redux maintainer. Please note that &quot;modern Redux&quot; code is very different than what most older tutorials show. We&#x27;ve introduced newer APIs like Redux Toolkit, which is a set of utilities that provide a light abstraction to simplify the most common Redux tasks, and the React-Redux hooks API, which is generally easier to use than the traditional `connect` API.<p>I strongly recommend reading through the newly rewritten official tutorials in the Redux docs, which have been specifically designed to show our recommended practices:<p>- &quot;Redux Essentials&quot; tutorial [0]: teaches &quot;how to use Redux, the right way&quot;, by building a real-world app using Redux Toolkit\n- &quot;Redux Fundamentals&quot; tutorial [1]: teaches &quot;how Redux works, from the bottom up&quot;, by showing how to write Redux code by hand and why standard usage patterns exist, and how Redux Toolkit simplifies those patterns<p>The older patterns shown in almost all other tutorials on the internet are still valid, but not how we recommend writing Redux code today.<p>You should also check out the Redux &quot;Style Guide&quot; docs page [2], which explains our recommended patterns and best practices, and why they exist. Following those will result in better and more maintainable Redux apps.<p>[0] <a href="https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;essentials&#x2F;part-1-overview-concepts" rel="nofollow">https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;essentials&#x2F;part-1-overview-co...</a><p>[1] <a href="https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;fundamentals&#x2F;part-1-overview" rel="nofollow">https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;fundamentals&#x2F;part-1-overview</a><p>[2] <a href="https:&#x2F;&#x2F;redux.js.org&#x2F;style-guide&#x2F;style-guide" rel="nofollow">https:&#x2F;&#x2F;redux.js.org&#x2F;style-guide&#x2F;style-guide</a>')
