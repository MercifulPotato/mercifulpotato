Item(by='slezyr', descendants=None, kids=[25521774], score=None, time=1608752475, title=None, item_type='comment', url=None, parent=25520549, text='&gt; Session-based Auth<p>&gt; It&#x27;s stateful. The server keeps track of each session on the server-side. The session store, used for storing user session information, needs to be shared across multiple services to enable authentication. Because of this, it doesn&#x27;t work well for RESTful services, since REST is a stateless protocol.<p>What stops you from keeping the JWT token in there? In fact, I doubt that it&#x27;s some random session ID and not some encrypted payload that gets decrypted instead of looking it up in the database.')
