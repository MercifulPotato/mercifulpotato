Item(by='Arnavion', descendants=None, kids=[25802119], score=None, time=1610796758, title=None, item_type='comment', url=None, parent=25799504, text='Filesystems, networking, ... Basically anything a program needs to interact with things outside its address space requires C interop and thus unsafe code. I sure hope your web server is listening on a socket because it&#x27;s not going to be much of a web server otherwise.<p>Then there&#x27;s crypto, for which the corelib defers to the OS (CNG &#x2F; openssl) because doing crypto in managed code is hard.<p>But forget all that; even pure C# code isn&#x27;t safe because it occasionally needs to hook into the CLR for unsafe things. Eg <a href="https:&#x2F;&#x2F;source.dot.net&#x2F;#System.Private.CoreLib&#x2F;Dictionary.cs,827" rel="nofollow">https:&#x2F;&#x2F;source.dot.net&#x2F;#System.Private.CoreLib&#x2F;Dictionary.cs...</a> - unsafe code called from Dictionary.Remove')
