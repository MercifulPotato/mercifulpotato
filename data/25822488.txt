Item(by='sleavey', descendants=None, kids=[25822556, 25822523, 25822922], score=None, time=1610982661, title=None, item_type='comment', url=None, parent=25822358, text='I sometimes use your &quot;bug&quot; on purpose in tests, when checking that two values are either both defined or both undefined, but not one defined and one undefined:<p>$ python<p>&gt;&gt;&gt; a, b = 1, 2<p>&gt;&gt;&gt; bool(a) ^ bool(b)<p>False<p>&gt;&gt;&gt; a, b = 1, 1<p>&gt;&gt;&gt; bool(a) ^ bool(b)<p>False<p>&gt;&gt;&gt; a, b = None, None<p>&gt;&gt;&gt; bool(a) ^ bool(b)<p>False<p>&gt;&gt;&gt; a, b = None, 1<p>&gt;&gt;&gt; bool(a) ^ bool(b)<p>True<p>(Note: this doesn&#x27;t work if a value can be 0, because bool(0) is False in Python).<p>This helps to avoid having to write something like `(a is not None and b is not None) or (a is None and b is None)`.')
