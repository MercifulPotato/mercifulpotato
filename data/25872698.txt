Item(by='SigmundA', descendants=None, kids=[25872834], score=None, time=1611330821, title=None, item_type='comment', url=None, parent=25872134, text='Just putting &quot;order-550e8400-e29b-11d4-a716-446655440000&quot; does not make it an order id either, thats what foreign keys are for, mistakes can be made either way.<p>If your sending id via json you should be using json to encode type either:<p>{ order:&quot;550e8400-e29b-11d4-a716-446655440000&quot; }<p>Or if for some reason the value could contain multiple types:<p>{ thing: { t:&quot;order&quot;, id:&quot;550e8400-e29b-11d4-a716-446655440000&quot; } }<p>Even then just use:<p>{ thing: { order :&quot;550e8400-e29b-11d4-a716-446655440000&quot; } }<p>This is redundant and not necessary adding overhead:<p>{ order: &quot;order-550e8400-e29b-11d4-a716-446655440000&quot; }<p>Storing that in your db as json even worse (Mongo, JSONB), storing as varchar in normal column not much better.<p>If your not storing ID with a type prefix in the db, then thats not the form of your id&#x27;s that just the UI&#x2F;encoding for them in your app which eliminates half the supposed advantage which was easy debugging say in db query tools. It also means you are parsing &#x2F; deparsing &quot;order-550e8400-e29b-11d4-a716-446655440000&quot; somewhere in your db mapper code instead of just using your json or query string parser or protobuf or whatever, why?.')
