Item(by='jfmengels1', descendants=None, kids=[25956885], score=None, time=1611905006, title=None, item_type='comment', url=None, parent=25953271, text='Elm doesn&#x27;t have asserts. Instead, what you&#x27;d do is to have the function return a potential error (what in Elm we&#x27;d call a Result).<p>so instead of<p>function a(b) {\n  assert(b &gt; 0);\n  return b + 1;\n}<p>you&#x27;d do<p>a b =\n  if b &gt; 0 then\n    Ok (b + 1)\n  else\n    Err &quot;b was not &gt; 0&quot;<p>To then use the result of the function call, you&#x27;ll need to  unwrap the result, by handling the case where the function returned Ok, but also the case where you return Err<p>case a -2 of\n  Ok b -&gt; -- display the value b\n  Err errorMessage -&gt; -- display the error message<p>THis is the kind of technique Elm uses to ensure that you&#x27;ll have no runtime errors in your code: by forcing you to handle the error cases. The nice thing is that the types can indicate whether something can fail or can&#x27;t.')
