They&#x27;re handled by the underlying QtWebEngine&#x2F;Chromium, and only for a few of those things there&#x27;s a way for qutebrowser to delete them. There&#x27;s an issue about it here:\n<a href="https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;58" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;58</a> - but many things probably can&#x27;t be done, or only in a hackish way (e.g. by qutebrowser reading&#x2F;manipulating those internal Chromium files, before it starts up fully).<p>You can however remove ~&#x2F;.cache&#x2F;qutebrowser and ~&#x2F;.local&#x2F;share&#x2F;qutebrowser&#x2F;webengine&#x2F; manually.<p>&gt; The ultimate goal would be some way to replace the functionality of &quot;Temporary Containers&quot; or alternatively &quot;Cookie Autodelete&quot; (you keep all the history, but everything else is in Incognito mode except for whitelisted sites.)<p>Via qutebrowser&#x27;s `--temp-basedir` (or `--basedir`) flag, you can already start an isolated instance - but indeed the history won&#x27;t be shared with the main instance (and neither will e.g. the config, but there&#x27;s the --config-py flag).<p>There are also some &quot;profile managers&quot; which are based on the `--basedir` flag: <a href="https:&#x2F;&#x2F;www.reddit.com&#x2F;r&#x2F;qutebrowser&#x2F;comments&#x2F;l6zj0z&#x2F;comparing_qpm_and_qutebrowserprofile_as&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.reddit.com&#x2F;r&#x2F;qutebrowser&#x2F;comments&#x2F;l6zj0z&#x2F;compari...</a><p>Finally, I&#x27;d indeed like to have a container-like feature integrated into qutebrowser itself in some way: <a href="https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;4102" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;4102</a> - automatically switching containers based on the URL unfortunately won&#x27;t be possible, but I think even just launching a new window with a different storage path (i.e. profile) would provide a lot of value.<p>&gt; Can&#x27;t disable WebRTC (= unique device id can always be tracked)<p>I think to fully disable it, you&#x27;ll need to rebuild QtWebEngine. I&#x27;m only aware of WebRTC leaking internal IPs, which can be prevented via the `content.webrtc_ip_handling_policy` setting.<p>&gt; Great project, keep it up :)<p>Thanks for all the feedback, I really appreciate it!')
