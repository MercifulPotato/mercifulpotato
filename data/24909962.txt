Item(by='ihnorton', descendants=None, kids=None, score=None, time=1603821122, title=None, item_type='comment', url=None, parent=24908960, text='A small warning for code with many CMake ExternalProject dependencies: the defaults for cmake+ninja will recursively run `ninja -jN` for each level of the dependency tree, which can spawn a very large number of compile jobs.<p>There is a proposal to add jobserver support for global process count management [1], but it has not been merged for philosophical reasons (IIRC). The CMake folks maintain a soft fork with this patch applied, available in the `ninja` package on PyPI and probably elsewhere.<p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;ninja-build&#x2F;ninja&#x2F;issues&#x2F;1139" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;ninja-build&#x2F;ninja&#x2F;issues&#x2F;1139</a>')
