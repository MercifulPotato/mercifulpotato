Item(by='cbmuser', descendants=None, kids=None, score=None, time=1607295040, title=None, item_type='comment', url=None, parent=25327384, text='Another Bountysource campaign around the GCC project has been completed successfully [1].<p>The VAX backend in GCC has been converted to MODE_CC register representation [2] reducing the number of backends still using cc0 to two: AVR and CR16 [3]. For AVR, there is still a pending Bountysource campaign currently at $7,215.<p>All backends must be converted to MODE_CC prior the release of GCC 11 or they will be removed [5]. So, if AVR and CR16 are supposed to be supported by GCC in future releases, they need to be converted to MODE_CC register representation.<p>The VAX backend in GCC is used by VAX port of NetBSD and therefore essential for the port.<p>&gt; [1] <a href="https:&#x2F;&#x2F;www.bountysource.com&#x2F;issues&#x2F;91495157-vax-convert-the-backend-to-mode_cc-so-it-can-be-kept-in-future-releases" rel="nofollow">https:&#x2F;&#x2F;www.bountysource.com&#x2F;issues&#x2F;91495157-vax-convert-the...</a><p>&gt; [2] <a href="https:&#x2F;&#x2F;gcc.gnu.org&#x2F;wiki&#x2F;CC0Transition" rel="nofollow">https:&#x2F;&#x2F;gcc.gnu.org&#x2F;wiki&#x2F;CC0Transition</a><p>&gt; [3] <a href="https:&#x2F;&#x2F;gcc.gnu.org&#x2F;backends.html" rel="nofollow">https:&#x2F;&#x2F;gcc.gnu.org&#x2F;backends.html</a><p>&gt; [4] <a href="https:&#x2F;&#x2F;www.bountysource.com&#x2F;issues&#x2F;84630749-avr-convert-the-backend-to-mode_cc-so-it-can-be-kept-in-future-releases" rel="nofollow">https:&#x2F;&#x2F;www.bountysource.com&#x2F;issues&#x2F;84630749-avr-convert-the...</a><p>&gt; [5] <a href="https:&#x2F;&#x2F;gcc.gnu.org&#x2F;legacy-ml&#x2F;gcc-patches&#x2F;2019-09&#x2F;msg01256.html" rel="nofollow">https:&#x2F;&#x2F;gcc.gnu.org&#x2F;legacy-ml&#x2F;gcc-patches&#x2F;2019-09&#x2F;msg01256.h...</a>')
