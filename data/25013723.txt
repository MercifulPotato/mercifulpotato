Item(by='papaf', descendants=None, kids=[25013878], score=None, time=1604735660, title=None, item_type='comment', url=None, parent=25013400, text='<i>Your options are your http status are your application&#x27;s status, which I think is what most of us are used to.</i><p>I used to use HTTP status codes in this way because I understood it was the correct REST way of doing things.<p>However, one day, a sysadmin contacted me to tell me that we had broken a release because our API was returning a 404. Actually it was a problem in the checking script that was checking for data that was no longer in the DB.<p>By making application codes equal to HTTP status codes, we had removed any way to distinguish between fatal errors and API results.')
