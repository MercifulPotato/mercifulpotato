The OS&#x2F;400 command language shell used its own rich knowledge of each command&#x27;s syntax to present a full-screen form you could fill out to interactively compose a command:\n<a href="https:&#x2F;&#x2F;www.ibm.com&#x2F;support&#x2F;knowledgecenter&#x2F;ssw_ibm_i_74&#x2F;rzal2&#x2F;rzal2commands.htm" rel="nofollow">https:&#x2F;&#x2F;www.ibm.com&#x2F;support&#x2F;knowledgecenter&#x2F;ssw_ibm_i_74&#x2F;rza...</a><p>(Also, OS&#x2F;400 commands all follow a very strict naming convention, such as the example WrkActJob in the link. The consistency of the words used and the abbreviations for those words were intended to make learning easier.)<p>You see a bit of this today in PowerShell&#x27;s command auto-completion for built-in commands, PowerShell script files, and cmdlets (user-defined commands written as PowerShell script functions or .NET CLI methods). The graphical PowerShell Integrated Scripting Environment (ISE) even pops up windows that let you interactively fill out forms to enter command options, just like Commando: <a href="https:&#x2F;&#x2F;blog.netwrix.com&#x2F;2018&#x2F;02&#x2F;21&#x2F;windows-powershell-scripting-tutorial-for-beginners&#x2F;" rel="nofollow">https:&#x2F;&#x2F;blog.netwrix.com&#x2F;2018&#x2F;02&#x2F;21&#x2F;windows-powershell-scrip...</a><p>Regrettably, PowerShell ISE is deprecated, and the recommended replacement of Visual Studio Code with the PowerShell extension isn&#x27;t an exact one.')
