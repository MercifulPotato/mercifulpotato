Item(by='chrismorgan', descendants=None, kids=[25707848], score=None, time=1610220481, title=None, item_type='comment', url=None, parent=25701842, text='And since we’re talking about optional things:<p><pre><code>  &lt;link rel=&quot;stylesheet&quot; href=&quot;&#x2F;style.css&quot; &#x2F;&gt;\n  &lt;meta name=&quot;viewport&quot; content=&quot;initial-scale = 1.0,maximum-scale = 1.0&quot; &#x2F;&gt;\n</code></pre>\nTrailing slashes on HTML tags are useless. They’re <i>allowed</i> on void elements, for XML compatibility, but are by definition simply ignored. I recommend against including them, because they’re simple visual noise, and misleading because they don’t actually close tags—you can only use them on on elements that are defined as having no children.<p>(Note that I say <i>HTML</i> tags; on foreign elements—meaning inline SVG and MathML—trailing slashes <i>do</i> make tags self-closing, XML-style.)<p>Also since I’m writing, that viewport declaration is wonky. It should have device-width, and it should not have maximum-scale which is user-unfriendly.<p>All up:<p><pre><code>  &lt;link rel=&quot;stylesheet&quot; href=&quot;&#x2F;style.css&quot;&gt;\n  &lt;meta name=&quot;viewport&quot; content=&quot;device-width,initial-scale=1&quot;&gt;\n</code></pre>\nAnd for completeness, <a href="https:&#x2F;&#x2F;html.spec.whatwg.org&#x2F;multipage&#x2F;syntax.html#elements-2" rel="nofollow">https:&#x2F;&#x2F;html.spec.whatwg.org&#x2F;multipage&#x2F;syntax.html#elements-...</a> defines void and foreign elements.')
