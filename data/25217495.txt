to make sure I have that correct... and make sure I&#x27;m pointing at the correct thing. :)<p><a href="https:&#x2F;&#x2F;developer.blender.org&#x2F;p&#x2F;yuzukyo&#x2F;" rel="nofollow">https:&#x2F;&#x2F;developer.blender.org&#x2F;p&#x2F;yuzukyo&#x2F;</a><p>---<p>Blender doesn&#x27;t really have any presence in the CAD&#x2F;CAM space because (my understanding) is the geometry Blender uses isn&#x27;t &quot;solid geometry&quot;.<p>Because it&#x27;s not solid geometry, Blender isn&#x27;t able to export to <i>any</i> of the widely used file formats in CAD&#x2F;CAM.  eg STEP, IGES, 3DM, etc.<p>Nor even to other OSS packages (eg FreeCAD), as they too use solid geometry.<p>---<p>Solid models are used in industry (CNC, plasma, waterjet, etc), somewhat similar to how STL files are used in 3D printing.<p>With 3D printing, the model file (eg STL) is turned into a gcode action plan for printing by a slicer. eg Cura (<a href="https:&#x2F;&#x2F;ultimaker.com&#x2F;software&#x2F;ultimaker-cura" rel="nofollow">https:&#x2F;&#x2F;ultimaker.com&#x2F;software&#x2F;ultimaker-cura</a>)<p>In industry, the model file (eg STEP) is turned into a gcode action plan for the machine cut out the part by CAM software. eg MasterCAM (<a href="https:&#x2F;&#x2F;www.mastercam.com" rel="nofollow">https:&#x2F;&#x2F;www.mastercam.com</a>)<p>---<p>With that branch (D6807), my understanding was it enables proper in-depth NURBS (solid geometry) support.<p>The commit comments in that branch indicate it also allows importing of Rhino (3DM) and subdivision surface (subd) files.<p>If that&#x27;s the case, it&#x27;s the needed foundation piece to allow for creation of importers and exporter for widely accepted CAD&#x2F;CAM formats.<p>The second an even basic STEP exporter is functional in Blender, it will start pulling users across from the expensive proprietary CAD&#x2F;CAM systems.  As almost no-one actually <i>wants</i> to use them, it&#x27;s just a &quot;lack of choice&quot; problem. ;)')
