Item(by='tylerhou', descendants=None, kids=None, score=None, time=1602370478, title=None, item_type='comment', url=None, parent=24739206, text='I am under the impression that JIT compilers do not modify the compiler’s own bytecode. They write generated code into a separate data region and mark that region as executable. If the code needs to modified, then control transfers back to the compiler which can mark the region as writable again.<p>The same can be done for a binary and it’s own code, but I wonder if it’s used as a signal in antivirus protections if done too often.<p><a href="https:&#x2F;&#x2F;en.m.wikipedia.org&#x2F;wiki&#x2F;W%5EX" rel="nofollow">https:&#x2F;&#x2F;en.m.wikipedia.org&#x2F;wiki&#x2F;W%5EX</a>')
