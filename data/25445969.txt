Item(by='ar_lan', descendants=None, kids=None, score=None, time=1608142190, title=None, item_type='comment', url=None, parent=25441362, text='The problem is you might end up writing strange endpoints just to satisfy a specific UI page.<p>For example, take a small blog, that holds several posts, each with an author.<p>In GraphQL, you could fetch all the posts, and all info related to the author in the same query (without any new, funny business).<p>In REST, if you wanted this in a single query, you&#x27;d need to define some endpoint like &#x2F;post-and-authors-list that fetches all the posts and complete author information for each post, whereas most REST implementations would probably involve a small set of CRUD steps on all posts, and then several queries to fetch each author for further information as well (based on the author ID in the post).<p>So it&#x27;s not that REST can&#x27;t implement anything GraphQL wants to implement, but there is a lot more work involved, is harder to scale out with developers, etc.')
