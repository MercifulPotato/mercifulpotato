Item(by='yyyk', descendants=None, kids=None, score=None, time=1604620458, title=None, item_type='comment', url=None, parent=25001784, text='There&#x27;s a lot of internet content praising git&#x27;s &#x27;simple immutable data tree structure&#x27; and how simple it is to implement. git&#x27;s data structure is also the #1 reason behind limitations that many people try to bypass...<p>e.g. giant monorepos weren&#x27;t a problem even for ancient centralized VCs because these were file based, and if you wanted to work on a part of the tree it didn&#x27;t matter - just push and pull the part you care about. But git has a data structure that forces everything in a single tree, so you have to use hacks (submodules etc.).<p>Same thing for the much of the UX and the other complaints (changeset&#x2F;patch model). When you get down to it, the data structure is behind 90% of difficulties with git.')
