Item(by='danabramov', descendants=None, kids=None, score=None, time=1608585009, title=None, item_type='comment', url=None, parent=25499473, text='It&#x27;s just regular chunked encoding, which we consume via a ReadableStream[1]. The key part is that we&#x27;re <i>able</i> to consume partial data because we read it row by row. Each row represents a piece of JSON with placeholders for missing values. When we receive a row, we try to &quot;continue&quot; rendering on the client, and if we have a loading state we can show (even before all values arrived), we do that.<p><i>&gt;Will the architecture have the potential to support realtime applications where the server will push updates to the client without the need to poll on an interval?</i><p>I don&#x27;t see why that wouldn&#x27;t be possible.<p>[1]: <a href="https:&#x2F;&#x2F;developer.mozilla.org&#x2F;en-US&#x2F;docs&#x2F;Web&#x2F;API&#x2F;Streams_API&#x2F;Using_readable_streams#Consuming_a_fetch_as_a_stream" rel="nofollow">https:&#x2F;&#x2F;developer.mozilla.org&#x2F;en-US&#x2F;docs&#x2F;Web&#x2F;API&#x2F;Streams_API...</a>')
