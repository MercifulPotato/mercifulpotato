Item(by='hydroxideOH-', descendants=None, kids=None, score=None, time=1611115674, title=None, item_type='comment', url=None, parent=25841988, text='In Racket and Scheme, false is the only &#x27;falsy&#x27; value, with everything else evaluating &#x27;truthy&#x27;, which is nice since you don&#x27;t have to worry about the specifics of the return type.<p>I think the bigger issue with null or None, though, is not necessarily using it as a return type (where that makes sense, and can be enforced by a type system) but when objects are mutated to the value of null. The bigger issue there is with mutation and side-effects, but having null there as a convenient short cut can be a foot gun that leads to lots of errors.')
