We are a distributed revision control database - so the model is a bit like git and github (only they piggy backed on an already successful open source project and we have to build both open source and hub!).<p>We really don&#x27;t like the &#x27;traditional&#x27; open source economic model - with a community version and all the good features in a proprietary enterprise edition. The community version is almost necessarily weak and you end up annoying your users. We&#x27;d prefer to keep as open as possible and have a commercial SaaS or open core model.')
