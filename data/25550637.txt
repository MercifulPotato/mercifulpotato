Item(by='and0x000', descendants=None, kids=None, score=None, time=1609073028, title=None, item_type='comment', url=None, parent=25548158, text='This.<p>It is also possible to create a &quot;next page&quot;. You only need to take your current page&#x27;s last id and use it in your query like &quot;where id &gt; current_last_element_id&quot;.<p>&quot;first page&quot; and &quot;last page&quot; are possible as well, simply do &quot;where id &gt; 0&quot; for first and &quot;where id &lt;= (subselect max id)&quot;.<p>You may need to invert order here and there, but it is perfectly possible to use cursor base pagination in a GUI as long as you don&#x27;t need to provide jumping to a specific page number.')
