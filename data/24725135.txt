Item(by='bvallelunga', descendants=None, kids=None, score=None, time=1602200370, title=None, item_type='comment', url=None, parent=24725060, text='We actually support that exact use case natively. Each Doppler project has the concept of root and branch configs where branch configs inherit their secrets from the root. There is a root config for each environment (dev, stg, prd). More here: <a href="https:&#x2F;&#x2F;docs.doppler.com&#x2F;docs&#x2F;enclave-config-branching" rel="nofollow">https:&#x2F;&#x2F;docs.doppler.com&#x2F;docs&#x2F;enclave-config-branching</a><p>In your specific case I would recommend creating a Doppler project for each app. Then you can add the common secrets to the &quot;prd&quot; root config. From there create branch configs for each locations:<p>prd (holds common secrets)<p>- prd_us_east_1 (inherits secrets from root plus hold us_east_1 specific vars)<p>- prd_us_central_1 (inherits secrets from root plus hold us_central_1 specific vars)<p>When you need to add&#x2F;modify&#x2F;delete a secret for all production configs, just modify the &quot;prd&quot; config.')
