Item(by='kube-system', descendants=None, kids=[25871577], score=None, time=1611322592, title=None, item_type='comment', url=None, parent=25871211, text='I can think of a lot of reasons why it might happen.<p>Maybe you have a process accepts multiple types of objects.  Maybe you think you are passing in the correct type of object but you are not.  Maybe the person reporting the error to you omitted information.  Maybe the person who wrote the application didn&#x27;t log enough context.  Yes, all of these situations are not ideal, but if the process was ideal you wouldn&#x27;t be debugging it in the first place.<p>&gt; I really can&#x27;t imagine how this could ever happen in a business. &quot;Yes, sir, there was a problem with 1087.&quot; &quot;1087 what?&quot; &quot;I don&#x27;t know, but it was number 1087.&quot;<p>This happens <i>all</i> the time in many businesses.  Users rarely generate perfect error reports, and it&#x27;s not uncommon for intermediaries to incorrectly communicate details either.  What developer hasn&#x27;t seen an error report that consists of a vague screenshot with &quot;doesn&#x27;t work, got this error&quot; as the description?')
