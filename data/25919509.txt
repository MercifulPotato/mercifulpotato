In the case of Firefox’s image cache, a tracker can create a supercookie by  “encoding” an identifier for the user in a cached image on one website, and then “retrieving” that identifier on a different website by embedding the same image. To prevent this possibility, Firefox 85 uses a different image cache for every website a user visits. That means we still load cached images when a user revisits the same site, but we don’t share those caches across sites.</i><p>Wait, so one form of &quot;supercookie&quot; is basically the same as the transparent gif in an email?<p><a href="https:&#x2F;&#x2F;help.campaignmonitor.com&#x2F;email-open-rates#accuracy" rel="nofollow">https:&#x2F;&#x2F;help.campaignmonitor.com&#x2F;email-open-rates#accuracy</a>')
