Item(by='waheoo', descendants=None, kids=[25965246], score=None, time=1611969271, title=None, item_type='comment', url=None, parent=25962609, text='Dynamic typing is a trade off, one that requires some explicit type checking in some situations.<p>It doesn&#x27;t make it a stupid solution or a stupid problem.<p>If you validate and conform your types at program edges it&#x27;s rare you ever really need to do type checking.<p>That is, a properly structured PHP application has all the benefits of dynamic typing and few of the problems that static type checking supposedly eliminates.<p>I like both static and dynamically typed languages.<p>Just like I like Async and non Async languages.<p>All language types require practice and learning to adjust to the different patterns that are required.<p>If you write PHP like JavaScript, or if you write PHP like java, or php like C++, you&#x27;re going to have these problems.<p>If you write PHP like PHP, function heavy, class light, and a mix of functional and procedural depending on context it actually works out very well for a lot of things.<p>But go ahead, call the language stupid and irrespectable.')
