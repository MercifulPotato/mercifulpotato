Item(by='chmod775', descendants=None, kids=[25159870], score=None, time=1605875104, title=None, item_type='comment', url=None, parent=25159758, text='It&#x27;s nice to explicitly state types at module boundaries, which will make it easier for other developers, theoretically allows some type-checking optimizations, and will prevent you from accidentally changing your API.<p>I&#x27;ve been starting to build speed-focused TS type-checker that is intended to be used in single-file editors like VIM to highlight errors, because it starts checking from a single file and just traverses files necessary to check types within that one file. If you defined your types at a module boundary it can bail out super early, since it doesn&#x27;t need to look at the rest of the code of that module at all.<p>Most linters have rules that will force you to explicitly declare types at module boundaries.<p>Other than that, do whatever you prefer.')
