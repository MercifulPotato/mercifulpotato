Fortunately nobody is forcing you to use this (or forcing me to not use it).<p>&gt; You&#x27;re throwing away all the organizational learning and preexisting systemd documentation, and forcing something different on the world. `man systemd.timer` contains no mention of `startAt`<p>Not <i>quite</i> throwing it all away, because you can easily observe the output of this before making it live.  Yes, systemd.timer contains no mention of startAt because as you correctly observed this is somethign inherently different from systemd.  startAt is used by other configuration options to specify items running at specific calendar times, so it&#x27;s reasonably consistent within nixOS itself.<p>To read the nix documentation is quite simple (and it shows the currently configured value for you):<p><pre><code>  % nixos-option systemd.services.tarsnapback.startAt\n    Value:\n    [ &quot;*-*-* 05:20:00&quot; ]\n\n    Default:\n    [ ]\n\n    Type:\n    &quot;string or list of strings&quot;\n\n    Example:\n    &quot;Sun 14:00:00&quot;\n\n    Description:\n    &#x27;&#x27;\n      Automatically start this unit at the given date&#x2F;time, which\n      must be in the format described in\n      &lt;citerefentry&gt;&lt;refentrytitle&gt;systemd.time&lt;&#x2F;refentrytitle&gt;\n      &lt;manvolnum&gt;7&lt;&#x2F;manvolnum&gt;&lt;&#x2F;citerefentry&gt;.  This is equivalent\n      to adding a corresponding timer unit with\n      &lt;option&gt;OnCalendar&lt;&#x2F;option&gt; set to the value given here.\n    &#x27;&#x27;\n</code></pre>\n&gt; what you have there is something inherently different from systemd.<p>That&#x27;s kind of the point.  If it was inherently the same as systemd there would be no point to it.  Systemd timers are quite boilerplate heavy (compare to e.g. a crontab entry), so when I&#x27;m not using nixos, I often end up copying an existing timer and modifying it.<p>&gt; And what if I want more complex rules, like a combination of intervals and time from boot?<p>Add a time from boot of 120 seconds with this:<p><pre><code>  systemd.timers.tarsnapBack.timerConfig = { OnBootSec = &quot;120&quot;; };\n</code></pre>\nFor things that actually use all the bells and whistles of systemd, you&#x27;ll need to specify all the various details.<p>[edit]<p>For a nice hyperlinked searching of options see also:<p><a href="https:&#x2F;&#x2F;search.nixos.org&#x2F;options?query=startAt&amp;from=0&amp;size=30&amp;sort=relevance&amp;channel=unstable" rel="nofollow">https:&#x2F;&#x2F;search.nixos.org&#x2F;options?query=startAt&amp;from=0&amp;size=3...</a>')
