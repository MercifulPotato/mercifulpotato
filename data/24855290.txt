Item(by='dragonwriter', descendants=None, kids=None, score=None, time=1603346054, title=None, item_type='comment', url=None, parent=24855032, text='&gt; That works if all your YAML authors are carefully avoiding any use of the features JSON can’t represent, like multiple documents, non-string map keys, and application-defined type tags.<p>That&#x27;s true if you want round-trip conversion to and from <i>JSON</i>, but you don&#x27;t need that for automated round-trip conversion between JSON-style (delimited, or, in the language of the YAML spec “flow style”) YAML and the somewhat more idiomatic for most uses indented, (or, in the spec language, “block style&quot;) YAML, e.g., if you find it easier to edit in flow-style but you are in an environment that wants the end product delivered in block style.<p>Notably, whitespace sensitivity is a feature of block style, but not flow style.')
