Item(by='CaptArmchair', descendants=None, kids=None, score=None, time=1603565249, title=None, item_type='comment', url=None, parent=24880952, text='This article does a fair bit of explaining: [1]<p>&gt; The primary thing to understand is that the rc files are for all shell invocations while the profiles are strictly for interactive shells. An interactive shell is where you (end user) types the command while an non-interactive shell is when shells are launched by other programs such as a script with #!&#x2F;bin&#x2F;bash as SHEBANG value<p>[1] <a href="https:&#x2F;&#x2F;www.golinuxcloud.com&#x2F;bashrc-vs-bash-profile&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.golinuxcloud.com&#x2F;bashrc-vs-bash-profile&#x2F;</a><p>So, what are use cases in which a non-interactive shell is launched? One example is a cron jobt that runs every night. It&#x27;s basically the &quot;cron&quot; process executing a command which could be a bash script. Another example would be startup scripts that launch background processes and services which you&#x27;d find in &#x2F;etc&#x2F;system.<p>There&#x27;s a distinction between interactive v. non-interactive and login v non-login as well [2]<p><a href="https:&#x2F;&#x2F;dev.to&#x2F;bijaykumarpun&#x2F;linux-shells-interactive-non-interactive-login-and-non-login-2ddp" rel="nofollow">https:&#x2F;&#x2F;dev.to&#x2F;bijaykumarpun&#x2F;linux-shells-interactive-non-in...</a><p>Typical issue would be adding something to your PATH variable in .profile, having your script work just fine in a shell, but having it fail when you run it as a cron job because .profile never gets sourced by cron.')
