Item(by='pliu', descendants=None, kids=None, score=None, time=1602712712, title=None, item_type='comment', url=None, parent=24780270, text='It would be interesting if the networking model for the end targets could also be inverted, so that an agent (or something) on the end target could make an outbound connection to establish a reverse tunnel to the proxy that user connections could then be sent over.<p>The use case I&#x27;m thinking of is for IoT or robotics, where you have devices you want to manage being deployed into remote networks that you don&#x27;t have much control over. It&#x27;s really helpful in this situation if devices make outbound connections only, so that network operators don&#x27;t have to configure their firewalls to port forward or set up a VPN.<p>Edit: clearer language')
