Item(by='brucehoult', descendants=None, kids=None, score=None, time=1610668400, title=None, item_type='comment', url=None, parent=25767157, text='If you&#x27;re in an embedded situation where you&#x27;re building all the software yourself then that&#x27;s fine.<p>If you&#x27;re on a general purpose PC&#x2F;smartphone with packaged software then the OS vendor specifies a base set of extensions that everything must implement -- for Linux at the moment that is RV64IMAFDC aka RV64GC.<p>All of those extensions (except maybe A) are very generally useful and pervasive in code.<p>Some other extensions, such as the Vector extension, will provide significant benefits to applications that don&#x27;t even know whether the system they are running on has them -- you&#x27;ll just get dynamically linked to a library version that uses V or doesn&#x27;t, as appropriate.<p>To take a very trivial example, on a system with V, every application will automatically use highly efficient (and also very short) V versions of memcpy, memcmp, memset, bzero, strlen, strcpy, strcmp and similar.<p>The same will apply to libraries for bignums, BLAS, jpeg and other media types, and many others.')
