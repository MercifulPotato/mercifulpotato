Item(by='tialaramex', descendants=None, kids=None, score=None, time=1604750775, title=None, item_type='comment', url=None, parent=25012783, text='As well as the affirmatively &quot;insecure&quot; marking in the URL bar for unencrypted HTTP, it&#x27;s not a Secure Context (except localhost) and so modern browsers don&#x27;t offer any new features for this case.<p>If you&#x27;re writing an HTML 4.0 web page that&#x27;s no issue. If you were building a video chat, or you were going to add geolocation to your &quot;Where&#x27;s my closest store?&quot; page or you want to send Notifications, or you&#x27;ve got a framework that needs Service Workers, those APIs are Secure Context only and won&#x27;t exist.<p>The general rule of thumb is, if you could polyfill it, then it&#x27;s available even without Secure Context, otherwise you need Secure Context to get the new feature. Some older features are being retrospectively given this treatment.')
