Item(by='tleb_', descendants=None, kids=None, score=None, time=1604579606, title=None, item_type='comment', url=None, parent=24997512, text='You could just output raw frames into a file and read it using ffplay or vlc. The easiest pixel format is probably RGB 24 bits.<p><pre><code>    ffplay -f rawvideo -pixel_format rgb24 -video_size 42x42 -framerate 42 foo.raw\n    vlc --demux rawvideo --rawvid-fps 42 --rawvid-width 42 --rawvid-height 42 --rawvid-chroma=RV24 --rawvid-aspect-ratio 1:1 foo.raw\n</code></pre>\nI experienced noticeable latency using ffplay so not great for real-time (multiple 100ms, upping the framerate to something like 100-200 reduces the latency but it&#x27;s stupid). I have much better latency sending the frame data to a browser using websockets and displaying it in a canvas...<p>Sources:\n<a href="https:&#x2F;&#x2F;ffmpeg.org&#x2F;ffmpeg-formats.html#rawvideo" rel="nofollow">https:&#x2F;&#x2F;ffmpeg.org&#x2F;ffmpeg-formats.html#rawvideo</a>\n<a href="https:&#x2F;&#x2F;forum.videolan.org&#x2F;viewtopic.php?f=13&amp;t=57938&amp;p=193678" rel="nofollow">https:&#x2F;&#x2F;forum.videolan.org&#x2F;viewtopic.php?f=13&amp;t=57938&amp;p=1936...</a>\n<a href="https:&#x2F;&#x2F;wiki.videolan.org&#x2F;Chroma&#x2F;" rel="nofollow">https:&#x2F;&#x2F;wiki.videolan.org&#x2F;Chroma&#x2F;</a>')
