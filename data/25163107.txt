Item(by='tracyhenry', descendants=None, kids=[25163217, 25168043, 25163564], score=None, time=1605895698, title=None, item_type='comment', url=None, parent=25161866, text='We are building a data vis tool (<a href="https:&#x2F;&#x2F;github.com&#x2F;tracyhenry&#x2F;kyrix" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;tracyhenry&#x2F;kyrix</a>) to solve this exact problem many web vis tools have.<p>The idea is not surprising: using database indexes and a backend to send the frontend necessary data to render <i>on demand</i>. DB and the backend are containerized so the installation, data loading and authoring are all one command.<p>For creators, we offer D3 and JSON-based declarative primitives that enable creation of pan&#x2F;zoom like visualizations very easily (e.g. 10s of lines of JSON for a 15-zoom-level vis). You can check out an interactive demo here, which visualizes 1.88 million wildfires: <a href="http:&#x2F;&#x2F;wildfire.kyrixdemo.live&#x2F;" rel="nofollow">http:&#x2F;&#x2F;wildfire.kyrixdemo.live&#x2F;</a><p>Since big data vis is very use case dependent, I&#x27;d like to reference two other tools that might be useful:<p>Falcon: <a href="https:&#x2F;&#x2F;github.com&#x2F;uwdata&#x2F;falcon" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;uwdata&#x2F;falcon</a>\nNanoCubes: <a href="https:&#x2F;&#x2F;nanocubes.net&#x2F;" rel="nofollow">https:&#x2F;&#x2F;nanocubes.net&#x2F;</a><p>Disclaimer: I&#x27;m still in academia so I&#x27;m apparently biased towards academic works. But fortunately they are all open source.')
