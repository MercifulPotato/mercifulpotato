Item(by='DouweM', descendants=None, kids=None, score=None, time=1611684009, title=None, item_type='comment', url=None, parent=25918209, text='I think that if the wider open source community can maintain API client libraries for every imaginable SaaS API and every popular programming language, there&#x27;s no reason that it can&#x27;t maintain open source ELT connectors for all of these sources as well.<p>I work at GitLab as project lead of Meltano (<a href="https:&#x2F;&#x2F;meltano.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;meltano.com&#x2F;</a>) — which embraces Singer instead of abandoning it — and we&#x27;ve seen a lot of interest from data consultancies looking for mature tooling around deploying and developing Singer taps, many of whom have expressed that they&#x27;d be happy to maintain open source ELT connectors for data sources that are commonly used by their clients, if they can significantly save on ELT costs that would otherwise get passed on to those clients.<p>Of course, only one data consultancy (or data team at a company) would need to maintain an open source tap, and others that need the same source for _their_ clients can contribute and help keep it up to date.')
