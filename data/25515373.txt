I tend to use CSS Grid and Flex almost exclusively, since they are a lot more intuitive than the block and inline combo.<p>But then again if you look at the origin of CSS, it all makes sense. CSS was never developed to be used for app-like interfaces. At it&#x27;s inception, it was intended as a style sheet for documents (specifically <i>scientific</i> documents). The original idea was that everybody would have their own CSS which they could then apply to HTML locally. Interestingly, 30 years later, HTML and CSS are used for almost <i>anything but scientific</i> documents and nobody uses a personal style sheet.<p>The technology is really not well suited for the cases it is most often used for today and there is a superb blog post on the topic that has featured several times on HN:<p>The Blog Post:\n<a href="https:&#x2F;&#x2F;eager.io&#x2F;blog&#x2F;the-languages-which-almost-were-css&#x2F;" rel="nofollow">https:&#x2F;&#x2F;eager.io&#x2F;blog&#x2F;the-languages-which-almost-were-css&#x2F;</a><p>HN 4 years ago:\n<a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=11994405" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=11994405</a><p>HN only 4 months ago:\n<a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24118151" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24118151</a>')
