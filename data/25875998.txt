Item(by='cycloptic', descendants=None, kids=None, score=None, time=1611347471, title=None, item_type='comment', url=None, parent=25875397, text='For whatever reason this is not well documented, but all the theme&#x27;s colors are available with a special syntax in the CSS: <a href="https:&#x2F;&#x2F;gitlab.gnome.org&#x2F;GNOME&#x2F;gtk&#x2F;-&#x2F;blob&#x2F;master&#x2F;gtk&#x2F;theme&#x2F;Adwaita&#x2F;_colors-public.scss" rel="nofollow">https:&#x2F;&#x2F;gitlab.gnome.org&#x2F;GNOME&#x2F;gtk&#x2F;-&#x2F;blob&#x2F;master&#x2F;gtk&#x2F;theme&#x2F;A...</a><p>So for example you should be using this to use the user&#x27;s foreground and background color:<p><pre><code>    color: @theme_fg_color;\n    background-color: @theme_bg_color;\n</code></pre>\nYou can use some modifier functions on those colors too to compute additional colors: <a href="https:&#x2F;&#x2F;developer.gnome.org&#x2F;gtk4&#x2F;stable&#x2F;ch39s02.html" rel="nofollow">https:&#x2F;&#x2F;developer.gnome.org&#x2F;gtk4&#x2F;stable&#x2F;ch39s02.html</a><p>I would suggest against trying to piggy back on built-in CSS styles; usually with the default widgets you want to favor composition over inheritance. But if you really need to you can use the GTK inspector to look at the styles on any given system widget, that should be a bit easier than grepping through the CSS.')
