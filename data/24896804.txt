Item(by='flohofwoe', descendants=None, kids=[24898259], score=None, time=1603724137, title=None, item_type='comment', url=None, parent=24895488, text='Please don&#x27;t, inline code in headers is the single reason why C++ compiles so slowly, and besides: the inline keyword is also available in C (since C99), but that doesn&#x27;t mean it&#x27;s a good idea to use it:<p>The IMPLEMENTATION define cleanly separates the public API declarations (which are required in each source file where functions from this library are called), from the implementation code and any implementation-private declarations.<p>If you put all the implementation code into inline code, this code is parsed over and over again in each source file which would only need to include the declarations.<p>This is the main reason why C++ projects compile so slowly, and what gives &quot;header-only libraries&quot; such a bad reputation in the C++ world.<p>&quot;STB-style single file libraries&quot; don&#x27;t suffer from this problem.')
