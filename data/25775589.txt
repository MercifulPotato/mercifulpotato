Item(by='Wintamute', descendants=None, kids=None, score=None, time=1610627498, title=None, item_type='comment', url=None, parent=25774140, text='I know this pattern and have used it myself. The problem is though, that generally you should only throw an error when you reach an exception - it&#x27;s like saying &quot;I don&#x27;t know what to do next&quot;, and you defer that decision up the stack.<p>Is a request that technically succeeds (the response has come back), but has a status code where the server has indicated something didn&#x27;t work always an _exception_ in your client code? I&#x27;d argue it isn&#x27;t, and should just be handled in the normal control flow of your client code via conditionals.')
