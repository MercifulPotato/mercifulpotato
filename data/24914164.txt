Item(by='jiggawatts', descendants=None, kids=None, score=None, time=1603849981, title=None, item_type='comment', url=None, parent=24910392, text='In PowerShell, this is a native feature of the entire shell and hence scripts and commands.<p>The following prefix in a ps1 script enables the -WhatIf and -Confirm parameters:<p><pre><code>    [CmdletBinding(SupportsShouldProcess=$true)]\n</code></pre>\nTo enable -Confirm by default for scary scripts, just use:<p><pre><code>    [CmdletBinding(SupportsShouldProcess=$true,ConfirmImpact=&#x27;High&#x27;)]\n </code></pre>\nThe nice thing is that in PowerShell, unlike bash, this <i>flows through</i> to the vast majority of other commands. If the script has the snippet above, then you don&#x27;t have to litter it with &quot;if ( $userSaidYes ) { ... }&quot; blocks all over the place.<p>Similarly, PowerShell automatically wires up logic to produce all of the useful modes you might want:<p><pre><code>    [Y] Yes  [A] Yes to All  [N] No  [L] No to All  [S] Suspend\n</code></pre>\nThis is very fiddly to implement manually, and &quot;Suspend&quot; is likely impossible for most shells.<p>See: <a href="https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;powershell&#x2F;scripting&#x2F;learn&#x2F;deep-dives&#x2F;everything-about-shouldprocess?view=powershell-7" rel="nofollow">https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;powershell&#x2F;scripting&#x2F;learn&#x2F;...</a>')
