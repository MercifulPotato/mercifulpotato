Item(by='siraben', descendants=None, kids=[25027520, 25026010, 25029167], score=None, time=1604845505, title=None, item_type='comment', url=None, parent=25024639, text='This is a good overview of NixOS however it&#x27;s very limited and barely scratches the surface of what Nix can do.  I consider it a complete replacement for the &quot;reproducible environment&quot; problem that some programming languages solve (e.g. Python&#x27;s virtualenv, Haskell&#x27;s cabal repl), among others.<p>A simple example of this is when I want to run a Python script from the internet, I can execute<p><pre><code>  nix-shell --run &quot;python3 foo.py&quot; -p &quot;python3.withPackages (ps: with ps; [ numpy ])&quot;\n</code></pre>\nand now I&#x27;m in a shell where numpy is avaliable to the Python 3 interpreter. Or I could test the Haskell QuickCheck library and run<p><pre><code>  nix-shell --run &quot;ghci&quot; -p &quot;haskellPackages.ghcWithPackages (ps: with ps; [ QuickCheck ])&quot; \n</code></pre>\nSee[0] for a way to run programs without even installing them by prefixing them with a comma, e.g. `, hello`. Or what about running any Linux ELF binary by automatically getting their shared dependencies at runtime[1]? Or generating bootable ISOs from your NixOS configuration[2], cross-compiling with little effort[3], and so on.<p>These problems have already been solved to varying degrees in other places, but Nix lets you unify them (think of the huge amounts of libraries downloaded by cargo, cabal, node and so on scattered amongst projects and your home folder, Nix stores everything in &#x2F;nix&#x2F;store&#x2F;) into a single framework.<p>I&#x27;d like to also say that the Nixpkgs repository[4] is super easy to contribute to, just open a PR on GitHub as opposed to sending patches via mailing lists, which is checked via the CI. I&#x27;m not sure if there&#x27;s something analogous on other package repositories but there&#x27;s also a bot[5] that opens thousands of PRs updating and testing packages automatically.<p>[0] <a href="https:&#x2F;&#x2F;github.com&#x2F;Shopify&#x2F;comma" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;Shopify&#x2F;comma</a><p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;Lassulus&#x2F;nix-autobahn" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;Lassulus&#x2F;nix-autobahn</a><p>[2] <a href="https:&#x2F;&#x2F;github.com&#x2F;nix-community&#x2F;nixos-generators" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;nix-community&#x2F;nixos-generators</a><p>[3] <a href="https:&#x2F;&#x2F;matthewbauer.us&#x2F;blog&#x2F;beginners-guide-to-cross.html" rel="nofollow">https:&#x2F;&#x2F;matthewbauer.us&#x2F;blog&#x2F;beginners-guide-to-cross.html</a><p>[4] <a href="https:&#x2F;&#x2F;github.com&#x2F;NixOS&#x2F;nixpkgs&#x2F;" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;NixOS&#x2F;nixpkgs&#x2F;</a><p>[5] <a href="https:&#x2F;&#x2F;github.com&#x2F;r-ryantm" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;r-ryantm</a>')
