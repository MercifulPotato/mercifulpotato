Item(by='stevekemp', descendants=None, kids=None, score=None, time=1605857451, title=None, item_type='comment', url=None, parent=25157413, text='I did something similar for a self-hosted github enterprise installation:<p>* A webhook would be fired at my host when a new pull-request was created, or updated.<p>* When the webhook was received we&#x27;d store details of the repository, the branch-name, and the PR-ID into a queue, from the webhook paylad.<p>* A bunch of worker machines would poll the queue, and when a new message was received they&#x27;d checkout the repository, run &quot;make test&quot;, and add the output of the run as a comment to the PR.<p>This was done as a temporary thing, rather than spinning up Jenkins&#x2F;similar, with the expectation we&#x27;d get Github Actions coming to Github Enterprise in the near future.')
