Item(by='Tade0', descendants=None, kids=[25705905], score=None, time=1610211562, title=None, item_type='comment', url=None, parent=25700135, text='<a href="https:&#x2F;&#x2F;github.com&#x2F;Tade0&#x2F;emergency-poncho" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;Tade0&#x2F;emergency-poncho</a><p>This is a HTTP mock server for reproducing issues on the front-end using a recorded HTTP archive(*.har file).<p>The gimmick is that for a given endpoint it stores all the responses and serves them in a round-robin sequence.<p>This way you can simulate situations where e.g. a request has been retried after the JWT expired, or something special happens when an item is added to a list, and the list is refreshed afterwards - basically every case in which you need backend state.<p>It works... sometimes. Each new project I&#x27;m in uncovers new issues.<p>That being said with the test team giving me both videos and *.har files of the bug reproduction I was able to solve a few long standing bugs in one legacy system.')
