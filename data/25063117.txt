Item(by='Jiocus', descendants=None, kids=None, score=None, time=1605129637, title=None, item_type='comment', url=None, parent=25062275, text='From the comfort of a terminal[1], a few options comes to mind,<p>git-annex[2] will allow you to index all, or just some, of those files where they are - and keep track if you shuffle them around. The really useful feature in your case, is that git-annex will keep tabs on even your disconnected harddrives, flashdrives or cloud storage. It will let you know if you have redundant copies and how many, or if you&#x27;re about to trash the last known instance of IMG001.jpg. It will point you to specific storage media if query some file not currently local.<p>Note that it&#x27;s not entirely as trivial as I make it out to be - git vcs experience helps. Some love it.<p>In your situation, I&#x27;d might try borg[3] - No experience, but I heard appreciative voices about it and docs seem OK.<p>Personally, I always end up using rmlint&#x2F;fdupe and unix tools, but that&#x27;s a secret.<p>[2] <a href="https:&#x2F;&#x2F;git-annex.branchable.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;git-annex.branchable.com&#x2F;</a>\n[2] <a href="https:&#x2F;&#x2F;github.com&#x2F;borgbackup&#x2F;borg" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;borgbackup&#x2F;borg</a> \n[3] There&#x27;s GUI implementations of these')
