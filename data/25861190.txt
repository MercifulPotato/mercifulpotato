It&#x27;s hidden behind a `curl &lt;url&gt; | sh` in their post:<p><pre><code>    #!&#x2F;bin&#x2F;sh\n    bputil -d | grep &quot;CustomerKC&quot; | grep -v &quot;absent&quot;\n    KC=$?\n    if [ $KC -eq 1 ]\n    then\n      bputil -n -k -c -a -s\n      csrutil disable\n      csrutil authenticated-root disable\n    fi\n    cd &#x2F;Volumes&#x2F;Preboot\n    curl https:&#x2F;&#x2F;$LONGURL&#x2F;linux.macho &gt; linux.macho\n    kmutil configure-boot -c linux.macho -v &#x2F;Volumes&#x2F;Macintosh\\ HD&#x2F;\n    echo &quot;Kernel installed. Please reboot&quot;;\n</code></pre>\nSo, it looks like they download their pre-compiled kernel onto a Preboot volume. This is then set as the booting kernel?  How long until there is a grub-like option?<p>I should add -- I don&#x27;t know what most of these commands are. It would probably be helpful if they had spelled these out more, but this is all a great place to start.<p>bputil - Utility to precisely modify the security settings on Apple Silicon Macs.<p>csrutil - Configure System Integrity Protection (SIP)<p>kmutil - replaces kextload, kextunload, and other earlier tools for loading and managing kexts')
