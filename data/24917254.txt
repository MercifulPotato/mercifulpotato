Item(by='an_ko', descendants=None, kids=None, score=None, time=1603881620, title=None, item_type='comment', url=None, parent=24912760, text='During compile-time, it doesn&#x27;t care.  So you can use &quot;illegal&quot; characters for the names of macros.<p><pre><code>  (macro what? (function () (return &#x27;(a b c))))\n  (what?)\n</code></pre>\nThat compiles just fine to the JavaScript code<p><pre><code>  a(b, c);\n</code></pre>\nIt does try to validate identifier names during code generation though (using the <i>esvalid</i> module), so if you give it<p><pre><code>  (? 1 2)\n</code></pre>\nthen it will error with<p><pre><code>  [Error] Identifier `name` member must be a valid IdentifierName\n  At line 1, offset 1:\n\n  (? 1 2)\n</code></pre>\n(In a terminal, the offending character is highlighted.)')
