Item(by='volta87', descendants=None, kids=None, score=None, time=1611658401, title=None, item_type='comment', url=None, parent=25913999, text='From the POV of an HPC cluster user, when using SLURM&#x27;s `srun` or similar to schedule a job, this now allows you to use `srun --container=&lt;your container&gt;`, and it will start each node where you app run using the container, and make sure MPI, GPUs, etc, all work.<p>If you don&#x27;t know anything about containers, it probably will be a bit hard to imagine what this buys you, but don&#x27;t worry, as more clusters start moving towards this model, you&#x27;ll have to learn about containers at some point.<p>From the POV of the HPC cluster, it means that the `module` system can be replaced with containers, and that can significantly lower the maintenance overhead of the cluster. In a sense, it turns HPC cluster users into HPC cluster maintainers (that have to build their own images, preparing their own environment, etc).')
