Item(by='Someone', descendants=None, kids=[25420196], score=None, time=1607887410, title=None, item_type='comment', url=None, parent=25407297, text='<i>“It, like the semicolon, always has been and will be.”</i><p>Not the best of examples. It hasn’t always been. Early fortran and lisp didn’t have it. Reading <a href="https:&#x2F;&#x2F;medium.com&#x2F;better-programming&#x2F;a-brief-history-of-the-8efda9dde2b8" rel="nofollow">https:&#x2F;&#x2F;medium.com&#x2F;better-programming&#x2F;a-brief-history-of-the...</a>, it seems Algol and PL&#x2F;I set the trend to make it popular.<p>Looking at newer languages, I also get the impression it’s on the way out. Rust has them, but python, scala, go and swift try to avoid them (<a href="https:&#x2F;&#x2F;python-reference.readthedocs.io&#x2F;en&#x2F;latest&#x2F;docs&#x2F;operators&#x2F;semicolon.html" rel="nofollow">https:&#x2F;&#x2F;python-reference.readthedocs.io&#x2F;en&#x2F;latest&#x2F;docs&#x2F;opera...</a>: <i>“Avoid having multiple statements on a single line. Though the language definition allows one to use a semi-colon to delineate statements, doing so without reason makes one’s code harder to read”</i>, <a href="https:&#x2F;&#x2F;golang.org&#x2F;doc&#x2F;effective_go.html#semicolons" rel="nofollow">https:&#x2F;&#x2F;golang.org&#x2F;doc&#x2F;effective_go.html#semicolons</a>: <i>“Like C, Go&#x27;s formal grammar uses semicolons to terminate statements, but unlike in C, those semicolons do not appear in the source. Instead the lexer uses a simple rule to insert semicolons automatically as it scans, so the input text is mostly free of them.”</i>, <a href="https:&#x2F;&#x2F;docs.swift.org&#x2F;swift-book&#x2F;LanguageGuide&#x2F;TheBasics.html#ID316" rel="nofollow">https:&#x2F;&#x2F;docs.swift.org&#x2F;swift-book&#x2F;LanguageGuide&#x2F;TheBasics.ht...</a>: <i>“Unlike many other languages, Swift doesn’t require you to write a semicolon (;) after each statement in your code, although you can do so if you wish. However, semicolons are required if you want to write multiple separate statements on a single line”</i>)')
