Item(by='robocat', descendants=None, kids=[25269187], score=None, time=1606852137, title=None, item_type='comment', url=None, parent=25268388, text='However if the translation layer is designed for consistency, like Babelfish, it would have to replicate many of the MySQL “faults” too... e.g. it would need to have the same non-transactional DDL so it acted “correctly”!<p>And it would need to replicate the specialised data types like utf8mb3.<p>To use PostgreSQL features (such as JSON datatype) would presumably need a separate data connection and transaction, because if it isn’t implemented by MySQL, then the syntax wouldn’t be supported by Babelfish.<p>You would get some wins, but you don’t get a blend of the best of both.')
