Item(by='chrismorgan', descendants=None, kids=[25135412], score=None, time=1605695473, title=None, item_type='comment', url=None, parent=25134678, text='One important technical reason is that you need to be able to add attributes to modules, like these examples:<p><pre><code>  &#x2F;&#x2F; Only compile this code if the “foo” Cargo feature is enabled.\n  #[cfg(feature = &quot;foo&quot;)]\n  pub mod foo;\n\n  &#x2F;&#x2F; Platform-specific stuff, as with std::os.\n  pub mod os {\n      #[cfg(windows)]\n      pub mod windows;\n\n      #[cfg(unix)]\n      pub mod unix;\n  }\n\n  &#x2F;&#x2F; You can even choose which file to load the module from.\n  #[cfg_attr(windows, path = &quot;windows&#x2F;mod.rs&quot;)]\n  #[cfg_attr(unix, path = &quot;unix&#x2F;mod.rs&quot;)]\n  #[cfg_attr(not(any(windows, unix)), path = &quot;null&#x2F;mod.rs&quot;)]\n  mod platform_impl;\n</code></pre>\nSo they need to at least be <i>addressable</i>.')
