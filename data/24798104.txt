There are open source and closed source ones.<p>With the closed source ones you can take a leap of faith and trust the developer of the project, how it&#x27;s documented, what it does, and decide whether or not to use it. If you submit to an app store, they can decide whether to approve it.<p>There are closed-source third-party component libraries out there that have a lot less info about the project and the creator than this one, that get distributed to end users.<p>GitHub has quite a large and varied audience. Some expect GitHub to have only open source software that can be vetted by examining the source code. For those it&#x27;s a red flag. There are plenty of other developers for which it isn&#x27;t a red flag, though.')
