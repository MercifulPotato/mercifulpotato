Item(by='flohofwoe', descendants=None, kids=None, score=None, time=1603104568, title=None, item_type='comment', url=None, parent=24824962, text='IMHO it&#x27;s less of a security feature, and more of a debugging aid (the overlap with security is when a failed check causes program termination). You might get more fine-grained overflow checks (vs guard pages), or dangling pointer checks (depending on how that key is stored, e.g. more than one key per memory page?).<p>It&#x27;s faster then doing the same in code (e.g. via &#x27;tagged handles&#x27;: <a href="https:&#x2F;&#x2F;floooh.github.io&#x2F;2018&#x2F;06&#x2F;17&#x2F;handles-vs-pointers.html" rel="nofollow">https:&#x2F;&#x2F;floooh.github.io&#x2F;2018&#x2F;06&#x2F;17&#x2F;handles-vs-pointers.html</a>), or memory debugging tools like Valgrind or clang ASAN (and unlike Valgrind or ASAN, the checks are always enabled, also in the &quot;release version&quot;).')
