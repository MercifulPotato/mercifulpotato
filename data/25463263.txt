Especially when you&#x27;re saving a large file often, making constant single changes to ZIP files is just inefficient.<p>2) The insane memory gulps that ZIP takes.<p>3) The pile of files method. The data in OpenDocument files could be much better sorted&#x2F;accessed through SQLite tables, rather than through XML files in ZIPs.<p>I think SQLite makes far more sense for OpenDocument. Incidentally, SQLite has also made good arguments previously for its use to replace application file formats: <a href="https:&#x2F;&#x2F;www.sqlite.org&#x2F;appfileformat.html" rel="nofollow">https:&#x2F;&#x2F;www.sqlite.org&#x2F;appfileformat.html</a>')
