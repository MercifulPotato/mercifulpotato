Item(by='HighChaparral', descendants=None, kids=None, score=None, time=1611879003, title=None, item_type='comment', url=None, parent=25943600, text='That&#x27;s really interesting, can&#x27;t wait to take a look.<p>There&#x27;s a good webos plugin for homebridge (<a href="https:&#x2F;&#x2F;github.com&#x2F;merdok&#x2F;homebridge-webos-tv" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;merdok&#x2F;homebridge-webos-tv</a>) which is based on LGTV2 (<a href="https:&#x2F;&#x2F;github.com&#x2F;hobbyquaker&#x2F;lgtv2" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;hobbyquaker&#x2F;lgtv2</a>) which exposes a ton of features, including the ability to instantly switch between apps&#x2F;inputs. I have my Harmony calling this via my fork of Harmony Span (<a href="https:&#x2F;&#x2F;github.com&#x2F;garethflynn&#x2F;harmony-span" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;garethflynn&#x2F;harmony-span</a>) which very cleverly impersonates a Roku device via SSDP to allow Harmony keypresses to be captured by a server (a Raspberry Pi in my case) and used to run a custom shell script.<p>It sounds super-fussy, but it actually delivers a really clean way of switching inputs, especially in combination with my Samsung soundbar which, amazingly, only offers direct input selection via http (you can only cycle through them with the remote).')
