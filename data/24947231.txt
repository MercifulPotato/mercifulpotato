Item(by='iscfrc', descendants=None, kids=[24947429], score=None, time=1604094754, title=None, item_type='comment', url=None, parent=24946601, text='You might want to try enabling jumbo frames by setting the MTU to something &gt;1500 bytes. Doing so should reduce the number of IRQs per unit of time since each frame will be carrying more data and therefore there will be fewer of them.<p>According to the Intel 82580EB datasheet[1] it supports an MTU of &quot;9.5KB.&quot; It&#x27;s unclear if that means 9500 or 9728 bytes.<p>I looked briefly for a datasheet that includes the ethernet specs. of the Broadcom 2711 but didn&#x27;t immediately find anything.<p>Recent versions of iproute2 can output the maximum MTU of an interface via:<p><pre><code>  # Look for &quot;maxmtu&quot; in the output\n  ip -d link list\n</code></pre>\nBarring that you can try incrementally upping the MTU until you run in to errors.<p>The MTU of an interface can be set via:<p><pre><code>  ip link set $interface mtu $mtu\n</code></pre>\nNote that for symmetrical testing via direct crossover you&#x27;ll want to have the MTU be the same on each interface pair.<p>[1] <a href="https:&#x2F;&#x2F;www.intel.com&#x2F;content&#x2F;www&#x2F;us&#x2F;en&#x2F;embedded&#x2F;products&#x2F;networking&#x2F;82580-eb-db-gbe-controller-datasheet.html" rel="nofollow">https:&#x2F;&#x2F;www.intel.com&#x2F;content&#x2F;www&#x2F;us&#x2F;en&#x2F;embedded&#x2F;products&#x2F;ne...</a> (pg. 25, &quot;Size of jumbo frames supported&quot;)')
