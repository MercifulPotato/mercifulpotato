You can either query the S3 file directly [1], or define an external table on top of it [2]. If the data is JSON, Avro, XML, or some other &quot;semi-structured&quot; format, then Snowflake&#x27;s &quot;variant&quot; data type makes this type of data very easy to work with. For raw text or log lines, you can still load to Snowflake and process using regex or UDFs. For unstructured or &quot;complex&quot; data like images and PDFs, then you may need to reach for a different compute environment.<p>[1] <a href="https:&#x2F;&#x2F;docs.snowflake.com&#x2F;en&#x2F;user-guide&#x2F;querying-stage.html" rel="nofollow">https:&#x2F;&#x2F;docs.snowflake.com&#x2F;en&#x2F;user-guide&#x2F;querying-stage.html</a>\n[2] <a href="https:&#x2F;&#x2F;docs.snowflake.com&#x2F;en&#x2F;user-guide&#x2F;tables-external-intro.html" rel="nofollow">https:&#x2F;&#x2F;docs.snowflake.com&#x2F;en&#x2F;user-guide&#x2F;tables-external-int...</a>')
