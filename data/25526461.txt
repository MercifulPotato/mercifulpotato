Item(by='beagle3', descendants=None, kids=None, score=None, time=1608806444, title=None, item_type='comment', url=None, parent=25521301, text='&gt; I really despise the &#x27;common&#x27; module approach because it scatters code that should be in the same file.<p>If you have circular dependencies then, in my opinion and experience, it <i>unifies</i> code that should be in the same file, and which was <i>wrongly</i> scattered among different ones.<p>I remember reading the release notes on Turbo Pascal 6 (or was it 5? or Borland Pascal 7) with a weird half-assed support for circular type dependencies among different units, and trying to find a case where putting all the definitions in one file (and avoiding all these issues) wouldn&#x27;t make more sense to me. It&#x27;s been 30 years since, and I haven&#x27;t seen an example as such yet.<p>I acknowledge there&#x27;s no accounting for taste, of course.')
