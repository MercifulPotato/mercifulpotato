Item(by='bloblaw', descendants=None, kids=[25410219], score=None, time=1607884915, title=None, item_type='comment', url=None, parent=25408662, text='This was written a few years ago with FPC v2.2 (latest is FPC v3.2), but generally matches my experience:\n<a href="https:&#x2F;&#x2F;wiki.freepascal.org&#x2F;Size_Matters#Analysis_of_various_options" rel="nofollow">https:&#x2F;&#x2F;wiki.freepascal.org&#x2F;Size_Matters#Analysis_of_various...</a><p>tl;dr is that if you create a RELEASE build, should be about 2 - 5mb for a GUI app, depending on the platform. And if you run `upx` against the resulting binary, it will be even smaller.<p><pre><code>  Win 7 x64:  \n  ------------  \n  - 12.3 mb == Default build      \n  -  3.2 mb == Default build without debug info\n  -  2.1 mb == Optimized compiler\n  -  2.1 mb == Optimized compiler without debug info\n  \n  Ubuntu x64:  \n  ------------  \n  - 13.4 mb == Default build     \n  -  7.5 mb == Default build without debug info\n  -  4.4 mb == Optimized compiler\n  -  2.7 mb == Optimized compiler without debug info\n</code></pre>\nOptimized compiler means:<p><pre><code>    1. Project|Compiler Options|Code|Smart Linkable (-CX) -&gt; Checked\n    2. Project|Compiler Options|Linking|Debugging| Uncheck all except `Strip Symbols From Executable (-Xs) -&gt; Checked`Strip Symbols From Executable (-Xs) -&gt; Checked\n    3. Project|Compiler Options|Linking|Link Style|Link Smart (-XX) -&gt; Checked\n    4. (Optional) Project|Compiler Options|Code|Optimizations|smaller rather than faster -&gt; Checked (Warning: this might decrease performance)</code></pre>')
