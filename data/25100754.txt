Item(by='pabs3', descendants=None, kids=None, score=None, time=1605444324, title=None, item_type='comment', url=None, parent=25099862, text='An alternative approach might be to create a way for the community of folks using the software to help each other, you might even get some users turn into contributors that way. This is how the user support for Linux distros works. For Debian at least, I have seen some of the more advanced users in the support channels move up to maintaining packages as a result of the support channel existing.<p>GitHub now has a discussions feature that could be used for this. Perhaps the right approach might be to open discussions for all GitHub users, open issues solely for project admins&#x2F;maintainers and open PRs for all GitHub users. Folks who want to provide support can look at the discussions tab, reformulate valid bugs into issues and then maintainers&#x2F;developers can turn those into patches.<p><a href="https:&#x2F;&#x2F;github.blog&#x2F;2020-05-06-new-from-satellite-2020-github-codespaces-github-discussions-securing-code-in-private-repositories-and-more&#x2F;" rel="nofollow">https:&#x2F;&#x2F;github.blog&#x2F;2020-05-06-new-from-satellite-2020-githu...</a>')
