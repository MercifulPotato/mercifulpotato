Item(by='janoc', descendants=None, kids=[25409867], score=None, time=1607888387, title=None, item_type='comment', url=None, parent=25409093, text='NEON is not API, it is literally the SIMD instruction set for ARM.<p><a href="https:&#x2F;&#x2F;developer.arm.com&#x2F;architectures&#x2F;instruction-sets&#x2F;simd-isas&#x2F;neon" rel="nofollow">https:&#x2F;&#x2F;developer.arm.com&#x2F;architectures&#x2F;instruction-sets&#x2F;sim...</a><p>So x86 is certainly not going to adopt it.<p>The GPU&#x2F;neural stuff is basically nothing else but SIMD, just in slightly different guise and running on different hardware.<p>On PC it is the difference between MMX&#x2F;AVX (CPU) and shaders&#x2F;CUDA&#x2F;&quot;compute&quot; (GPU), on M1 which is all-in-one SoC it is just a different part of the CPU being used, optimized for somewhat different tasks.<p>SIMD is only a technical term for one type of parallel computation - single instruction, multiple data. It is not some sort of Intel-specific magic technology (and the term far predates Intel&#x27;s support for it, going back to vector processing on Crays and such).')
