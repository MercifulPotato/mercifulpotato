On Fedora I suppose that&#x27;s DNF or RPM now, but it used to be yum or RPM.<p>Some analogous folders between Windows and Linux (the File Hierarchy Standard):<p>System-wide binaries (.exe) for userland software on Windows:<p>C:\\Program Files (x86)\\<p>C:\\Program Files\\<p>System-wide binaries for userland software on Linux:<p>&#x2F;usr&#x2F;bin&#x2F;<p>&#x2F;usr&#x2F;local&#x2F;bin&#x2F;<p>System-wide configuration files on Windows:<p>C:\\Program Data\\<p>System-wide configuration files on Linux:<p>&#x2F;etc&#x2F;<p>Per-user binaries (.exe) on Windows:<p>C:\\Users\\JillS\\AppData\\Local\\<p>C:\\Users\\JillS\\AppData\\Local\\Programs\\<p>Per-user binaries on Linux:<p>&#x2F;home&#x2F;jills&#x2F;.local&#x2F;bin&#x2F;<p>(also various user-specified locations under &#x2F;home&#x2F;jills&#x2F;)<p>Per-user configuration files on Windows:<p>C:\\Users\\JillS\\<p>C:\\Users\\JillS\\AppData\\Roaming\\<p>Per-user configuration files on Linux:<p>&#x2F;home&#x2F;jills&#x2F;.config&#x2F;<p>(A different location can be specified on most Linux desktop setups (i.e., those that use the X Window System) by changing environment variable $XDG_CONFIG_HOME to equal the desired location)<p>Admin (root) binaries on Windows:<p>C:\\Windows\\<p>Root (admin) binaries on Linux:<p>&#x2F;bin&#x2F;<p>&#x2F;sbin&#x2F;<p>The environment variable PATH works the same in principle on both Windows and Linux. On Windows you change PATH in the GUI &quot;Edit environment variables for your account&quot;. On Linux you typically use a text editor to open one of the standard &quot;dot&quot; files in &#x2F;home&#x2F;jills&#x2F;, such as .profile or .bashrc, and specify PATH in there.')
