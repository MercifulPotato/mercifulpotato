Starlark is not IIRC. Dhall I don&#x27;t know, but I would presume it is?<p>Nix as an example:<p><pre><code>  nix-repl&gt; { foo = 5 &#x2F; 0; bar = 5; }    \n  error: division by zero, at (string):1:9\n\n  nix-repl&gt; { foo = 5 &#x2F; 0; bar = 5; }.bar \n  5\n</code></pre>\nvs. Python as an obvious example of a language with eager evaluation:<p><pre><code>  &gt;&gt;&gt; { &quot;foo&quot;: 5 &#x2F; 0, &quot;bar&quot;: 5 }.bar\n  Traceback (most recent call last):\n    File &quot;&lt;stdin&gt;&quot;, line 1, in &lt;module&gt;\n  ZeroDivisionError: division by zero\n</code></pre>\nThis lazy evaluation allows for a very nice construct in Jsonnet:<p><pre><code>  local widget = {\n    id:: error &quot;id must be set&quot;,\n    name: &quot;widget-%d&quot; % [self.id],\n    url: &quot;https:&#x2F;&#x2F;factory.com&#x2F;widget&#x2F;%d&quot; % [self.id],\n  };\n  widgetStandard: widget { id: 42 },\n  widgetSpecial: widget { name: &quot;foo&quot;; url: &quot;https:&#x2F;&#x2F;foo.com&quot; },\n</code></pre>\nWhen the resulting code only expects a widget to have a &#x27;name&#x27; and &#x27;url&#x27; field, you can either have both automatically defined based on a single to-level ID, or override them, even fully skipping the ID if not needed. (a :: in jsonnet is a hidden field, ie. one that will not be evaluated automatically when generating a YAML&#x2F;JSON&#x2F;..., but can be evaluated by other means).')
