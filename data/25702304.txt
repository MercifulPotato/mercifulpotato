Item(by='lukadante3', descendants=None, kids=[25702461], score=None, time=1610213333, title=None, item_type='comment', url=None, parent=25700135, text='It&#x27;s a pipeline engine for Kubernetes[1]. The idea is to build a tool that can be flexible enough for CI&#x2F;CD, but also used for other things such as for example data processing.<p>The pipelines are just designed as serial and parallel execution of pods with no fancy features (i.e. loops or conditions) and the idea is to let other tools generate more complex pipelines using code.<p>Responsibility of triggering the pipelines is decoupled to standalone controllers such as github-screener[2] which enables triggering of pipelines used for CI&#x2F;CD.<p>The project currently builds itself, but the docs are a bit outdated and there&#x27;s quite a lot progress to be made.<p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;kuberik&#x2F;engine" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;kuberik&#x2F;engine</a><p>[2] <a href="https:&#x2F;&#x2F;github.com&#x2F;kuberik&#x2F;github-screener" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;kuberik&#x2F;github-screener</a><p>[3] <a href="https:&#x2F;&#x2F;kuberik.io" rel="nofollow">https:&#x2F;&#x2F;kuberik.io</a>')
