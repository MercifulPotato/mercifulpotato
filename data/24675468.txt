Item(by='smcl', descendants=None, kids=[24675729], score=None, time=1601768016, title=None, item_type='comment', url=None, parent=24675004, text='You were right - Javascript lacks a modulo operator. Javascript <i>does</i> have a `%` operator, and in many languages that performs the modulo operation ... but in JS it actually calculates the remainder. This is confusing because in some situations remainder and modulo look similar, but they&#x27;re not. So if we compare JS and Python (where `%` means &quot;modulo&quot;) ...<p><pre><code>    $ python3\n    Python 3.7.3 (default, Jul 25 2020, 13:03:44) \n    [GCC 8.3.0] on linux\n    Type &quot;help&quot;, &quot;copyright&quot;, &quot;credits&quot; or &quot;license&quot; for more information.\n    &gt;&gt;&gt; 10 % 3\n    1\n    &gt;&gt;&gt; 10 % -3\n    -2\n    &gt;&gt;&gt; \n    $ node\n    &gt; 10 % 3\n    1\n    &gt; 10 % -3\n    1\n</code></pre>\nIn the case we&#x27;re talking about - is_odd - I don&#x27;t think it matters, but there is a difference and it could cause some surprises if you&#x27;re not prepared.<p>edit: I called this operation &quot;remainder&quot; ... but you don&#x27;t exactly say &quot;10 remainder 3&quot; for this operation. I don&#x27;t know what to call it, &quot;divide-by-and-return-absolute-of-whats-left&quot; is not very catchy')
