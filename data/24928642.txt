Item(by='brunoluiz', descendants=None, kids=[24930641], score=None, time=1603963670, title=None, item_type='comment', url=None, parent=24928316, text='Is a service for feature flags, where feature flags are a software pattern in a way. If you want to learn more about it, have a look into this: <a href="https:&#x2F;&#x2F;martinfowler.com&#x2F;articles&#x2F;feature-toggles.html" rel="nofollow">https:&#x2F;&#x2F;martinfowler.com&#x2F;articles&#x2F;feature-toggles.html</a><p>In a nutshell, it allows developers to deploy new features but hide them based on a flag. Once it is ready to be released (example: product gave an ok, big announcement happened etc), then this feature just needs to be enabled through this flag. Now, this flag can be an environment variable, but for apps you will probably need a service, which the app always fetch the latest feature configs.<p>We use it here (although using Flaggio <a href="https:&#x2F;&#x2F;github.com&#x2F;victorkt&#x2F;flaggio" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;victorkt&#x2F;flaggio</a>) and it is super useful for continuous delivery. And you can as well control features based on groups (example: show just for beta users) or hide features if some bug was discovered.')
