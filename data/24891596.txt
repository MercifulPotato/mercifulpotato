Item(by='comex', descendants=None, kids=[24891903], score=None, time=1603672698, title=None, item_type='comment', url=None, parent=24891066, text='&gt; That&#x27;s an unnecessarily abrasive view: the Wayland protocol designers do not hate power users. But allowing programs to constantly take in arbitrary input &amp; output information in the background, as well as simulate arbitrary input to other programs, is an obvious and glaring security flaw. Unfortunately, that forbids general-purpose screenshotting and key-rebinding programs - but there is no sensible middle ground.<p>Of course there&#x27;s a middle ground: require special permission for programs that wish to do those things.  For example, macOS has permission prompts for &quot;[Application] would like to record this computer&#x27;s screen.&quot; and &quot;[Application] would like to control this computer using accessibility features.&quot;')
