Item(by='ParadigmComplex', descendants=None, kids=[25989273], score=None, time=1612191317, title=None, item_type='comment', url=None, parent=25988506, text='My understanding of Bedrock Linux&#x27;s security concerns are addressed in this FAQ entry [0].  In the decade or so I&#x27;ve been working on Bedrock, I have yet to come across any situations where the differing assumptions between distros results in a security issue.  Historically, every time assumptions differed in a way Bedrock did not handle, this resulted in things locking down rather than opening up.  For a concrete recent example, Void Linux&#x27;s build of OpenDoas updated recently [1], which resulted in removing Bedrock entries from the $PATH.<p>The primary concern I personally hold is in Bedrock&#x27;s code to bootstrap parts of other distros, which does not currently verify the initial set of files used.  I am hoping to address this with Bedrock&#x27;s next major release (0.8.X) [2].<p>When Bedrock is far enough along that the architecture&#x2F;code churn slows, I plan to seriously investigate the possibility of independent security reviews.  For the time being, if someone claims heightened security needs, I usually redirect them to Qubes OS [3].<p>[0] <a href="https:&#x2F;&#x2F;bedrocklinux.org&#x2F;faq.html#security" rel="nofollow">https:&#x2F;&#x2F;bedrocklinux.org&#x2F;faq.html#security</a>.\n[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;Duncaen&#x2F;OpenDoas&#x2F;releases&#x2F;tag&#x2F;v6.8.1" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;Duncaen&#x2F;OpenDoas&#x2F;releases&#x2F;tag&#x2F;v6.8.1</a>\n[2] <a href="https:&#x2F;&#x2F;github.com&#x2F;bedrocklinux&#x2F;bedrocklinux-userland&#x2F;issues&#x2F;207" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;bedrocklinux&#x2F;bedrocklinux-userland&#x2F;issues...</a>\n[3] <a href="https:&#x2F;&#x2F;www.qubes-os.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.qubes-os.org&#x2F;</a>')
