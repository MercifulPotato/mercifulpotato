Item(by='jlokier', descendants=None, kids=[24760035], score=None, time=1602505989, title=None, item_type='comment', url=None, parent=24750248, text='&gt; Or maybe it&#x27;s impossible to implement that hash function reversibly without preserving a copy of the input in the output.<p>Cryptographic hash functions rely on locally non-reversible non-linear operations.[1]<p>So you don&#x27;t need to copy the original input, but you will need to copy bits here and there throughout the hash function, which allow the input to be reconstructed.<p>But that&#x27;s not surprising, even a humble AND gate is non-reversible.  So even a humble AND gate program will need to copy its inputs to the output if it&#x27;s to be reversible.<p>[1] (As well, cryptographic hash functions are generally not reversible anyway.  As in, different inputs hash to the same output value, so there&#x27;s no unambiguous reversal.)')
