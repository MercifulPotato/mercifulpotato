Item(by='TheDong', descendants=None, kids=[25911364, 25911365], score=None, time=1611620344, title=None, item_type='comment', url=None, parent=25909765, text='You don&#x27;t have to give up the convenience of it if you don&#x27;t want to.<p>If you&#x27;re using zsh, it has the &quot;command_not_found&quot; option, which will be triggered for shell commands that aren&#x27;t found, but won&#x27;t be triggered for subcommands or non-interactive shells.<p>If you add the following as your command not found handler, you&#x27;ll be able to keep on typing &quot;a.out&quot; instead of &quot;.&#x2F;a.out&quot; without the security issue.<p><pre><code>    command_not_found_handler() { [ -x &quot;.&#x2F;$0&quot; ] &amp;&amp; exec &quot;.&#x2F;$@&quot; }\n</code></pre>\nBash has a similar mechanism (command_not_found_handle) as well.')
