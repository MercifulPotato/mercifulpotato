Item(by='jabo', descendants=None, kids=None, score=None, time=1607930547, title=None, item_type='comment', url=None, parent=25414389, text='Quick context: I built this from the Open Library (Internet Archive) books dataset (<a href="https:&#x2F;&#x2F;openlibrary.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;openlibrary.org&#x2F;</a>), as a follow up to this comment on the GoodReads post earlier today: <a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=25408186" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=25408186</a><p>Thank you @mekarpeles for helping me get access to the data quickly and giving me pointers about the schema.<p>&gt; Details about the Tech Stack:<p>The dataset has ~28.6 million books and is indexed on Typesense [1], an open source alternative to Algolia&#x2F;ElasticSearch that a friend and I are working on.<p>The UI was built using the Typesense adapter for InstantSearch.js [2] and is a static site bundled using ParcelJS.<p>The app is hosted on S3, with CloudFront for a CDN.<p>The search backend is powered by a geo-distributed 3-node Typesense cluster running on Typesense Cloud [3], with nodes in Oregon, Frankfurt and Mumbai.<p>Here&#x27;s the source code: <a href="https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;showcase-books-search" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;showcase-books-search</a><p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense</a><p>[2] <a href="https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense-instantsearch-adapter" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense-instantsearch-adapter</a><p>[3] <a href="https:&#x2F;&#x2F;cloud.typesense.org" rel="nofollow">https:&#x2F;&#x2F;cloud.typesense.org</a>')
