Item(by='deathanatos', descendants=None, kids=None, score=None, time=1611242653, title=None, item_type='comment', url=None, parent=25858141, text='In the specific case of the Transfer-Encoding header, it is defined as,<p><pre><code>     Transfer-Encoding = 1#transfer-coding\n\n     transfer-coding    = &quot;chunked&quot; ; Section 4.1\n                        &#x2F; &quot;compress&quot; ; Section 4.2.1\n                        &#x2F; &quot;deflate&quot; ; Section 4.2.2\n                        &#x2F; &quot;gzip&quot; ; Section 4.2.3\n                        &#x2F; transfer-extension\n     transfer-extension = token *( OWS &quot;;&quot; OWS transfer-parameter )\n     token = 1*tchar\n     tchar = &quot;!&quot; &#x2F; &quot;#&quot; &#x2F; &quot;$&quot; &#x2F; &quot;%&quot; &#x2F; &quot;&amp;&quot; &#x2F; &quot;&#x27;&quot; &#x2F; &quot;*&quot; &#x2F; &quot;+&quot; &#x2F; &quot;-&quot; &#x2F; &quot;.&quot; &#x2F;\n             &quot;^&quot; &#x2F; &quot;_&quot; &#x2F; &quot;`&quot; &#x2F; &quot;|&quot; &#x2F; &quot;~&quot; &#x2F; DIGIT &#x2F; ALPHA\n</code></pre>\n… which does not permit non-ASCII. As a generic header, maybe (but the decoding should be into ISO-8859-1, as the RFC notes…), but at the point at which you parse it into a Transfer-Encoding header, it is no longer valid.')
