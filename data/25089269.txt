Item(by='Liskni_si', descendants=None, kids=None, score=None, time=1605314752, title=None, item_type='comment', url=None, parent=25078176, text='Note that just updating Cairo will not automatically bring subpixel positioning to Gtk. Pango still defaults to rounding glyph positions to full pixels (rightly so, as there&#x27;s no released version of Cairo that would render it correctly otherwise), and there&#x27;s absolutely no code in Gtk that would allow disabling this rounding. I reminded Matthias about this half a year ago: <a href="https:&#x2F;&#x2F;gitlab.gnome.org&#x2F;GNOME&#x2F;gtk&#x2F;-&#x2F;issues&#x2F;7#note_795665" rel="nofollow">https:&#x2F;&#x2F;gitlab.gnome.org&#x2F;GNOME&#x2F;gtk&#x2F;-&#x2F;issues&#x2F;7#note_795665</a> but it still hasn&#x27;t been resolved (the issue is marked as closed but that, I believe, is a mistake). I suspect they&#x27;re not going to enable this in Gtk until Cairo with support for subpixel positioning is released.<p>Anyway, I&#x27;ve manually built a snapshot of Cairo for Debian: <a href="https:&#x2F;&#x2F;salsa.debian.org&#x2F;liskin&#x2F;cairo&#x2F;-&#x2F;commits&#x2F;debian&#x2F;master" rel="nofollow">https:&#x2F;&#x2F;salsa.debian.org&#x2F;liskin&#x2F;cairo&#x2F;-&#x2F;commits&#x2F;debian&#x2F;maste...</a> and I have a wrapper around Pango which enables subpixel positioning: <a href="https:&#x2F;&#x2F;github.com&#x2F;liskin&#x2F;dotfiles&#x2F;tree&#x2F;79cef7167a57dde02edf97e35286dc45b1262497&#x2F;src&#x2F;pango-force-subpixel-positioning" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;liskin&#x2F;dotfiles&#x2F;tree&#x2F;79cef7167a57dde02edf...</a>. It really it noticeably prettier! :-)')
