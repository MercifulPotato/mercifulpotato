Item(by='coldpie', descendants=None, kids=None, score=None, time=1607696658, title=None, item_type='comment', url=None, parent=25385539, text='No, Windows provides a stable ABI for decoding PNG images, it&#x27;s called the Windows Imaging Components library[1]. I don&#x27;t think it&#x27;s sustainable to expect every application to ship effectively an entire distro to provide stability. Consider if my app shipped an old version of gnutls, would you be happy to use that? Should it be my responsibility to update it for eternity? With win32, on both Windows and Linux via Wine, you can just use the win32 bcrypt library and assume the underlying implementation is secure and up to date.<p>Win32 is the only solution for a usable, stable, application-level ABI on Linux.
