Item(by='jitl', descendants=None, kids=None, score=None, time=1609686768, title=None, item_type='comment', url=None, parent=25621936, text='Here are some facts I gathered last time Varlink was discussed:<p>Varlink is produced by Red Hat, apparently - or by people who use Red Hat things: both examples (DNF, Podman) are Redhat projects.<p>It seems to be an effort to apply the Interface Definition Language model to UNIX daemons&#x2F;services&#x2F;tools by adding a new “standard IO file descriptor” for <i>structured IO</i> using a typed interface, as opposed to the current STDIN&#x2F;STDOUT&#x2F;STDERR which are existing un typed byte streams.<p>Using a new FD instead of agreeing to layer Varlink protocol over STDIO means that existing programs can adopt Varlink without breaking backwards compat or making negotiation even more complicated.<p>EDIT: Podman removed Varlink <a href="https:&#x2F;&#x2F;podman.io&#x2F;blogs&#x2F;2020&#x2F;01&#x2F;17&#x2F;podman-new-api.html" rel="nofollow">https:&#x2F;&#x2F;podman.io&#x2F;blogs&#x2F;2020&#x2F;01&#x2F;17&#x2F;podman-new-api.html</a>')
