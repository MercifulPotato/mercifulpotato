Item(by='PowerBar', descendants=None, kids=None, score=None, time=1603011418, title=None, item_type='comment', url=None, parent=24778433, text='&gt;&gt; Do not store secrets in environment variables<p>&gt; Yes, definitely don&#x27;t put secrets in the Dockerfile itself. I&#x27;m curious if there are reasons not to use a .env file though?<p>Environment variables are a terrible place to store secrets, regardless of whether you&#x27;re using docker.<p>Environment variable values get dumped all over the place. &#x2F;proc&#x2F;*&#x2F;environ, docker inspect, &#x2F;var&#x2F;log&#x2F;..., core dumps, error messages, info pages (phpinfo), etc. Also, unlike file handles and secrets services (hashicorp vault, etc), every child process inherits all of its parents&#x27; environment variables, greatly increasing the attack surface.')
