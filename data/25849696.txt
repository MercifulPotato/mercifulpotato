They have their reasons[1]. I figured it was time to investigate using Snap and the benefits it could provide.<p>While researching, I found many users reporting that forced updates of software installed by Snap caused many problems and I decided against using it; I was able to install Certbot via a good old-fashioned RPM from EPEL.<p>I also removed Snap from a different Ubuntu server which had recently been upgraded to 20.04 (I wasn&#x27;t using LXD on that server so there was no need for it).<p>1. <a href="https:&#x2F;&#x2F;community.letsencrypt.org&#x2F;t&#x2F;how-to-install-certbot-without-snap&#x2F;137634" rel="nofollow">https:&#x2F;&#x2F;community.letsencrypt.org&#x2F;t&#x2F;how-to-install-certbot-w...</a><p>FWIW, I&#x27;ve been allowing Apt and Yum package managers to automatically update for about 8 years without any problems. The only manual OS updating I do is for a set of physical (non-virtual) servers that are operational 24&#x2F;7.')
