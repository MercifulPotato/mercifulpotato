Item(by='bilkow', descendants=None, kids=None, score=None, time=1610753632, title=None, item_type='comment', url=None, parent=25797383, text='You clearly have a different experience using 1-based index than me, it has caused me many off-by-one mistakes. Also, I almost never need to -1 with 0-based index, but in 1-based index its pretty common, specially if moving slices inside an array, the new index of the element is move_position + old_index - 1 (see example). Also see arguments by Djikstra [0].<p>Example:<p><pre><code>  # python\n  lov = [&#x27;l&#x27;, &#x27;o&#x27;, &#x27;v&#x27;]\n  o_index = 1;\n  array = [&#x27;a&#x27;, &#x27;b&#x27;, &#x27;c&#x27;, &#x27;d&#x27;, &#x27;e&#x27;, &#x27;f&#x27;]\n  move_position = 3\n  # elements moved here\n  array = [&#x27;a&#x27;, &#x27;b&#x27;, &#x27;c&#x27;, &#x27;l&#x27;, &#x27;o&#x27;, &#x27;v&#x27;]\n  new_o_index = move_position + o_index\n  print(array[new_o_index]) # prints &#x27;o&#x27;\n\n  -- lua\n  local lov = { &#x27;l&#x27;, &#x27;o&#x27;, &#x27;v&#x27; }\n  local o_index = 2;\n  local array = { &#x27;a&#x27;, &#x27;b&#x27;, &#x27;c&#x27;, &#x27;d&#x27;, &#x27;e&#x27;, &#x27;f&#x27; }\n  local move_position = 4\n  -- elements moved here\n  array = { &#x27;a&#x27;, &#x27;b&#x27;, &#x27;c&#x27;, &#x27;l&#x27;, &#x27;o&#x27;, &#x27;v&#x27; }\n  local new_o_index = move_position + o_index - 1\n  print(array[new_o_index]) -- prints &#x27;o&#x27;\n</code></pre>\nNote that I&#x27;m not saying 0-based indexing is always absolutely better, but I generally prefer it and believe it usually makes more sense even outside of programming (including math), we&#x27;re just not taught that way.<p>[0] <a href="https:&#x2F;&#x2F;www.cs.utexas.edu&#x2F;users&#x2F;EWD&#x2F;ewd08xx&#x2F;EWD831.PDF" rel="nofollow">https:&#x2F;&#x2F;www.cs.utexas.edu&#x2F;users&#x2F;EWD&#x2F;ewd08xx&#x2F;EWD831.PDF</a>\ntranscript: <a href="https:&#x2F;&#x2F;www.cs.utexas.edu&#x2F;users&#x2F;EWD&#x2F;transcriptions&#x2F;EWD08xx&#x2F;EWD831.html" rel="nofollow">https:&#x2F;&#x2F;www.cs.utexas.edu&#x2F;users&#x2F;EWD&#x2F;transcriptions&#x2F;EWD08xx&#x2F;E...</a>')
