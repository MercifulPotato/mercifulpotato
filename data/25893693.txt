Item(by='wegs', descendants=None, kids=None, score=None, time=1611508567, title=None, item_type='comment', url=None, parent=25889275, text='There is a self-reinforcing network of assumptions in that statement:<p>- Good architecture beats good code.<p>- There are things I can express in dynamic languages which are hard to express in static languages.<p>- If you&#x27;re not expressing those things, typed static languages win.<p>- When you&#x27;re contorting your architecture to fit into static languages, dynamic ones win.<p>- Of course, if you don&#x27;t have experience with dynamic programming, you won&#x27;t think to express certain things or structure code certain ways.<p>There are, of course, academic languages which do both well (including optional static type systems for Lisp), but I&#x27;m not aware of any in widespread industry use (with an ecoystem of libraries, documentation, etc.).')
