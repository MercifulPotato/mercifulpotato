Item(by='TrianguloY', descendants=None, kids=[25461963, 25461959], score=None, time=1608247881, title=None, item_type='comment', url=None, parent=25458230, text='When I need to explain git to someone for the first time I always start with the &#x27;imagine you are copy&#x2F;pasting your project folder, and you have multiple folders tagged with some details. Each folder is a commit&#x27;. People understand very easily how to keep copies of their work at a given time by simply coping the full folder (probably we all did this before using git).<p>So yes, a commit is just a snapshot, a copy, it is simply saved in a much more efficient way.<p>The issue is that some advanced (and not so advanced) commands, like a merge, aren&#x27;t easily explainable with this metaphor, and that&#x27;s when the &#x27;commits are also diffs&#x27;, or better &#x27;commits can be seen as diffs&#x27; is needed.')
