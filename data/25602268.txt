Item(by='iam-TJ', descendants=None, kids=[25602297], score=None, time=1609497327, title=None, item_type='comment', url=None, parent=25595416, text='Many comment threads here discussing the (in)ability of an attacker to modify the source-code that Microsoft builds from, or use it to more easily discover vulnerabilities.<p>What I&#x27;ve not seen anyone discuss is the potential for an attacker to take the source-code of a single Windows core component (a system DLL for example), add in a backdoor, build it and then distribute the binary via a compromise such as the SolarWinds update mechanism.<p>In other words, insert a modified core Windows DLL into some other popular Windows driver or application package updater published and signed via a &#x27;trusted&#x27; channel other than Microsoft itself.')
