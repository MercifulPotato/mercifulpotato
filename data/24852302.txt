Item(by='kazinator', descendants=None, kids=[24856185], score=None, time=1603315131, title=None, item_type='comment', url=None, parent=24851817, text='&gt; <i>A script that calls a piece of software is not a derived work</i><p>If that script relies on that software, it is a derived work. If that script is GPLed, then that software must be GPL-compatible, or else it must be interpreted as falling under the &quot;system library exception&quot;.<p>&gt; <i>all build systems that have ever been used with GCC would have to be GPL-licensed.</i><p>The tools required to build GCC on a proprietary OS can be regarded as landing under the system exception.<p>It would almost certainly be a GPL violation to add a patch to GCC which causes it to require a third-party proprietary program or script in order to build (something not coming from a target system itself).')
