Item(by='kondbg', descendants=None, kids=[25449373, 25448600, 25448505, 25448303, 25448724], score=None, time=1608150836, title=None, item_type='comment', url=None, parent=25445725, text='Devil&#x27;s advocate: why should I choose this yet-to-exist distribution over something already existing, such as Oracle Linux?<p>The most common argument (Oracle is evil and litigious. Therefore, using Oracle Linux will result in me being sued) honestly seems like FUD.<p>All RHEL downstream distributions rebuild the same SRPMs that RHEL provides. Doing a quick comparison over some common packages (kernel, httpd, openssl, etc.) between CentOS 8.3 (<a href="https:&#x2F;&#x2F;vault.centos.org&#x2F;8.3.2011&#x2F;BaseOS&#x2F;Source&#x2F;SPackages&#x2F;" rel="nofollow">https:&#x2F;&#x2F;vault.centos.org&#x2F;8.3.2011&#x2F;BaseOS&#x2F;Source&#x2F;SPackages&#x2F;</a>) Oracle Linux 8.3 (<a href="https:&#x2F;&#x2F;yum.oracle.com&#x2F;repo&#x2F;OracleLinux&#x2F;OL8&#x2F;baseos&#x2F;latest&#x2F;x86_64&#x2F;index_src.html" rel="nofollow">https:&#x2F;&#x2F;yum.oracle.com&#x2F;repo&#x2F;OracleLinux&#x2F;OL8&#x2F;baseos&#x2F;latest&#x2F;x8...</a>) shows that they are indeed byte identical (with the exception of certain spec files including debranding patches).<p>What is the value of having a separate RHEL derivative? It isn&#x27;t as if the &quot;community&quot; can propose&#x2F;submit any changes, since any changes will cease to make the downstream distribution a &quot;bug for bug&quot; compatible RHEL derivative. If I actually wanted to participate in the larger RHEL-derivative community, I would need to actually submit my changes to the CentOS stream project.')
