Item(by='RobKohr', descendants=None, kids=None, score=None, time=1602876397, title=None, item_type='comment', url=None, parent=24779624, text='Keeping it simple until it is needed to expand is the best way forward, and have a little bit of resistance to the expansion.<p>One issue that comes up is the function arguments can evolve to be a mess over time with this method.<p>I like to always use a prop object as function arguments, even when it is simple.<p>function calculateArea({x, y}) rather than (x, y).<p>This makes it so things can be added, and be optional, and the calling function clearly can see what those values are being assigned to.<p>This means that the 100 times you called that function don&#x27;t need to be rewritten with a new parameter added, and you don&#x27;t have to have a bunch of null values set for unused cases.')
