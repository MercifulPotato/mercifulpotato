It&#x27;s an incredibly customizable and scriptable file explorer that allows you to fit it exactly to your workflow. It&#x27;s like IntelliJ of file explorers.<p>- Tabbed interface with multiple panes<p>- Every function can be assigned a shortcut<p>- Every button can be scripted<p>- Defaults can be overwritten<p>- Dynamic layouts based on folder contents, path or determined by a script<p>- Custom file previews, context menus, rename scripts<p>- Hooks for reacting to different events<p>- Every corner of the UI is themable<p>- Virtual filesystem for archiving and bookkeeping<p>- ...<p>One disadvantage is that you have to use an ancient version of JScript (ES3 &#x2F; Microsoft flavor) or VBScript (possibly because of the need for communicating over a COM interface). Or use its own DSL for simpler scripts.<p>I wish it had support for a more modern &#x2F; more popular languages for scripting. I wonder if QuickJS[1] can be integrated into it. Or Python. That would be really sweet.<p>[0]: <a href="https:&#x2F;&#x2F;www.gpsoft.com.au&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.gpsoft.com.au&#x2F;</a><p>[1]: <a href="https:&#x2F;&#x2F;bellard.org&#x2F;quickjs&#x2F;" rel="nofollow">https:&#x2F;&#x2F;bellard.org&#x2F;quickjs&#x2F;</a>')
