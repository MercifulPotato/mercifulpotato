Item(by='tsimionescu', descendants=None, kids=None, score=None, time=1608916143, title=None, item_type='comment', url=None, parent=25537015, text='Since Unix doesn&#x27;t (traditionally at least) offer any access control mechanism more granular than a User, it was only natural that you&#x27;d get a lot of &quot;system users&quot; that are required for all sorts of operations but are not in any way associated with human users.<p>And now, since you have so many system users and almost always a single human user, you start wanting to install programs that will be used by both yourself and those system users, and it would be extremely problematic to go writing into 10 different bin&#x2F; and lib&#x2F; folders to install the same thing for 10 users.<p>Not to mention, a lot of software you want to install also runs some daemons or manages some system settings, and obviously requires root access to do so, but mag run into problems if it is not installed for root.')
