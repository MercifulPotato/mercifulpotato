Item(by='ethbr0', descendants=None, kids=[25458724], score=None, time=1608228715, title=None, item_type='comment', url=None, parent=25458111, text='That feels like an abrogation of responsibility.<p>At this point in computing system complexity, secure defaults over configuration is a requirement for a sane system.<p>As we increasingly convert infrastructure into code, this means infrastructure configuration requires secure defaults as well.<p>That the Docker ecosystem doesn&#x27;t seem overlying concerned (as a core, drop-everything, #1 priority) about a decades-hard security problem (detecting and resolving insecure dependencies) seems shortsighted.<p>To put it another way, there are two realities:<p>(A) No tooling support is provided to address a risk efficiently and quickly = no one addresses the risk<p>(B) Efficient, effective tooling is built, maintained as platform evolves, and advertised to the community = everyone does what they should<p>I realize the structure of Docker dissuades them from actually addressing (B), but it seems like an existential threat to adoption.')
