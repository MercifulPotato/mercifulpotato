Item(by='dfabulich', descendants=None, kids=[24926375, 24925634, 24925795, 24927083, 24925989, 24925993], score=None, time=1603925286, title=None, item_type='comment', url=None, parent=24920702, text='IMO, TurboLinks + service workers are the way to go.<p>Not many people know this, but a service worker (previously called &quot;local server&quot;) allows you to run a little web server in the user&#x27;s browser that intercepts requests to your own web site. (There&#x27;s no open IP port.)<p>The service-worker web server can proxy requests to the remote server, and even build&#x2F;store entire pages on the client side, enabling offline support. Service workers also have access to a local database, IndexedDB, running in the user&#x27;s browser.<p>You can build a <i>very</i> fast web experience this way. You can easily cache individual pieces of a web page and glue them all together.<p>For example, you can easily implement server-side includes <a href="https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Server_Side_Includes" rel="nofollow">https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Server_Side_Includes</a> or edge-side includes <a href="https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Edge_Side_Includes" rel="nofollow">https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Edge_Side_Includes</a> but running on a server that&#x27;s running <i>on the client.</i>')
