Item(by='cesarb', descendants=None, kids=None, score=None, time=1603196186, title=None, item_type='comment', url=None, parent=24836295, text='Annoyingly, .into() is mostly useless for array indexing, since even when you&#x27;re compiling for a 64-bit machine (and thus x[my_u32.into()] would be perfectly fine) it&#x27;s only defined for up to u16 (because you might want the same multi-megabyte code to also work in a tiny microcontroller). If you don&#x27;t want to use x[my_u32 as usize], you are forced to use x[my_u32.try_into().unwrap()], or just use usize everywhere (of course, you could also use traits to create your own .into_usize() and use it everywhere).')
