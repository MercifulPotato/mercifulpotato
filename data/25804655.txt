Item(by='tbrock', descendants=None, kids=[25804854, 25805301], score=None, time=1610824926, title=None, item_type='comment', url=None, parent=25804635, text='Yes, just set up a localforward in your .ssh&#x2F;config like this:<p><pre><code>    Host dev\n        HostName &lt;ip_of_dev_box&gt;\n        User &lt;my_username&gt;\n        ForwardAgent yes\n        AddKeysToAgent yes\n        IdentityFile ~&#x2F;.ssh&#x2F;id_m1_air\n        LocalForward 8443 localhost:8443\n        LocalForward 8080 localhost:8080\n        LocalForward 8065 localhost:8065\n        LocalForward 3000 localhost:3000\n        ...\n</code></pre>\nThose line&#x27;s effectively mean forward my local port 8443 to the remote host&#x27;s 8443. The local port + remote port (and even the remote host) don&#x27;t need to be the same.<p>Since it’s forwarding a TCP port your aren’t limited to forwarding just the web traffic either. I forward:<p><pre><code>    - Database ports (run mysql or psql locally for example)\n    - Docker socket\n    - Backend api ports\n    - Redis port\n    - Webserver port\n</code></pre>\nAnd more...<p>Then you just use local clients as if those things were all running on your local machine')
