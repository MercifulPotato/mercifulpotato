Item(by='The-Compiler', descendants=None, kids=None, score=None, time=1611859724, title=None, item_type='comment', url=None, parent=25942444, text='&gt; since PyQt5-sip is missing from OS packages<p>Arguably this is a Ubuntu (Debian?) packaging bug. Upstream recommends using &quot;PyQt5.sip&quot; ever since PyQt 5.11, yet Ubuntu seems to package PyQt 5.14 but retaining the old &quot;sip&quot; name.<p>I wasn&#x27;t aware of it, or I would&#x27;ve kept the (quite small) compatibility shim in v2.0.0 - I opened an issue here: <a href="https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;6082" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;6082</a><p>In the meantime, you could probably symlink it instead:<p><pre><code>    ln -s &#x2F;usr&#x2F;lib&#x2F;python3&#x2F;dist-packages&#x2F;sip.cpython-38-x86_64-linux-gnu.so &#x2F;usr&#x2F;lib&#x2F;python3&#x2F;dist-packages&#x2F;PyQt5&#x2F; \n</code></pre>\n(Though you&#x27;ll then run into the next issue, apparently Ubuntu 20.04 doesn&#x27;t have importlib_resources packaged: <a href="https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;6084" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;issues&#x2F;6084</a> ...)<p>Alternatively, either use the Ubuntu &quot;qutebrowser&quot; package (pre-v2.0.0), or install in a virtualenv instead:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;blob&#x2F;master&#x2F;doc&#x2F;install.asciidoc#installing-qutebrowser-with-virtualenv" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;qutebrowser&#x2F;qutebrowser&#x2F;blob&#x2F;master&#x2F;doc&#x2F;i...</a><p>There&#x27;s an option there to install a binary Qt&#x2F;PyQt from PyPI, so you don&#x27;t need to build anything from source. However, note that it comes without proprietary codec support.')
