Item(by='ivan_ah', descendants=None, kids=None, score=None, time=1607668862, title=None, item_type='comment', url=None, parent=25381191, text='I&#x27;m pretty proud of this `getvoicerecorder.sh` script that I use to pull .wav voice recordings from my phone, convert them to .mp3, and delete the originals from the phone memory (assuming you have Android Debug Bridge adb tool installed):<p><pre><code>    #!&#x2F;bin&#x2F;bash\n\n    # On the computer\n    MYHOME=$HOME\n    MYDESKTOP=$HOME&#x2F;Desktop\n    DEST_DIR=&quot;Voice Recordings `date &quot;+%Y-%m-%d&quot;`&quot;\n\n    # On the phone\n    VOICE_RECORDER_SOURCE_DIR=&quot;&#x2F;sdcard&#x2F;EasyVoiceRecorder&quot;\n\n\n    # 1. make the dest dir and cd into it\n    cd $MYDESKTOP\n    mkdir &quot;$DEST_DIR&quot;\n    cd &quot;$DEST_DIR&quot;\n    # list files that will be downloaded:\n    echo &quot;The following files will be adb-pulled, converted, and saved to $DEST_DIR&quot;\n    adb shell ls &quot;$VOICE_RECORDER_SOURCE_DIR&quot;\n\n    # 2. PULL\n    adb shell &quot;cd $VOICE_RECORDER_SOURCE_DIR; tar -cf allwavs.tar *wav&quot;\n    adb pull $VOICE_RECORDER_SOURCE_DIR&#x2F;allwavs.tar allwavs.tar\n\n    # 3. PROCESS\n    tar -xf allwavs.tar\n    for f in *.wav; do\n      echo &quot;Converting $f to mp3...&quot;;\n      ffmpeg -hide_banner -loglevel panic    -i &quot;$f&quot; &quot;${f%.wav}.mp3&quot;;\n    done\n    # set timestamp to match what was on the original .wav files\n    for f in *.wav; do\n      gtouch -d &quot;$(date -R -r &quot;$f&quot;)&quot; &quot;${f%.wav}.mp3&quot;;\n    done\n\n    # 4. cleanup\n    rm *wav\n    adb shell &quot;rm $VOICE_RECORDER_SOURCE_DIR&#x2F;allwavs.tar&quot;\n    rm allwavs.tar\n    echo &quot;Deleting all .wav files from phone...&quot;\n    adb shell &quot;rm $VOICE_RECORDER_SOURCE_DIR&#x2F;*wav&quot;\n\n</code></pre>\nIt&#x27;s part of a general authoring workflow: whenever I want to write something in a conversational tone (as opposed to dry&#x2F;technical&#x2F;academic tone), I go for a walk-and-talk, then transcribe the dictated text + edit. Works really well for intros and conclusions in particular, and blog posts.')
