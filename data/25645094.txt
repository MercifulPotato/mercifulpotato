Item(by='magicalhippo', descendants=None, kids=[25645574], score=None, time=1609852196, title=None, item_type='comment', url=None, parent=25644176, text='&gt; I expected to have to do gamma correction when making a gradient recently, but in the end it looked like I didn&#x27;t have to<p>If you don&#x27;t explicitly specify the color space you&#x27;re working in, then you&#x27;re using some implicitly defined color space in which case you basically need to know what that is (at least roughly).<p>So traditionally in Windows, way back, when you created a bitmap, wrote some data to it and then drew it, there was no explicit mention of a color space. Instead it was implicit, and it was de-facto non-linear.<p>These days you can specify[1][2] the color space you&#x27;re working in, and Windows will then transform the colors into the specified device color space. So then you can specify if you want to work in linear RGB space or say non-linear sRGB.<p>Unity has something similar[3], which affects how you write shaders, how your textures should be saved etc.<p>[1]: <a href="https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;previous-versions&#x2F;windows&#x2F;desktop&#x2F;wcs&#x2F;windows-color-system" rel="nofollow">https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;previous-versions&#x2F;windows&#x2F;d...</a><p>[2]: <a href="https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;windows&#x2F;win32&#x2F;api&#x2F;wingdi&#x2F;nf-wingdi-setcolorspace" rel="nofollow">https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;windows&#x2F;win32&#x2F;api&#x2F;wingdi&#x2F;nf...</a><p>[3]: <a href="https:&#x2F;&#x2F;docs.unity3d.com&#x2F;Manual&#x2F;LinearRendering-LinearOrGammaWorkflow.html" rel="nofollow">https:&#x2F;&#x2F;docs.unity3d.com&#x2F;Manual&#x2F;LinearRendering-LinearOrGamm...</a>')
