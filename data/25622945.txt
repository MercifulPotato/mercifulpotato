Item(by='rualca', descendants=None, kids=None, score=None, time=1609693091, title=None, item_type='comment', url=None, parent=25622099, text='&gt; \nPutting the binary in entrypoint instead of CMD seems like a misuse of what CMD was intended for, no?<p>No. From Docker&#x27;s official reference:<p>&gt; <i></i> The main purpose of a CMD is to provide defaults for an executing container. These defaults can include an executable, or they can omit the executable, in which case you must specify an ENTRYPOINT instruction as well.<i></i><p><a href="https:&#x2F;&#x2F;docs.docker.com&#x2F;engine&#x2F;reference&#x2F;builder&#x2F;#cmd" rel="nofollow">https:&#x2F;&#x2F;docs.docker.com&#x2F;engine&#x2F;reference&#x2F;builder&#x2F;#cmd</a><p>ENTRYPOINT should point to the entrypoint, and CMD should store default command line arguments.  This allows a container image to be executed as a command line application.')
