Item(by='veidr', descendants=None, kids=None, score=None, time=1607705392, title=None, item_type='comment', url=None, parent=25387534, text='Yeah, &quot;no DOM in tests&quot; is actually a pretty good mantra.<p>Code that is just inspecting the DOM to find elements, or to do stuff to them that is just working up to the state where you can actually test something, should be in some shared object&#x2F;library, but that is as easy as:<p><pre><code>    import { Whatever } from &#x27;&#x2F;foo&#x2F;bar&#x27;;\n</code></pre>\nHaving that kind of code outside of the test probably makes the introductory &quot;test hello world&quot; example more complicated, but I think it&#x27;s definitely a good practice in the real world, and isn&#x27;t documented well.')
