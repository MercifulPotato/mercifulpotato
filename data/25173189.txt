Item(by='SahAssar', descendants=None, kids=None, score=None, time=1605997284, title=None, item_type='comment', url=None, parent=25171275, text='&gt; What about PostgREST, the biggest issue I have with it is a DB server being available publicly in the net, I usually try my best to either place DB servers in the private network or &quot;hide&quot; them.<p>I&#x27;ve heard this argument many times (and thought it myself), but when dealing with postgrest it seems that if you have a proper JWT setup (which is how postgrest handles AuthN) and use postgres&#x27; security features (like row level security) perhaps it should not be thought as a rule anymore.<p>IMO it seems like having the api layer only assume a role and having the DB handle AuthZ would mean better security since you can implement more fine grained rules that are actually verified by the part of the stack that knows the data structure already.<p>It&#x27;s also not allowing arbitrary SQL, it&#x27;s translating from HTTP to SQL, so nobody can do &quot;SET ROLE &#x27;admin&#x27;;&quot; unless you write a specific SQL function that does that.')
