Item(by='kiwicopple', descendants=None, kids=None, score=None, time=1611414690, title=None, item_type='comment', url=None, parent=25882742, text='Hey HN, this is <a href="https:&#x2F;&#x2F;repository.surf" rel="nofollow">https:&#x2F;&#x2F;repository.surf</a>, an open source[1] site for viewing insights across all repo&#x27;s in a GitHub org.<p>The total star count will work for any github organization (it may take a while to fetch the data for big orgs).<p>If you log in and provide a GitHub key, repository.surf can track the total number of issues every hour, to give a burndown chart.<p>At this stage it&#x27;s just stars and total issue count, but we will probably add more graphs&#x2F;metrics some stage.<p>This came out of a need to see the total number of open issues across our entire org. \nWe wanted to track our burndown progress (or lack of progress in our case), which you can&#x27;t do from the github API.<p>You can see an example with issue tracking here: <a href="https:&#x2F;&#x2F;repository.surf&#x2F;supabase" rel="nofollow">https:&#x2F;&#x2F;repository.surf&#x2F;supabase</a><p>The site uses the GitHub API to fetch all the repos for an org, then fetch the star count for every repo. \nThe data is cached in a Postgres server in Supabase[2] so that it doesn&#x27;t hit the GitHub api for old data. \nOn new visit, the site will only pull current data.<p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;supabase&#x2F;repository.surf" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;supabase&#x2F;repository.surf</a><p>[2] <a href="https:&#x2F;&#x2F;supabase.io" rel="nofollow">https:&#x2F;&#x2F;supabase.io</a>')
