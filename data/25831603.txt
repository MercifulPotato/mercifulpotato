Item(by='insin', descendants=None, kids=None, score=None, time=1611049021, title=None, item_type='comment', url=None, parent=25826055, text='If you&#x27;re using VS Code or WebStorm&#x2F;IntelliJ (and doubtless any other editor which has its own internal copy of TypeScript and uses its language service) you can write JavaScript and still get most of the benefits of using TypeScript without buying into the tooling - it&#x27;s my preferred way to make use of it.<p>e.g. for VS Code, stick a jsconfig.json in your project&#x27;s root:<p><pre><code>    {\n      &quot;compilerOptions&quot;: {\n        &quot;checkJs&quot;: true,\n        &quot;target&quot;: &quot;es2020&quot;\n      }\n    }\n</code></pre>\nIf you want TypeScript to know about the types in your project, or help it with stuff it can&#x27;t figure out on its own you can add them using JSDoc comments:<p><a href="https:&#x2F;&#x2F;www.typescriptlang.org&#x2F;docs&#x2F;handbook&#x2F;jsdoc-supported-types.html" rel="nofollow">https:&#x2F;&#x2F;www.typescriptlang.org&#x2F;docs&#x2F;handbook&#x2F;jsdoc-supported...</a><p>The best thing is if TypeScript is being silly about your working code (its type definitions for the DOM can be particularly painful to work with), you can just ignore it :)')
