Item(by='ledauphin', descendants=None, kids=None, score=None, time=1606180725, title=None, item_type='comment', url=None, parent=25183694, text='I think this (allowing some form of default for backwards compatibility) is a underrated point.<p>The common modern example being async functions, I would say it&#x27;s actually quite surprising that JavaScript can&#x27;t seem to provide the sane default behavior of &quot;block this non-async function until the async function it calls returns a value&quot;.<p>The reason it can&#x27;t is simply that it chooses not to provide a runtime with a true threading model - blocking a non-async function means literally blocking the entire runtime. But there&#x27;s a possible world in which a &quot;colored&quot; (async) function could be used directly in a synchronous function using an escape hatch provided by the runtime, and then async functions would no longer be colored in the same way.')
