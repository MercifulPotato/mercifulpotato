Item(by='kbenson', descendants=None, kids=[24875944], score=None, time=1603492881, title=None, item_type='comment', url=None, parent=24874990, text='I agree.  Generally I find the approach to have a few tools that work well in their specific niches to be superior to having one tool for all jobs.<p>I use a shell for shell stuff, I use a scripting language for most other stuff, and I might use a systems level compiled language (or a scripting language that calls into a compiled library) for more performance specific needs.  If you&#x27;re already within a specific area and only need to venture into the other for a minimal aspect of the current project, it can be useful to stick with what you&#x27;re in, but you quickly reach the point where it&#x27;s better to choose a better tool because of diminishing returns from using a tool for something it&#x27;s not good for.<p>Maybe the hammer in the hand is fine for prying the single board off fence or wall, but if you&#x27;re going to be doing it to ten or twenty boards, maybe walking over to the shed to get the crowbar will save a lot of time and effort in the end.')
