Item(by='helgie', descendants=None, kids=[25060353], score=None, time=1605111561, title=None, item_type='comment', url=None, parent=25059324, text='If you use numba (or cython, c extensions, etc) you can make them run without requiring that they hold the GIL, and they can run in parallel. Here&#x27;s an example that should keep a CPU pegged at 100% utilization for a while:<p><pre><code>  import numba as nb\n  from concurrent.futures import ThreadPoolExecutor\n  from multiprocessing import cpu_count\n\n  @nb.jit(nogil=True)\n  def slow_calculation(x):\n      out = 0\n      for i in range(x):\n          out += i**0.01\n      return out\n\n  ex = ThreadPoolExecutor(max_workers=cpu_count())\n  futures = [ex.submit(slow_calculation, 100_000_000_000+i) for i in range(cpu_count())]</code></pre>')
