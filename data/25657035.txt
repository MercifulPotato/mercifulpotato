Your point is still definitely valid for the real world consequences of this, but i think that theoretically things could be way more simpler.<p>1.) Click &quot;download&quot; to get a piece of software you need (a static executable)<p>2.) Put said piece of software in a folder called &quot;&#x2F;apps&#x2F;$APP_NAME&quot;<p>3.) Run said software and have it persist all of its data to &quot;&#x2F;apps&#x2F;$APP_NAME&#x2F;data&quot; or some other convention like that.<p>Perhaps that&#x27;d need to be enforced with sandboxing and limiting access to directories, which would then make sharing data across them more difficult, which would end up with something like &quot;&#x2F;data&#x2F;$APP_NAME&quot;, which brings us back to the horror of Windows&#x27; &quot;AppData&quot; and &quot;Documents&quot; and &quot;Program Data&quot;, or other folders like that, resulting in what can only be described as an Eldritch nightmare, in which you don&#x27;t have any idea in which of the 20-30 different folders which particular stores its data...<p>But theorethically things could be simpler, i&#x27;ve heard people mention how easy installing apps used to be on Apple devices back in the day. And recently, after running the Godot game engine ( <a href="https:&#x2F;&#x2F;godotengine.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;godotengine.org&#x2F;</a> ), which is literally a single executable, i&#x27;m inclined to agree somewhat.')
