The original WebSocket handshake is done over HTTP&#x2F;1.1, there is an RFC specifying how to perform the negotiation also over HTTP&#x2F;2 connection but its support lacking.<p>On top of that, HTTP&#x2F;2 itself can do full-duplex communication so it can be used as a replacement of WebSocket, no negotiation or protocol upgrade involved. This capability is rarely used though but is, for example, leveraged by gRPC which is able to do bidirectional streaming [1].<p>[1] <a href="https:&#x2F;&#x2F;grpc.io&#x2F;docs&#x2F;what-is-grpc&#x2F;core-concepts&#x2F;" rel="nofollow">https:&#x2F;&#x2F;grpc.io&#x2F;docs&#x2F;what-is-grpc&#x2F;core-concepts&#x2F;</a>')
