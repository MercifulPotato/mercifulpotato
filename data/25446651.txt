Item(by='alexhutcheson', descendants=None, kids=[25450157], score=None, time=1608144598, title=None, item_type='comment', url=None, parent=25443234, text='Octave is ok for that, but not as good as Matlab. All the Matlab plotting functions work, but the plots don&#x27;t look as nice, and require more tweaking to get them to &quot;publication quality&quot;. Octave includes a Qt-based GUI, but it is not nearly as polished as the Matlab GUI.<p>Python is also just ok. You can use matplotlib[1] to create plots using an interface explicitly modeled on the Matlab interface, or use higher-level wrappers like Seaborn[2] which reduce the amount of work required to create nice-looking plots of certain types. &quot;Zoom to different features&quot; can be done via Pandas[3], or directly via NumPy, although the syntax can get a little cumbersome. The best Matlab-like GUI is probably &quot;Scientific mode&quot; in PyCharm[4], which requires a paid license. Jupyter notebooks[5] are a free option, but don&#x27;t include all the features you&#x27;d have in a Matlab-style GUI.<p>R is, in my opinion, best-in-class for exploratory data analysis. The combination of ggplot2[6] and dplyr[7] makes plotting, drill-down, aggregation, etc. easier than any other environment I&#x27;ve used. Chapters 2 and 4 of the free &quot;R for Data Science&quot; book[8] provide great examples of this. The RStudio[9] GUI is comparable to Matlab in features and overall polish, and it&#x27;s available with either an AGPLv3 or commercial license. Jupyter notebooks also work.<p>The major downside of R is that it&#x27;s very optimized for interactive use, which makes deploying it to any sort of production environment tricky and error-prone. I&#x27;d mostly advise doing your ad hoc analysis in R, but rewriting things in another language if you want to deploy them. The other downside is that all the major deep learning libraries use Python, so if you want to do work in that space you should really just use Python.<p>[1] <a href="https:&#x2F;&#x2F;matplotlib.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;matplotlib.org&#x2F;</a><p>[2] <a href="https:&#x2F;&#x2F;seaborn.pydata.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;seaborn.pydata.org&#x2F;</a><p>[3] <a href="https:&#x2F;&#x2F;pandas.pydata.org&#x2F;docs&#x2F;user_guide&#x2F;indexing.html" rel="nofollow">https:&#x2F;&#x2F;pandas.pydata.org&#x2F;docs&#x2F;user_guide&#x2F;indexing.html</a><p>[4] <a href="https:&#x2F;&#x2F;www.jetbrains.com&#x2F;help&#x2F;pycharm&#x2F;matplotlib-support.html#scientific-project" rel="nofollow">https:&#x2F;&#x2F;www.jetbrains.com&#x2F;help&#x2F;pycharm&#x2F;matplotlib-support.ht...</a><p>[5] <a href="https:&#x2F;&#x2F;jupyter-notebook.readthedocs.io&#x2F;en&#x2F;stable&#x2F;notebook.html" rel="nofollow">https:&#x2F;&#x2F;jupyter-notebook.readthedocs.io&#x2F;en&#x2F;stable&#x2F;notebook.h...</a><p>[6] <a href="https:&#x2F;&#x2F;ggplot2.tidyverse.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;ggplot2.tidyverse.org&#x2F;</a><p>[7] <a href="https:&#x2F;&#x2F;dplyr.tidyverse.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;dplyr.tidyverse.org&#x2F;</a><p>[8] <a href="https:&#x2F;&#x2F;r4ds.had.co.nz&#x2F;" rel="nofollow">https:&#x2F;&#x2F;r4ds.had.co.nz&#x2F;</a><p>[9] <a href="https:&#x2F;&#x2F;rstudio.com&#x2F;products&#x2F;rstudio&#x2F;" rel="nofollow">https:&#x2F;&#x2F;rstudio.com&#x2F;products&#x2F;rstudio&#x2F;</a>')
