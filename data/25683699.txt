Item(by='yakubin', descendants=None, kids=[25684937, 25686278], score=None, time=1610106613, title=None, item_type='comment', url=None, parent=25682766, text='Then you changed the semantics of the code. Now a program that e.g. prints a message in its destructor, which gives this output:<p><pre><code>  Message 1\n  Message 2\n  Message 3\n</code></pre>\nWill give this output:<p><pre><code>  Message 3\n  Message 2\n  Message 1\n</code></pre>\nYour odd&#x2F;even fix also won&#x27;t work, because a function may push an argument to a list, and pass the list in another argument, making the original temporary accessible from any recursion level.<p>A call to a destructor is a normal call. If you need to call destructors, then it means your tail call is a call to a destructor and not what you&#x27;re seeing in the code. It&#x27;s a fundamental semantical problem. You can&#x27;t have &quot;make the last call in this function be an implicit call to some function&quot; and &quot;make the last call be the explicit call to this other function&quot; at the same time.')
