Whatever address word you put into the 6502 reset vector ($FFFC&#x2F;$FFFD) it will happily jump to on power up or reset and start chugging right along. The PPU (aka GPU) takes roughly 2 frames (33ms) to &quot;warm up&quot; and receive information, so, you want to wait for that by polling PPU_STATUS ($2002) with BIT and waiting for BVS twice when NMI fires. It is a good idea to disable decimal mode, disable audio IRQs, set the stack pointer, and zero the memory in a loop. Any booting and loading speed is entirely code dependent.\n<a href="https:&#x2F;&#x2F;wiki.nesdev.com&#x2F;w&#x2F;index.php&#x2F;Init_code" rel="nofollow">https:&#x2F;&#x2F;wiki.nesdev.com&#x2F;w&#x2F;index.php&#x2F;Init_code</a>')
