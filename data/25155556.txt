Item(by='leetcrew', descendants=None, kids=None, score=None, time=1605828043, title=None, item_type='comment', url=None, parent=25155323, text='&gt; Technology can&#x27;t simultaneously progress while still remaining backwards-compatible for years or decades.<p>microsoft and x86 have entered the chat...<p>in all seriousness, there is a legitimate tradeoff to be made here. the accumulated legacy cruft can make it quite annoying to write code targeting windows. but once you get it done, you can be confident that it will work for a long time. on macOS, the APIs tend to be a lot cleaner, but you pay for that with features that are routinely deprecated, triggering major rewrites. as a developer, I do prefer microsoft&#x27;s stance. I don&#x27;t use macOS enough to have a real opinion as a user, but it is nice that I can run pretty much any windows software from the last twenty years without spinning up a VM or booting an older machine.')
