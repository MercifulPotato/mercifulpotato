Item(by='pmoriarty', descendants=None, kids=[24898313], score=None, time=1603723196, title=None, item_type='comment', url=None, parent=24894774, text='<i>&quot;how many lines of code do I have to read in order to grasp what is going on and what can I do to solve my problem?&quot;</i><p>I&#x27;m not sure that lines of code alone are that useful here, or APL would be everyone&#x27;s idea of a perfect language, since it can express so much in so few lines.<p>The Lisp community in particular seems to value verbosity over terseness, preferring long, descriptive function names and variable names, which make for more lines of code, but arguably greater readability.<p>I personally value clear code far higher than terse or clever code.  I&#x27;d much rather read over a page of easily understandable code in 5 minutes than puzzle over a single line that does the same exact thing for an hour.')
