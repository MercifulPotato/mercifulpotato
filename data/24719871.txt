Item(by='scudd', descendants=None, kids=None, score=None, time=1602170224, title=None, item_type='comment', url=None, parent=24719269, text='In my experience, I did eventually start to appreciate the compiler forcing me to address things up-front.<p>But as an aside, if you&#x27;re just trying to prototype quickly, some useful &quot;shortcuts&quot; can be:<p>- Using `Box&lt;dyn std::error::Error&gt;` for functions which need to handle a result<p>- Using `unwrap` or `expect` to easily access Option internal values<p>- Using `unimplemented!()` if I&#x27;m trying to mock an API design. This one is nice because it will let you build out the signature including specifying a return type without having to build out the function logic.<p>Obviously these are shortcuts and not production suitable, but can be good to just get some code on the page that compiles')
