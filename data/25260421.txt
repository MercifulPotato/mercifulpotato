Item(by='westurner', descendants=None, kids=[25260802], score=None, time=1606779063, title=None, item_type='comment', url=None, parent=25258654, text='Pip supports constraints files. <a href="https:&#x2F;&#x2F;pip.pypa.io&#x2F;en&#x2F;stable&#x2F;user_guide&#x2F;#constraints-files" rel="nofollow">https:&#x2F;&#x2F;pip.pypa.io&#x2F;en&#x2F;stable&#x2F;user_guide&#x2F;#constraints-files</a> :<p>&gt; <i>Constraints files are requirements files that only control which version of a requirement is installed, not whether it is installed or not. Their syntax and contents is nearly identical to Requirements Files. There is one key difference: Including a package in a constraints file does not trigger installation of the package.</i><p>&gt; <i>Use a constraints file like so:</i><p><pre><code>  python -m pip install -c constraints.txt</code></pre>')
