Item(by='SomeCallMeTim', descendants=None, kids=[25993984], score=None, time=1612207539, title=None, item_type='comment', url=None, parent=25992166, text='PostgreSQL directly supports table sharding. <a href="https:&#x2F;&#x2F;pgdash.io&#x2F;blog&#x2F;postgres-11-sharding.html" rel="nofollow">https:&#x2F;&#x2F;pgdash.io&#x2F;blog&#x2F;postgres-11-sharding.html</a><p>Schema evolution is supported by pretty much every ORM you&#x27;d care to use; it&#x27;s not the job of the SQL database to handle the migration. I&#x27;m using Prisma and you literally change the software spec for the schema and say &quot;migrate&quot;, and it creates the migration SQL and applies it to the Postgres DB programmatically. That gets you <i>deterministic</i> schema evolution and not the &quot;my schema isn&#x27;t actually reliable&quot; that NoSQL&#x2F;no-schema databases rely on.<p>And then you have CockroachDB&#x2F;YugabyteDB that give you extreme horizontal scalability, with full PostgreSQL compatibility...<p>And bang, the last reason to use MongoDB vanishes.')
