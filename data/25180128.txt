Item(by='spijdar', descendants=None, kids=None, score=None, time=1606072467, title=None, item_type='comment', url=None, parent=25179896, text='ARM does, in fact, use standard buses, PCIe on most high&#x2F;medium end parts -- although, some low end chips just use a built-in USB controller or SPI&#x2F;I2C for all peripherals.<p>What isn&#x27;t always available is a way to dynamically enumerate a device&#x27;s, uh, device tree. On PCs, this is usually accomplished through ACPI tables, where the bootloader (BIOS or UEFI) populates a table of data and callbacks to AML bytecode functions which the OS kernel can read and eval. [0]<p>So on x86, the OS finds the PCIe memory mapped addressed in the MCFG table.<p>On most ARM devices, this data has to be passed to the kernel at runtime, in what Linux calls a &quot;device tree reference&quot;, compiled to a &quot;DTB&quot; file.<p>So, on a UEFI-based ARM system, it should be possible to just insert, say, an AMD graphics card onto the PCIe bus and get a working GPU.<p>[0] <a href="https:&#x2F;&#x2F;www.kernel.org&#x2F;doc&#x2F;html&#x2F;latest&#x2F;arm64&#x2F;acpi_object_usage.html" rel="nofollow">https:&#x2F;&#x2F;www.kernel.org&#x2F;doc&#x2F;html&#x2F;latest&#x2F;arm64&#x2F;acpi_object_usa...</a>')
