Item(by='ad404b8a372f2b9', descendants=None, kids=None, score=None, time=1611323474, title=None, item_type='comment', url=None, parent=25871360, text='Yes, the read-write disk partition also holds my files.<p>No docs I&#x27;m afraid and I set it up too long ago to remember the exact details. I used overlayroot, there are some really good resources on google to set it up like this. If I remember correctly it&#x27;s just a matter of setting the overlayroot.conf file to:<p><pre><code>  overlayroot_cfgdisk=&quot;disabled&quot;  \n  overlayroot=&quot;tmpfs:swap=1,recurse=0&quot;  \n</code></pre>\nAnd then a grub option to mount the base in read-only:<p><pre><code>  linux &#x2F;boot&#x2F;vmlinuz-5.3.0-51-generic root=UUID=... ro  $vt_handoff\n</code></pre>\nThen you add your mounts in fstab for persistent stuff.<p>I think this blog post describes it well: <a href="https:&#x2F;&#x2F;spin.atomicobject.com&#x2F;2015&#x2F;03&#x2F;10&#x2F;protecting-ubuntu-root-filesystem&#x2F;" rel="nofollow">https:&#x2F;&#x2F;spin.atomicobject.com&#x2F;2015&#x2F;03&#x2F;10&#x2F;protecting-ubuntu-r...</a><p>For modifications to the base, installing or modifying software, etc I have a grub option to disable the overlay system and mount the base partition in read-write so it can be used normally. So I reboot into this option, do my changes, then reboot immediately in overlay mode.<p><pre><code>  linux &#x2F;boot&#x2F;vmlinuz-5.3.0-51-generic root=UUID=... rw overlayroot=disabled  $vt_handoff  \n\n</code></pre>\nIt took me about a month to get used to it, sometimes I&#x27;d apt-get something then the next day I&#x27;d facepalm after realizing I had done it in overlay-mode and had to do it all over again. I haven&#x27;t lost any personal files though, it&#x27;s pretty easy to remember to avoid saving them to your home and instead go to the persistent partition.')
