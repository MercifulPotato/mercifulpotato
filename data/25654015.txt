Item(by='ben509', descendants=None, kids=[25654313, 25655470, 25654493], score=None, time=1609899377, title=None, item_type='comment', url=None, parent=25650298, text='&gt; I don&#x27;t think it&#x27;s Python&#x27;s fault.<p>Well, there&#x27;s a long legacy of half-baked solutions and kludges. And, especially, install instructions consistently suggest you do the worst thing: install stuff using pip.<p><i>Everyone</i> knows that&#x27;s bad advice, because it always fails in mysterious ways, and there&#x27;s no good way to roll everything back.<p>I recommend:<p><pre><code>    python3 -m pip install pipx  # Don&#x27;t rely on the pip3 bin.\n    pipx install jupyterlab\n    # Now jupyter has an independent virtualenv.\n    # To install dependencies:\n    pipx inject jupyterlab numpy\n    # Where is everything?\n    pipx list\n    venvs are in &#x2F;Users&#x2F;ben&#x2F;.local&#x2F;pipx&#x2F;venvs\n    apps are exposed on your $PATH at &#x2F;Users&#x2F;ben&#x2F;.local&#x2F;bin\n    ...\n</code></pre>\nOnce you&#x27;ve done that, <i>never</i> install anything directly with pip3 again. If you&#x27;re working on a project, use pipx to install poetry &#x2F; dephell &#x2F; pipenv and use them to manage the virtualenv.<p>And you can get rid of it all:<p><pre><code>    pipx uninstall-all\n    python3 -m pip uninstall pipx</code></pre>')
