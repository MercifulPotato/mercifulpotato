However, I wonder if there&#x27;s space for a command line git wrapper makes it much harder to shoot yourselves in the foot via scenarios documented at: <a href="https:&#x2F;&#x2F;ohshitgit.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;ohshitgit.com&#x2F;</a><p>I imagine this could be achieved by using more intuitive nouns for subcommands (reflog, rebase) and giving informative warning messages on the command line when you are about to screw something up irrevocably.<p>Eventually, one would hope that some of the more effective features of such a wrapper can find their way back to the git project itself.')
