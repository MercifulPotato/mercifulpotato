Item(by='narrationbox', descendants=None, kids=None, score=None, time=1603750344, title=None, item_type='comment', url=None, parent=24899409, text='You make a bootup disc (or USB) like your typical dual booting procedure. Then go into BIOS (I am using the legacy name for the modern UEFI menu) and boot up from there. It is the same as any OS development except that UEFI provides you with a number of APIs including a mini filesystem, a primitive shell, a graphics API, and some other stuff. To make it easier, you can think of your computer running pure UEFI apps as a giant, overpowered Arduino.<p><a href="https:&#x2F;&#x2F;wiki.osdev.org&#x2F;UEFI_Bare_Bones" rel="nofollow">https:&#x2F;&#x2F;wiki.osdev.org&#x2F;UEFI_Bare_Bones</a><p>Until recently, almost every UEFI app had to be written in C&#x2F;C++.')
