Item(by='DonHopkins', descendants=None, kids=None, score=None, time=1608069139, title=None, item_type='comment', url=None, parent=25435201, text='James Clark&#x27;s compact syntax for Relax&#x2F;NG XML schema validation language is quite tastefully designed, an equivalent but more convenient alternative syntax than XML, for writing tree regular expressions matching XML documents. It&#x27;s way more beautiful and coherent than the official &quot;XML Schema&quot; standard.<p><a href="https:&#x2F;&#x2F;www.oasis-open.org&#x2F;committees&#x2F;relax-ng&#x2F;compact-20021121.html" rel="nofollow">https:&#x2F;&#x2F;www.oasis-open.org&#x2F;committees&#x2F;relax-ng&#x2F;compact-20021...</a><p>RELAX NG compact syntax cheat sheet<p><a href="https:&#x2F;&#x2F;gist.github.com&#x2F;miikka&#x2F;b137a8705e88b0b6c164" rel="nofollow">https:&#x2F;&#x2F;gist.github.com&#x2F;miikka&#x2F;b137a8705e88b0b6c164</a><p><a href="https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;RELAX_NG#Compact_syntax" rel="nofollow">https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;RELAX_NG#Compact_syntax</a><p>&gt;RELAX NG compact syntax is a non-XML format inspired by extended Backus-Naur form and regular expressions, designed so that it can be unambiguously translated to its XML counterpart, and back again, with one-to-one correspondence in structure and meaning, in much the same way that Simple Outline XML (SOX) relates to XML. It shares many features with the syntax of DTDs. Here is the compact form of the above schema:<p><pre><code>    element book {\n        element page { text }+\n    }\n</code></pre>\n&gt;With named patterns, this can be flattened to:<p><pre><code>    start = element book { page+ }\n    page = element page { text }\n</code></pre>\n&gt;A compact RELAX NG parser will treat these two as the same pattern.<p>There&#x27;s a wonderful DDJ interview with James Clark called &quot;A Triumph of Simplicity: James Clark on Markup Languages and XML&quot; where he explains how a standard has failed if everyone just uses the reference implementation, because the point of a standard is to be crisp and simple enough that many different implementations can interoperate perfectly.<p>A Triumph of Simplicity: James Clark on Markup Languages and XML:<p><a href="https:&#x2F;&#x2F;web.archive.org&#x2F;web&#x2F;20130721072712&#x2F;https:&#x2F;&#x2F;www.drdobbs.com&#x2F;a-triumph-of-simplicity-james-clark-on-m&#x2F;184404686" rel="nofollow">https:&#x2F;&#x2F;web.archive.org&#x2F;web&#x2F;20130721072712&#x2F;https:&#x2F;&#x2F;www.drdob...</a><p>&quot;The standard has to be sufficiently simple that it makes sense to have multiple implementations.&quot; -James Clark')
