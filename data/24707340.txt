Item(by='Liskni_si', descendants=None, kids=None, score=None, time=1602072151, title=None, item_type='comment', url=None, parent=24707258, text='There are a million ways to do it, but dpkg-buildpackage has been here for ages and if all you need is rebuilding a package for your personal use and for your current system and architecture and don&#x27;t mind installing builddeps into your system, it&#x27;s definitely the easiest.<p>The other options are good if you need to ensure the package will build elsewhere, or if you need to cross build for a different architecture (which you might need if you&#x27;re running a multiarch system and you need to patch a library like Mesa that&#x27;s still needed by some 32 bit apps), or if you don&#x27;t want to clog your system with build deps or something like that.<p>Last time I needed to do that, I used cowbuilder, but I&#x27;m not really an expert in this area, I&#x27;m just a Debian power user, not involved with the project directly.')
