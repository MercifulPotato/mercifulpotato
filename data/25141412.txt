Item(by='rng_civ', descendants=None, kids=None, score=None, time=1605728419, title=None, item_type='comment', url=None, parent=25141238, text='To add to this comment, Stopify is a JS-to-JS compiler that instruments sync JS code to make them interruptible at set points. The paper [0] can explain it better than I ever could.<p>I work on an experimental Pyret [1] runtime that uses Stopify to instrument compiled Pyret code (plain old JS) so that we can run Pyret code on the main page thread without hanging it up. Main thread execution is important for quick&#x2F;easy DOM access. In terms of performance cost, we haven&#x27;t measured too extensively, but so far, on average, we&#x27;re seeing a 2x slow down compared to un-Stopified programs.<p>(Disclaimer: paid contributor for Pyret).<p>[0] <a href="https:&#x2F;&#x2F;www.stopify.org&#x2F;research.html" rel="nofollow">https:&#x2F;&#x2F;www.stopify.org&#x2F;research.html</a><p>[1] <a href="https:&#x2F;&#x2F;www.pyret.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.pyret.org&#x2F;</a>')
