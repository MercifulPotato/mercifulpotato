Item(by='jmnicolas', descendants=None, kids=None, score=None, time=1608211267, title=None, item_type='comment', url=None, parent=25447416, text='I experimented with my own OSM tile server a few months ago.<p>I used this tutorial <a href="https:&#x2F;&#x2F;www.linuxbabe.com&#x2F;ubuntu&#x2F;openstreetmap-tile-server-ubuntu-20-04-osm" rel="nofollow">https:&#x2F;&#x2F;www.linuxbabe.com&#x2F;ubuntu&#x2F;openstreetmap-tile-server-u...</a><p>If like me you have basic Linux skills it&#x27;s not that hard, just time consuming.<p>However the resources needed are quite consequent: I managed to load all Europe tiles on a Core i7 3770k &#x2F; 32 GB ram &#x2F; 1 TB SSD workstation.<p>I think it took 2+ days to load data into Postgres and many more days to render tiles zoom 0 to 18 for France. I estimate that to run a world tile server you would need as many core as possible, at least 64 GB of ram and 2 TB of disk (SSD is mandatory here).<p>I didn&#x27;t experiment with updating the DB with new OSM data but it looks quite a chore.')
