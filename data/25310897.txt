Item(by='spc476', descendants=None, kids=[25311050], score=None, time=1607133443, title=None, item_type='comment', url=None, parent=25288535, text='Starting back around 2009, I started marking bug fixes in git with the first line containing &quot;Bug fix---description&quot;.  For example (from my gopher server [1]):<p><pre><code>    a12c3fd Bug fix---sample forgot POST method.\n    0b1bce2 Bug fix---grab guid when only given numeric uid.\n    31408da Bug tracking---crash because filename is nil?\n    d993265 Bug fix---Clean the display text to prevent broken index files.\n    47e98e8 Bug fix---line comments were broken in .port70 files.\n</code></pre>\nI do the same at work.  I got the idea from Donald Knuth, who wrote a paper about classifying each error in TeX [2].  I didn&#x27;t go as far as he did, but I still have his list handy:<p>* Algorithm\n* Blunder\n* Cleanup\n* Data\n* Efficiency\n* Forgotten\n* Generalization\n* Interaction\n* Language\n* Mismatch\n* Portability\n* Quality\n* Robustness\n* Surprise\n* Typo<p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;spc476&#x2F;port70" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;spc476&#x2F;port70</a><p>[2] &quot;The Errors of TeX&quot; &lt;<a href="https:&#x2F;&#x2F;yurichev.com&#x2F;mirrors&#x2F;knuth1989.pdf" rel="nofollow">https:&#x2F;&#x2F;yurichev.com&#x2F;mirrors&#x2F;knuth1989.pdf</a>&gt;')
