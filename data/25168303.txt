Item(by='chrismorgan', descendants=None, kids=[25168464, 25168611, 25168328, 25168372, 25168504, 25169150, 25170208, 25169112], score=None, time=1605942737, title=None, item_type='comment', url=None, parent=25167928, text='The trouble with “<i>only</i> semantic HTML” (read: “HTML with no class attributes ever”) is when you try to represent things for which there <i>are</i> no semantic elements defined. I then observe that <i>most</i> projects like this then end up abusing other elements. In this case, the use of &lt;aside&gt; is unequivocally wrong, but <a href="https:&#x2F;&#x2F;github.com&#x2F;andybrewer&#x2F;mvp&#x2F;issues&#x2F;2" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;andybrewer&#x2F;mvp&#x2F;issues&#x2F;2</a> was closed in the end simply because there was no way to get what was desired without using class or similar. Other things that are also unequivocally semantically incorrect are the button links (a &gt; b, a &gt; i, a &gt; strong, a &gt; em), and raised text notification bubbles (sup).<p>The first rule of web accessibility is that doing nothing is better than doing the wrong thing. <a href="https:&#x2F;&#x2F;www.w3.org&#x2F;TR&#x2F;wai-aria-practices&#x2F;#no_aria_better_bad_aria" rel="nofollow">https:&#x2F;&#x2F;www.w3.org&#x2F;TR&#x2F;wai-aria-practices&#x2F;#no_aria_better_bad...</a> explains a bit.<p>There’s a reason why the class attribute exists. You can also validly use custom elements and attributes, which are approximately any name containing a -.')
