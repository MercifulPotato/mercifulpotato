Item(by='wffurr', descendants=None, kids=None, score=None, time=1603721841, title=None, item_type='comment', url=None, parent=24895328, text='Especially since WebGL doesn&#x27;t have mapped buffers[0].  There&#x27;s no way to do asynchronous texture (aka data) uploads.  At best, you can read back asynchronously but even that&#x27;s not guaranteed by the spec[1].  Async data transfer gives much higher throughput for sending data and retrieving results.<p>This is especially painful on mobile where GPU and CPU memory are the same physical RAM, and the &quot;map buffer&quot; operation corresponds to an actual instruction to the memory controller rather than synchronizing memory across PCIe lanes.<p>[0]: <a href="https:&#x2F;&#x2F;www.khronos.org&#x2F;registry&#x2F;webgl&#x2F;specs&#x2F;latest&#x2F;2.0&#x2F;#5.14" rel="nofollow">https:&#x2F;&#x2F;www.khronos.org&#x2F;registry&#x2F;webgl&#x2F;specs&#x2F;latest&#x2F;2.0&#x2F;#5.1...</a>\n[1]: <a href="https:&#x2F;&#x2F;www.khronos.org&#x2F;registry&#x2F;webgl&#x2F;specs&#x2F;latest&#x2F;2.0&#x2F;#3.73" rel="nofollow">https:&#x2F;&#x2F;www.khronos.org&#x2F;registry&#x2F;webgl&#x2F;specs&#x2F;latest&#x2F;2.0&#x2F;#3.7...</a> - Note the &quot;non-normative&quot; block describing the potential to bypass the specified blocking behavior for getBufferSubData.')
