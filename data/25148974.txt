I believe ZFS supports sending incremental snapshots without having the decryption key.<p>However, it become a pain-in-the-ass if you have recursive snapshots where only some of the datasets are encrypted. I think... in order to accomplish this, you need to send separate batches of snapshots... one for each set of encrypted vs. decrypted.<p>E.g.<p><pre><code>  &#x2F;zpool&#x2F;tmp          # not encrypted\n  &#x2F;zpool&#x2F;home&#x2F;someone # not encrypted\n  &#x2F;zpool&#x2F;home&#x2F;someone&#x2F;thunderbird # encrypted\n\n  $ zfs snapshot -r &#x2F;zpool zpool@today\n\n  # this will probably bork\n  $ zfs send -v -w -R -I zpool@yesterday zpool@today | ...\n\n  # but this will work\n  $ zfs send -v -R -I zpool&#x2F;tmp@yeserday zpool&#x2F;tmp@today | ...\n  $ zfs send -v -R -I zpool&#x2F;home&#x2F;someone@yeserday zpool&#x2F;home&#x2F;someone@today | ...\n  $ zfs send -v -w -R -I zpool&#x2F;home&#x2F;someone&#x2F;thunderbird@yeserday zpool&#x2F;home&#x2F;someone&#x2F;thunderbird@today | ...\n</code></pre>\nNOTE: I haven&#x27;t fully explored this. But from experience, loading the key on the remote solves a lot of problems. The most import feature of encryption for us is encryption-at-rest. I just want to pull the AC plug and ensure that the data is protected.')
