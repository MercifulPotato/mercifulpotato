Item(by='judge2020', descendants=None, kids=[24885681, 24884088], score=None, time=1603602765, title=None, item_type='comment', url=None, parent=24883944, text='I think it&#x27;s because GitHub wants to allow repo maintainers to merge in PRs without them having to add separate remotes themselves, ie `git remote add` isn&#x27;t required to `git merge`.<p>This basically means that any content can be injected into anyone&#x27;s GH repo (since PRs can&#x27;t be turned off), but really only in terms of being able to view it on the GitHub website. To give an example, pull 437 on torvalds&#x2F;linux[0] hasn&#x27;t been merged in, but if you go to the commit hash in the browser, suddenly main&#x2F;init.c has the relevant changes and commit that condense the file into one line[1].<p>This very well could be abused - imagine framing (or just &#x27;canceling&#x27;) someone with [insert illegal content here] by PRing their repo with a commit with a forged author[2] then linking people to their repo with the commit tree showing the illegal content.<p>0: <a href="https:&#x2F;&#x2F;github.com&#x2F;torvalds&#x2F;linux&#x2F;pull&#x2F;437" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;torvalds&#x2F;linux&#x2F;pull&#x2F;437</a><p>1: <a href="https:&#x2F;&#x2F;github.com&#x2F;torvalds&#x2F;linux&#x2F;blob&#x2F;2793ae1df012c7c3f13ea5c0f0adb99017999c3b&#x2F;init&#x2F;main.c" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;torvalds&#x2F;linux&#x2F;blob&#x2F;2793ae1df012c7c3f13ea...</a><p>2: <a href="https:&#x2F;&#x2F;stackoverflow.com&#x2F;a&#x2F;60900120&#x2F;3878893" rel="nofollow">https:&#x2F;&#x2F;stackoverflow.com&#x2F;a&#x2F;60900120&#x2F;3878893</a>')
