Item(by='progval', descendants=None, kids=None, score=None, time=1611530221, title=None, item_type='comment', url=None, parent=25894512, text='&gt; git dates are unsigned timestamps<p>You are thinking of the in-memory representation of the (main) implementation, not the git format itself.<p>The on-disk&#x2F;wire format is text representation, encoded in decimal. You can add a minus sign in front of it, git won&#x27;t crash when parsing it. The implementation will, however, display the wrong date on git log, but it&#x27;s a bug that git developers are working on.<p>For example (coincidentally, it is also a constitution):<p><pre><code>    $ git init\n    $ curl https:&#x2F;&#x2F;archive.softwareheritage.org&#x2F;api&#x2F;1&#x2F;vault&#x2F;revision&#x2F;5e8f80061263d818ae3fc6c8d2086b5278f271ea&#x2F;gitfast&#x2F;raw&#x2F; | gunzip | git fast-import &gt; &#x2F;dev&#x2F;null\n    $ git show 5e8f80061263d818ae3fc6c8d2086b5278f271e | grep Date\n    Date:   Thu Jan 1 00:00:00 1970 +0000\n    $ git cat-file -p 5e8f80061263d818ae3fc6c8d2086b5278f271e | grep author\n    author State Constitutional Conventions &lt;&gt; -5706313200 -0455</code></pre>')
