Item(by='simonw', descendants=None, kids=None, score=None, time=1610054154, title=None, item_type='comment', url=None, parent=25677554, text='This was inspired by <a href="https:&#x2F;&#x2F;css-tricks.com&#x2F;custom-properties-as-state&#x2F;" rel="nofollow">https:&#x2F;&#x2F;css-tricks.com&#x2F;custom-properties-as-state&#x2F;</a> which proposes stylesheets as an API - the stylesheet defines dynamic CSS custom properties (aka CSS variables) which can then be used by the rest of the page.<p>For example, adding the .css extension to a Datasette table page (having installed this plugin) gives you the following:<p><a href="https:&#x2F;&#x2F;latest-with-plugins.datasette.io&#x2F;fixtures&#x2F;roadside_attractions.css" rel="nofollow">https:&#x2F;&#x2F;latest-with-plugins.datasette.io&#x2F;fixtures&#x2F;roadside_a...</a><p><pre><code>    :root {\n      --pk: &#x27;1&#x27;;\n      --name: &#x27;The Mystery Spot&#x27;;\n      --address: &#x27;465 Mystery Spot Road, Santa Cruz, CA 95065&#x27;;\n      --latitude: &#x27;37.0167&#x27;;\n      --longitude: &#x27;-122.0024&#x27;;\n    }\n</code></pre>\nThat&#x27;s data you can display on your page like this:<p><pre><code>    &lt;link rel=&quot;stylesheet&quot; href=&quot;https:&#x2F;&#x2F;latest-with-plugins.datasette.io&#x2F;fixtures&#x2F;roadside_attractions.css&quot;&gt;\n    &lt;style&gt;\n    .attraction-name:after { content: var(--name); }\n    .attraction-address:after { content: var(--address); }\n    &lt;&#x2F;style&gt;\n    &lt;p class=&quot;attraction-name&quot;&gt;Attraction name: &lt;&#x2F;p&gt;\n    &lt;p class=&quot;attraction-address&quot;&gt;Address: &lt;&#x2F;p&gt;\n</code></pre>\nIs this a good idea? Almost certainly not, but it&#x27;s definitely a fun hack!')
