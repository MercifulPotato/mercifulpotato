Item(by='pmeunier', descendants=None, kids=None, score=None, time=1608285637, title=None, item_type='comment', url=None, parent=25458984, text='&gt; If I have a 100-line file and on &#x27;main&#x27; it changes near the top, but in my &#x27;topic&#x27; branch it changes near the bottom, then I can cherry-pick &#x27;topic&#x27; onto &#x27;main&#x27; and Git will resolve the diff correctly.<p>That is not true: sometimes Git will take the new lines from &quot;topic&quot; and merge them into the new lines from &quot;main&quot;, see <a href="https:&#x2F;&#x2F;pijul.org&#x2F;manual&#x2F;why_pijul.html" rel="nofollow">https:&#x2F;&#x2F;pijul.org&#x2F;manual&#x2F;why_pijul.html</a><p>&gt; You might hit a conflict in your &quot;git cherry-pick&quot; command which gives you an opportunity to resolve the unexpected diff issue in an appropriate way, which ends up with a different diff than before.<p>Sometimes when you cherry-pick, you might not even hit a &quot;true&quot; conflict, but if you forgot to run &quot;rerere&quot;, you might simply hit a previously solved conflict again.')
