Item(by='flohofwoe', descendants=None, kids=[25002446], score=None, time=1604587552, title=None, item_type='comment', url=None, parent=24998448, text='A similar problem exists in the C&#x2F;C++ world: designing a library in a way that supports good dead-code-elimination isn&#x27;t something that many library authors think about.<p>Interestingly plain C libraries often do this better than C++ libraries (e.g. too generous use of virtual-method-interfaces in C++ libraries, or making heavy use of C++ stdlib containers when simple C-style arrays would do as well).<p>The Unity engine also suffered from a similar problem: If you used a simple physics feature like a stabbing collision check somewhere in your code, the entire physics module was pulled in, even though the code needed for stabbing checks is only a few percent of that physics module.<p>This is why I&#x27;m a bit sceptical that improvements in the .NET &quot;machinery&quot; alone can solve this problem. You also need to restrict language features and select the right dependencies. This is a general problem, not just web-specific, it&#x27;s just that reducing the output binary size typically isn&#x27;t a high-priority problem outside the web.')
