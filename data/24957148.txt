While it might be a good idea to stop using null in your own code, you should probably still make it a habit to check for both null and undefined to avoid problems with external code.<p>&gt; we cannot remove null from JavaScript, but we can pretend it doesn&#x27;t exist<p>Even if it does become general convention to not use null, as a developer, you cannot depend on that convention and skip null checks in favor of undefined, because null is deeply engrained into the language itself.<p>So... we can&#x27;t really pretend that it doesn&#x27;t exist')
