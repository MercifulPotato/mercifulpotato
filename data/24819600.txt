Item(by='leonim', descendants=None, kids=None, score=None, time=1603042642, title=None, item_type='comment', url=None, parent=24818430, text='Totally agree.  I think mlr is a wonderful CLI tool.  It is very robust, and can handle&#x2F;convert multiple tabular formats including csv, tsv, json, fixed-format, etc. It has a pretty decent text output formatting, with the --opprint flag.<p>I use to be very comfortable using awk&#x2F;sed&#x2F;perl&#x2F;sort&#x2F;uniq&#x2F;tr&#x2F;tail&#x2F;head from the CLI for the sort of data cleaning this article is talking about.  However, over the past year I&#x27;ve found I use  VisiData <a href="https:&#x2F;&#x2F;github.com&#x2F;saulpw&#x2F;visidata" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;saulpw&#x2F;visidata</a> for interactive work.<p>If I need to clean up the data first, I&#x27;ll use mlr or jq as input to Visidata.  If my data is too dirty for mlr, then I&#x27;ll use Unix toolbox tools mentioned as input to mlr, jq or VisiData.<p>VisiData provides some ability to script, but when possible I prefer to have the shell do the scripting with all the tools mentioned as input to Visidata.')
