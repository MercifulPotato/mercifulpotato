But any worthy type system prevents lot of errors. This is not generally acclaimed by empirical studies claiming only 5% improvement because they are caught at compile time.<p>Other benefits are enough to use a sufficiently good static type system, by the way, even for people that think type systems aren&#x27;t better than TDD.<p><pre><code>    * Better autocomplete\n    * Better error highlighting - no need to run code or tests, IDE can highlight instantly.\n    * No typos in variable names (can be achieved in some but not all dynamic languages)\n    * Performance of compiled code</code></pre>')
