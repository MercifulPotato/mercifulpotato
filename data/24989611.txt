Item(by='HelloNurse', descendants=None, kids=None, score=None, time=1604502089, title=None, item_type='comment', url=None, parent=24988157, text='Two label that should have the same value are normally one too many, but if they are needed it is often possible to order them properly...<p><pre><code>    &#x2F;&#x2F;initialization\n    int counter=0;\n    &#x2F;&#x2F;each frame\n    label(&quot;count: %d\\n&quot;, counter);\n    label(&quot;count again, in case you missed the other label: %d\\n&quot;, counter);\n    if(button(&quot;increase&quot;)) { counter += 1; }\n</code></pre>\nor, if the order of widgets is constrained, to buffer values:<p><pre><code>    &#x2F;&#x2F;initialization\n    int counter_old=0;\n    int counter_new=0;\n    &#x2F;&#x2F;each frame\n    counter_old = counter_new;\n    label(&quot;count: %d\\n&quot;, counter_old);\n    if(button(&quot;increase&quot;)) { counter_new = counter_old+1; }\n    if(button(&quot;decrease&quot;)) { counter_new = counter_old-1; }\n    label(&quot;count again, in case you missed the other label: %d\\n&quot;, counter_old);</code></pre>')
