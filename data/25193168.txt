Do I have to use the Certera API for that?<p>Yes, and it&#x27;s very simple &amp; basic. A single CURL to get it like so: \ncurl <a href="https:&#x2F;&#x2F;&lt;your_certera&gt;&#x2F;api&#x2F;certificate&#x2F;&lt;cert_name&gt;" rel="nofollow">https:&#x2F;&#x2F;&lt;your_certera&gt;&#x2F;api&#x2F;certificate&#x2F;&lt;cert_name&gt;</a> \\\n  -H &quot;apiKey:&lt;your_api_key&gt;&quot;<p>You can pipe that out to a file directly as it&#x27;s in PEM format by default. More info here: <a href="https:&#x2F;&#x2F;docs.certera.io&#x2F;#certificates-api" rel="nofollow">https:&#x2F;&#x2F;docs.certera.io&#x2F;#certificates-api</a><p>The thing that&#x27;s unique about Certera is that it&#x27;s not opinionated on your existing setup. It doesn&#x27;t care whether it&#x27;s Traefik, apache, nginx or IIS. The &quot;glue&quot; is a standard PEM file format, the way it should be. It&#x27;s up to you how to tell whatever system cares about the PEM and do the &quot;reload&quot; of the cert.<p>I&#x27;m not sure how Traefik would communicate with it as I&#x27;m not familiar with Traefik in general. I&#x27;m assuming that you&#x27;d like Traefik to simply say: &quot;gimme the cert for xyz domain&quot; and have some endpoint&#x2F;system take care of the rest, right? Don&#x27;t hesitate to create an issue in GitHub and we can discuss further. Sometimes I lose track of HN comments due to a lack of notifications.')
