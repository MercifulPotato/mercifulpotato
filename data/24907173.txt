Item(by='tialaramex', descendants=None, kids=None, score=None, time=1603805232, title=None, item_type='comment', url=None, parent=24906988, text='Specifically SFTP is a SSH subsystem, when your client connects to a SSH server it gets to specify what subsystem it intends to talk to, and so a remote SSH server can choose to offer the &quot;traditional&quot; shell service, an SFTP server, or any arbitrary thing. In this way SSH subsystems are rather like ALPN in TLS (except years earlier) which is how HTTP&#x2F;2 works among other things.<p>The SFTP protocol itself, the thing spoken <i>over</i> SSH to the remote SFTP subsystem, is pretty simple, although I don&#x27;t think it was ever formally standardised, <a href="https:&#x2F;&#x2F;tools.ietf.org&#x2F;html&#x2F;draft-ietf-secsh-filexfer-13" rel="nofollow">https:&#x2F;&#x2F;tools.ietf.org&#x2F;html&#x2F;draft-ietf-secsh-filexfer-13</a><p>You could in principle talk SFTP over some transport other than SSH (e.g. you could use ALPN to select it over TLS), but nobody does.')
