Item(by='paranoidrobot', descendants=None, kids=[24776195], score=None, time=1602680670, title=None, item_type='comment', url=None, parent=24775784, text='The problem is that it makes testing every upstream component more difficult to write for, and more fragile.<p>To take the shopping cart example I gave, if I write it the way that I understand you&#x27;re suggesting, I would have to consider the behavior of all of the ICartDiscountStrategy implementations.<p>For instance, I&#x27;ve written tests for CartDiscountCalculator and ShoppingCart that have 10 different items in the cart from Acme Corp Manufacturing.<p>Some time later, I implement a new AcmeCorpDiscountStrategy which results in a different discount being returned. \nNow my tests for CartDiscountCalculator and ShoppingCart need to be modified.<p>I also need to potentially modify everything else up the dependency graph from CartDiscountCalculator - so my internal API that is used by customer service, an API used by mobiles, and the front end website.<p>Using Mocks&#x2F;Fakes to isolate the ShoppingCart, it doesn&#x27;t matter what the strategies are, we only care about how it interacts with ICartDiscountCalculator.<p>That interaction might change, maybe ShoppingCart uses the CartDiscountCalculator in a different way yeah, that&#x27;ll break ShoppingCart Unit Tests - but it&#x27;s that method only, and you&#x27;re already in there modifying ShoppingCart anyway.')
