Item(by='simiones', descendants=None, kids=[25059131], score=None, time=1605106398, title=None, item_type='comment', url=None, parent=25058601, text='No, REST puts the Resource at the forefront, and normally you use different verbs on the same resource: you PUT&#x2F;POST to create a new resource, you can then GET that resource to read it, you PUT&#x2F;PATCH to modify it, you DELETE a resource to delete it.<p>CQRS usually means that Commands (modifying the system) are handled differently than Queries (checking the current state of the system).<p>Of course, you can absolutely create a REST API where there is one set of resources that you can modify but not read (Command resources), and one set of resources you can read but not modify (Query resources). But that would definitely not fit the general idea of what people expect from a REST API.<p>Also, such an API would be relatively hard to make HATEOAS-compliant, for those that care about &quot;ultimate&quot;&#x2F;&quot;true&quot; REST.')
