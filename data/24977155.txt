Item(by='SPBS', descendants=None, kids=None, score=None, time=1604386917, title=None, item_type='comment', url=None, parent=24976397, text='I&#x27;ve edited 1GB sql dump files in Vim with no problem. I had to turn off my vimrc for it to work, but 300MB should be no problem as long as you are using barebones vim.<p>For performant buffer-wide operations (:g, :%s), I had success with using vim like an improved sed by passing in commands directly via the command line. That was noticeably faster than opening vim up and running the commands before exiting e.g.<p><pre><code>    # join together lines delimited by LF into one line, then convert all CRLF into LF\n    vim -Es \\\n        -c &#x27;g&#x2F;[^^M]$&#x2F;.,&#x2F;^M$&#x2F;join&#x27; \\\n        -c &#x27;:%s&#x2F;^M$&#x2F;&#x2F;g&#x27; \\\n        -c &#x27;wq&#x27;</code></pre>')
