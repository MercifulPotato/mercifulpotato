Item(by='shakna', descendants=None, kids=[24965423], score=None, time=1604297380, title=None, item_type='comment', url=None, parent=24964845, text='&gt; you still get the python3 in PATH, not a specific one (sbang aims to use a specific interpreter)<p>env lets you solve that one too. You can control PATH. From the FreeBSD manpages:<p><pre><code>    #!&#x2F;usr&#x2F;bin&#x2F;env -S -P&#x2F;usr&#x2F;local&#x2F;bin:&#x2F;usr&#x2F;bin perl\n</code></pre>\nAnd if you lack the -P flag... env controls the environment variables... So, you could extend (or replace) it like so:<p><pre><code>        #!&#x2F;usr&#x2F;bin&#x2F;env PATH=&#x2F;home&#x2F;my&#x2F;loc:${PATH} perl\n</code></pre>\n&gt; The -S argument to env is not POSIX<p>You&#x27;re right. POSIX only guarantees the -i flag, and the ability to set variables. It hasn&#x27;t been modified in the standard since 1991, despite other updates to the standard.<p>OpenBSD and Busybox only supports the crippled POSIX version. It isn&#x27;t completely portable.<p>Versions with -S and are available on Linux, FreeBSD (also has -P), macOS (also has -P).')
