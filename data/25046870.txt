Item(by='quesera', descendants=None, kids=None, score=None, time=1605021528, title=None, item_type='comment', url=None, parent=25046587, text='&gt; FreeBSD sed doesn&#x27;t have the in-place editing that GNU sed has<p>FWIW, that&#x27;s not correct. On BSD (incl macOS), the -i option takes an argument for the file extension to use as a tmp file:<p>So for GNU sed:<p><pre><code>  sed -i &#x27;s&#x2F;old&#x2F;new&#x2F;g&#x27; FILE\n</code></pre>\nTranslates to BSD sed as:<p><pre><code>  sed -i &#x27;&#x27; &#x27;s&#x2F;old&#x2F;new&#x2F;g&#x27; FILE\n</code></pre>\nIt is not recommended to use an empty string as the extension, but as long as your sed command does not fail due to disk space etc, it is fine.<p>Also, GNU sed takes an extension argument too, but it is optional:<p><pre><code>   sed -iEXT &#x27;s&#x2F;old&#x2F;new&#x2F;g&#x27; FILE</code></pre>')
