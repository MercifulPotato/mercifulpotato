I kept getting asked how large a dataset Typesense can handle. So I built a demo with the largest open structured dataset I could find.<p>You get instant search-as-you-type results in as little as <i>40ms</i> from (did I mention) 32 Million records!<p>Here&#x27;s the source code: <a href="https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;showcase-songs-search" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;showcase-songs-search</a><p>Some details about the tech stack:<p>- The search backend is powered by Typesense Server v0.17.0 running on a geo-distributed cluster (Oregon, Frankfurt, Mumbai) on Typesense Cloud: <a href="https:&#x2F;&#x2F;cloud.typesense.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;cloud.typesense.org&#x2F;</a><p>- The 32M songs dataset is from <a href="https:&#x2F;&#x2F;musicbrainz.org&#x27;s" rel="nofollow">https:&#x2F;&#x2F;musicbrainz.org&#x27;s</a> open library. Please contribute song metadata if you can.<p>- The Search UI was built with <a href="https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense-instantsearch-adapter" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense-instantsearch-adapter</a><p>- ParcelJS for an app bundler<p>- Deployment: `git push` &gt; Deploys to DigitalOcean&#x27;s App Platform<p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;typesense&#x2F;typesense</a>')
