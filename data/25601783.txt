Item(by='ithkuil', descendants=None, kids=None, score=None, time=1609490469, title=None, item_type='comment', url=None, parent=25596266, text='It seems that ubuntu 20.10 groovy gorilla is stuck at 1.43<p>As a library maintainer I need to make the choice of which version to target.<p>It&#x27;s easy to accidently use features of a recent rust version, as you can see in this example:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;mkmik&#x2F;slyce&#x2F;issues&#x2F;17" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;mkmik&#x2F;slyce&#x2F;issues&#x2F;17</a><p>I could work around this issue and pin the rustc version in the CI system to avoid regressions, but is it worth the effort since it&#x27;s likely that this particular user will eventually depend on another library that uses a feature of &gt;1.43 and thus eventually give in and use rustup? I want to be helpful towards users of my library, but I wonder if the most rational choice is to encourage users to upgrade the toolchain more liberally.')
