In the domain of structural transformations, one&#x27;s intuition combined with a decent type checker really does work, though.<p>Especially, I&#x27;ve done large refactorings of complex transformations, tracked down the typing errors, and then been pleasantly surprised when my test-suites passed the first time.<p>&gt; sqrt :: double -&gt; double<p>I can use QuickCheck[1]:<p><pre><code>    prop_Sqrt_Sqr n = sqrt n * sqrt n == abs n\n</code></pre>\nBecause it can exploit the type system, it can then plug in various values of Double to see if squaring my square root squares properly.<p>[1]: <a href="http:&#x2F;&#x2F;www.cse.chalmers.se&#x2F;~rjmh&#x2F;QuickCheck&#x2F;manual.html" rel="nofollow">http:&#x2F;&#x2F;www.cse.chalmers.se&#x2F;~rjmh&#x2F;QuickCheck&#x2F;manual.html</a>')
