Item(by='aidenn0', descendants=None, kids=None, score=None, time=1605257566, title=None, item_type='comment', url=None, parent=25078535, text='Path tracing is currently state-of-the-art for making photorealistic CG images because it can generate real reflections, refractions &amp;c. by simulating something very similar to how the physical world works.<p>A photo (or movie) is, however, taken through a lens that distorts the image (sometimes intentionally; what is and isn&#x27;t in focus in a composition is important for drawing the viewers attention).<p>A path tracer can already simulate this by just putting a bunch of pieces of glass shaped like the real lenses of a camera, <i>but</i> this is extremely computationally intensive.<p>This particular implementation uses a few (already known in the literature) techniques for making it faster: bidirectional tracing, selective oversampling, and thin-lens approximations.<p>This is an implementation of these techniques for Arnold, which is a renderer implementation used for many feature films[1].<p>1: <a href="https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Arnold_(software)" rel="nofollow">https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Arnold_(software)</a>')
