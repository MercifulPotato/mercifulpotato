I created this GitHub repo to use as my source material.<p><a href="https:&#x2F;&#x2F;github.com&#x2F;ryanrodemoyer&#x2F;writing_testable_code" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;ryanrodemoyer&#x2F;writing_testable_code</a><p>I created a cheesy pneumonic device to help remember the four principles.<p>La.S.I.C.<p>1. Loosely couple your code from the entry point.<p><pre><code>   * Entry points are consoles, apis, services, GUI&#x27;s, etc.\n\n      * The entry point always performs some type of setup or initialization.\n</code></pre>\n2. Simultaneously write code and write test.<p><pre><code>   * Will quickly discover what designer patterns are easy to easy, and which suck to test.\n</code></pre>\n3. Isolate dependencies using classes and interfaces.<p><pre><code>   * Use classes and interfaces to abstract&#x2F;isolate&#x2F;wrap away behavior with dependencies.\n</code></pre>\n4. Constructors are for declaring dependency needs, not initialization!')
