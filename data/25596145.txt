Item(by='nikki93', descendants=None, kids=None, score=None, time=1609442743, title=None, item_type='comment', url=None, parent=25591973, text='Refcounted refs have copy and destruct semantics, and then what you&#x27;re talking about is basically move semantics over those refs. Refcount increments be elided if the ref is &quot;moved.&quot; The increment is only needed on copy. Additionally, a moved-from value doesn&#x27;t need to be destructed (in the case of a refcounted ref -- decrementing the count and deallocating the pointed-at memory if count goes to zero). A talk about similar concepts in the &quot;Lobster&quot; language that explains it pretty well I think: <a href="https:&#x2F;&#x2F;youtu.be&#x2F;WUkYIdv9B8c" rel="nofollow">https:&#x2F;&#x2F;youtu.be&#x2F;WUkYIdv9B8c</a>')
