Item(by='TheRealPomax', descendants=None, kids=None, score=None, time=1605112331, title=None, item_type='comment', url=None, parent=25041772, text='Fun fact, I ran into the esampling problem myself  back in April when I posted <a href="https:&#x2F;&#x2F;github.com&#x2F;WebAudio&#x2F;web-audio-api&#x2F;issues&#x2F;30" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;WebAudio&#x2F;web-audio-api&#x2F;issues&#x2F;30</a>, V2 of the spec will allow this natively, but it also not seemingly making any headway towards initial release so who knows when it&#x27;ll land...<p>What you <i>can</i> do though is look at the first few bytes of an .mp3 file (since it&#x27;s a file drop&#x2F;file load) to just directly read the sample rate from the MP3 block header[1], where you directly check the value encoded by [data[19], data[20]]: if it&#x27;s [0,0] that means it&#x27;s 44100, [0,1] means it&#x27;s 48000, [1,0] means it&#x27;s 32000 and that&#x27;s it. There are no other sample rates allowed for MP3.<p>[1] <a href="http:&#x2F;&#x2F;mpgedit.org&#x2F;mpgedit&#x2F;mpeg_format&#x2F;MP3Format.html" rel="nofollow">http:&#x2F;&#x2F;mpgedit.org&#x2F;mpgedit&#x2F;mpeg_format&#x2F;MP3Format.html</a> for the full block format)')
