Item(by='ramchip', descendants=None, kids=[24881206, 24880420], score=None, time=1603557985, title=None, item_type='comment', url=None, parent=24880159, text='I think it’s a common trap for Erlang beginners to try to represent all entities (player, monster, room, etc.) as processes sending messages to each other, the way you’d use classes in OOP. It’s one way to learn the concepts, but in real apps it leads to messy, brittle code. Erlang processes really shine when they’re used as units of fault tolerance, not a code organization tool.<p>For an entity system ETS could be interesting, but it involves copying data on every read&#x2F;write which could be a problem in games...')
