Item(by='lmilcin', descendants=None, kids=None, score=None, time=1607112989, title=None, item_type='comment', url=None, parent=25306071, text='Because code coverage should measure coverage of REQUIREMENTS, not lines of code.<p>Right coverage metric would take as input list of all requirements broken into smallest conditions and then check if they all are verified by a unit test.<p>In particular, this would cause almost all code to be covered because if a line of code was not covered by test it would mean it is not necessary to meet any of the requirements.<p>Interestingly enough, unit tests would also be then complete documentation of every single requirement and that is how I understand &quot;test driven development&quot;. Sadly, I have yet to see it implemented anywhere.')
