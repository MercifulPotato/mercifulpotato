Then you can use an open source web UI rather than run the command-line tool.<p><pre><code>  cat &gt; youtube-dl-webui.Dockerfile &lt;&lt;EOF\n  FROM d0u9&#x2F;youtube-dl-webui\n  RUN pip3 install --upgrade youtube_dl\n  EOF\n  \n  cat &gt; conf.json &lt;&lt;EOCONF\n  { &quot;general&quot;: {&quot;download_dir&quot;: &quot;&#x2F;tmp&#x2F;ytdui&#x2F;download&quot;,&quot;db_path&quot;: &quot;&#x2F;tmp&#x2F;ytdui&#x2F;webui.db&quot;,&quot;log_size&quot;: 10},\n    &quot;server&quot;: {&quot;host&quot;: &quot;0.0.0.0&quot;,&quot;port&quot;: 5000} }\n  EOCONF\n  \n  docker build \\\n        -f youtube-dl-webui.Dockerfile \\\n        -t youtube-dl-webui:latest \\\n        .\n  \n  mkdir ytdui &amp;&amp; chmod 777 ytdui\n  docker run \\\n        --rm -it \\\n        --name youtube_dl_webui \\\n        -p 5000:5000 \\\n        -e FLASK_DEBUG=1 \\\n        -e CONF_FILE=&#x2F;conf.json \\\n        -v `pwd`&#x2F;conf.json:&#x2F;conf.json \\\n        -v `pwd`&#x2F;ytdui:&#x2F;tmp&#x2F;ytdui \\\n        youtube-dl-webui:latest\n  \n  firefox http:&#x2F;&#x2F;localhost:5000&#x2F;\n</code></pre>\nThat&#x27;s the simplest one to get running I think (<a href="https:&#x2F;&#x2F;github.com&#x2F;d0u9&#x2F;youtube-dl-webui" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;d0u9&#x2F;youtube-dl-webui</a>).
