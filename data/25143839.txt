In fact if anything it requires _less_ discipline, because a styling change can’t really break a layout elsewhere.<p>You avoid repeated classes by extracting components (either template partials, or actual components if you’re using a front-end framework). The components themselves define how they look, and then you assemble them, perhaps with a few additional classes for things like spacing.
