Item(by='compumike', descendants=None, kids=[24743378, 24742738], score=None, time=1602362149, title=None, item_type='comment', url=None, parent=24741077, text='I use MathJax for rendering equations on <a href="https:&#x2F;&#x2F;ultimateelectronicsbook.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;ultimateelectronicsbook.com&#x2F;</a> and it&#x27;s worked very well from a rendering quality perspective. I looked at Katex for performance, but it didn&#x27;t support all of the LaTeX formatting that I use.<p>You can scroll through <a href="https:&#x2F;&#x2F;ultimateelectronicsbook.com&#x2F;systems-of-equations&#x2F;" rel="nofollow">https:&#x2F;&#x2F;ultimateelectronicsbook.com&#x2F;systems-of-equations&#x2F;</a> for a particularly formatting-heavy example with lots of matrix equations and augmented matrices, working through Gaussian decomposition and LU factorization examples.<p>Instead, for performance, I wrote a small Ruby &#x2F; Jekyll plugin to do MathJax server-side rendering. It calls the mathjax-node npm package, with some caching on top because it&#x27;s so slow. This outputs static HTML and CSS with a few math fonts.<p>The mathjax-node-sre npm package even populates aria-label fields for each equation:<p><pre><code>    &lt;span class=&quot;mjx-math&quot; aria-label=&quot;y left-parenthesis x right-parenthesis equals StartFraction 1 Over 1 plus x EndFraction&quot;&gt;...&lt;&#x2F;span&gt;\n</code></pre>\nIf you have the option to consider server-side equation rendering, your users may thank you.')
