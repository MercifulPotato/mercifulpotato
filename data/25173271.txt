Item(by='jlokier', descendants=None, kids=None, score=None, time=1605998179, title=None, item_type='comment', url=None, parent=25173246, text='I agree with you, but the article puts in a polite request for filesystem changes already under &quot;potential enhancements&quot;.<p>My argument is if there were to be some work on potential enhancements that require kernel filesystem changes and syscall changes, fdatasync_rangev would be a more useful choice of enhancement, and probably easier to implement than fbarrier.<p>Better for performance, user convenience (no more separate files), and more kinds of applications (e.g. VMs and other kinds of in-file data store).<p>It&#x27;s not as big as it sounds.  fsync_range is already in FreeBSD, NetBSD and AIX, and Linux nearly got it too (there was a patch).  (Linux got sync_file_range instead, which is not useful for this unfortunately despite the similar name.)')
