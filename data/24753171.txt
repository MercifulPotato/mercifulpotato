Item(by='quietbritishjim', descendants=None, kids=[24754082, 24753983], score=None, time=1602500097, title=None, item_type='comment', url=None, parent=24752527, text='As many on the &#x2F;r&#x2F;cpp thread for this said, the ultimate result:<p><pre><code>    auto x = int(foo(blah));\n</code></pre>\nis no shorter than the version with an explicit template parameter for the return type:<p><pre><code>    auto x = foo&lt;int&gt;(blah);\n</code></pre>\nThat doesn&#x27;t apply to situations where you can genuinely take advantage of the implicit conversion:<p><pre><code>    takes_int_param(foo(blah));\n</code></pre>\nBut frankly the extra typing in an explicit version is worth it for improved code clarity:<p><pre><code>    takes_int_param(foo&lt;int&gt;(blah));\n</code></pre>\nThat&#x27;s just if you get it right - if you get something wrong and end up with a compiler error, the error message for the implicit conversion is going to be far more complex than for an explicit return type parameter.')
