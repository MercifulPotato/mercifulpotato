Item(by='carapace', descendants=None, kids=None, score=None, time=1608337816, title=None, item_type='comment', url=None, parent=25459506, text='The whole DRY concept irritates my inner curmudgeon because it is itself a lousy repetition of the (formal mathematical) concept of refactoring.  When you <i>refactor</i> code it&#x27;s just like <i>factoring</i> an algebraic equation.  If you&#x27;re just removing duplication without understanding how it affects the structure of the system then it&#x27;s a kind of &quot;cargo cult&quot; programming (IMO.)<p>Even when you know what you&#x27;re doing. there&#x27;s good refactoring[1] and bad refactoring[2].<p>[1] <a href="https:&#x2F;&#x2F;osdn.net&#x2F;projects&#x2F;joypy&#x2F;scm&#x2F;git&#x2F;Thun&#x2F;commits&#x2F;aa43eb668492e03a5d2c6cc5010479f6e0cba220" rel="nofollow">https:&#x2F;&#x2F;osdn.net&#x2F;projects&#x2F;joypy&#x2F;scm&#x2F;git&#x2F;Thun&#x2F;commits&#x2F;aa43eb6...</a>  This is a good clean bit of refactoring: it makes the branch symmetrical and lets you move code to a common post-branch step.<p>[2] <a href="https:&#x2F;&#x2F;github.com&#x2F;calroc&#x2F;HulloWurld&#x2F;blob&#x2F;master&#x2F;Hullo.html#L476-L488" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;calroc&#x2F;HulloWurld&#x2F;blob&#x2F;master&#x2F;Hullo.html#...</a>  This is a terrible function that, while it abstracts the core of the two following functions well, makes the system harder to understand.  In other words, the three factored functions are less desirable than the original pair of functions despite their redundancy, because the original functions were easy to understand and the new factored helper funtion is inscrutable.  Context counts.')
