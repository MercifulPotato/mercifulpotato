Item(by='jacques_chester', descendants=None, kids=None, score=None, time=1605120071, title=None, item_type='comment', url=None, parent=25057048, text='I think you&#x27;re on the edge of describing bitemporal databases. You have a range which represents &quot;time over which this fact is asserted to be true&quot; and a second range for &quot;time during which the assertion was recorded as valid&quot;. These typically get called &quot;valid time&quot; and &quot;transaction time&quot;.<p>You can express forecasts this way. The valid time range is the range for which you assert a given forecast is going to be true. The transaction time range is the time during which you held that forecast to be the current forecast.<p>Using transaction time you can then reconstruct your state of belief from any point in time. Using valid time you can make assertions about any fact in ranges over the past, present or future.<p>I think Snodgrass&#x27;s <i>Developing Time-Oriented Database Applications in SQL</i>[0] is still the best overall introduction, though the SQL used is fairly dated (ha). The relevant Wikipedia entry is OK too[1].<p>[0] <a href="https:&#x2F;&#x2F;www2.cs.arizona.edu&#x2F;~rts&#x2F;tdbbook.pdf" rel="nofollow">https:&#x2F;&#x2F;www2.cs.arizona.edu&#x2F;~rts&#x2F;tdbbook.pdf</a><p>[1] <a href="https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Temporal_database" rel="nofollow">https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Temporal_database</a>')
