These APIs in the `Symbol` backend [0] show this concept off really well I think.<p>The `Symbol` data structure knows how to, e.g. render its `Symbol#inspect` implementation [1]. To extract the underlying bytes associated with the `Symbol`, however, it needs an interpreter. But not a whole interpreter or any particular interpreter, just one that implements `Intern` [2].<p>All of these generics get monomorphized so there are no vtables or other indirection. Because it&#x27;s unlikely you&#x27;ll have more than one interpreter implementation in your program, there is no code bloat either.<p>[0]: <a href="https:&#x2F;&#x2F;github.com&#x2F;artichoke&#x2F;artichoke&#x2F;blob&#x2F;2d67d8537c1340f1f4981ca19a956d70d25ae585&#x2F;spinoso-symbol&#x2F;src&#x2F;lib.rs#L239-L326" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;artichoke&#x2F;artichoke&#x2F;blob&#x2F;2d67d8537c1340f1...</a><p>[1]: <a href="https:&#x2F;&#x2F;artichoke.github.io&#x2F;artichoke&#x2F;spinoso_symbol&#x2F;struct.Inspect.html" rel="nofollow">https:&#x2F;&#x2F;artichoke.github.io&#x2F;artichoke&#x2F;spinoso_symbol&#x2F;struct....</a><p>[2]: <a href="https:&#x2F;&#x2F;artichoke.github.io&#x2F;artichoke&#x2F;artichoke_core&#x2F;intern&#x2F;trait.Intern.html" rel="nofollow">https:&#x2F;&#x2F;artichoke.github.io&#x2F;artichoke&#x2F;artichoke_core&#x2F;intern&#x2F;...</a>')
