Item(by='xvector', descendants=None, kids=[25756391, 25755087, 25756163, 25759870], score=None, time=1610490253, title=None, item_type='comment', url=None, parent=25754624, text='&gt; I recall that many companies have switched from a perimeter model of defense where systems are secured from the outside to a &quot;defense in depth&quot; model where each system is secured on it&#x27;s own (plus the perimeter).<p>Yes, this is the current bleeding edge of cloud security, known as “zero trust.” Amongst other things, it usually involves provisioning mTLS identities in a secured manner to each service, with connections restricted to a whitelisted set of identities.<p>I found Evan Gilman and Doug Barth’s “Zero Trust Networks: Building Secure Systems in Untrusted Networks” [1] a pretty helpful read in understanding what modern&#x2F;next-gen cloud security looks like.<p>Some modern implementations of varying depth and scale include SPIRE [2], Tailscale [3], and BeyondCorp [4].<p>-—-<p>[1]: <a href="https:&#x2F;&#x2F;www.amazon.com&#x2F;Zero-Trust-Networks-Building-Untrusted&#x2F;dp&#x2F;1491962194" rel="nofollow">https:&#x2F;&#x2F;www.amazon.com&#x2F;Zero-Trust-Networks-Building-Untruste...</a><p>[2]: <a href="https:&#x2F;&#x2F;spiffe.io&#x2F;" rel="nofollow">https:&#x2F;&#x2F;spiffe.io&#x2F;</a><p>[3]: <a href="https:&#x2F;&#x2F;tailscale.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;tailscale.com&#x2F;</a><p>[4]: <a href="https:&#x2F;&#x2F;beyondcorp.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;beyondcorp.com&#x2F;</a>')
