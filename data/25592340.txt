Item(by='tarruda', descendants=None, kids=None, score=None, time=1609420525, title=None, item_type='comment', url=None, parent=25591489, text='&gt; Is it really an established idea in the security community that systemd is a security nightmare?<p>Here&#x27;s a list of systemd-specific CVEs over the years: <a href="https:&#x2F;&#x2F;www.cvedetails.com&#x2F;product&#x2F;38088&#x2F;Freedesktop-Systemd.html?vendor_id=7971" rel="nofollow">https:&#x2F;&#x2F;www.cvedetails.com&#x2F;product&#x2F;38088&#x2F;Freedesktop-Systemd...</a><p>Note that these apply to all programs in systemd distribution (not sure how many of these are specific to systemd init program).<p>It seems that a lot of criticism is due systemd doing more than a classic init process does, but some of it seem unjustified since you it is not an &quot;all or nothing&quot; package.<p>For example, this specific CVE affected systemd-resolved (systemd&#x27;s DNS daemon), but IIRC Debian was not affected since it did not use systemd-resolved by default: <a href="https:&#x2F;&#x2F;www.cvedetails.com&#x2F;cve&#x2F;CVE-2017-15908&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.cvedetails.com&#x2F;cve&#x2F;CVE-2017-15908&#x2F;</a><p>So it is perfectly fine for a distribution use systemd init system but use other network daemons.')
