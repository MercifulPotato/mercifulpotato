Item(by='octopoc', descendants=None, kids=None, score=None, time=1604582128, title=None, item_type='comment', url=None, parent=24997823, text='Here are some differences I was able to find:<p>- Uno uses the UWP API and XAML, whereas Avalonia uses a derivative of WPF<p>- Uno renders using the underlying platform, uses native controls whenever possible, particularly for input controls. Avalonia focuses on using its controls, not native ones (although support for native controls is WIP I think)<p>- Uno is based on WinUI, so in theory WinUI components should be usable in Uno apps<p>- Uno supports WebAssembly, but not Linux and MacOS natively<p>Links:<p>- <a href="https:&#x2F;&#x2F;github.com&#x2F;unoplatform&#x2F;uno&#x2F;issues&#x2F;17" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;unoplatform&#x2F;uno&#x2F;issues&#x2F;17</a><p>- <a href="https:&#x2F;&#x2F;github.com&#x2F;AvaloniaUI&#x2F;Avalonia&#x2F;issues&#x2F;587" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;AvaloniaUI&#x2F;Avalonia&#x2F;issues&#x2F;587</a>')
