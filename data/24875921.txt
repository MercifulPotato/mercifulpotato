Yet despite that fact thousands of the worlds most important Open Source projects continue to host their code on GitHub. People seem to have forgotten just how rotten Microsoft really is and how dangerous that situation is.<p>Developers probably know if github goes bad, they can relatively easily migrate their code off of github (much as many projects migrated away from sourceforge), and every developer already has a copy of the whole git repo.<p>Issues, pull requests, wiki, and actions are a bit harder to migrate but not really dealbreakers.<p>What evidence is there that Microsoft github is significantly worse (and&#x2F;or less trustworthy) than pre-Microsoft github?')
