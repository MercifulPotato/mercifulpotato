Item(by='tstack', descendants=None, kids=[25746878], score=None, time=1610407905, title=None, item_type='comment', url=None, parent=25738217, text='&gt; very few places AFAIK actually process their HTTP server logs into a form that can be queried by SQL.<p>There are at least two tools that support directly querying log files with SQL:<p>* logparser -- <a href="https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Logparser" rel="nofollow">https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Logparser</a><p>* lnav -- <a href="https:&#x2F;&#x2F;lnav.org" rel="nofollow">https:&#x2F;&#x2F;lnav.org</a><p>lnav uses SQLite and provides quite a few extensions, so quite a lot is possible.  The logs are exposed via virtual tables, so there is no separate import step.')
