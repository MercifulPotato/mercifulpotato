Item(by='rswail', descendants=None, kids=[24949064], score=None, time=1604116633, title=None, item_type='comment', url=None, parent=24944714, text='Because RPC is brittle. It requires both server and client to agree on the definition. It requires a shared IDL. If used as more than just the protobuf serialization format, it hides the <i>remote</i> part of RPC and programmers forget about network failures. It requires a different naming service from the standard internet naming service (DNS) which is brittle across network boundaries.<p>If you want to avoid bikeshedding for REST APIs, adopt a standard like JSON API [1].<p>Adopt a specification standard like OpenAPI [2] and JSON Schema [3]. There is tooling to give you the client&#x2F;server stubs like gRPC.<p>Implement using HTTP&#x2F;2 with fallback to HTTP and you get compression and multiple channels per connection.<p>gRPC is a repeat of CORBA is a repeat of ONC-RPC is a repeat of... there&#x27;s common reasons why RPC as a concept is brittle and tightly couples implementations of clients and servers.<p>[1] <a href="https:&#x2F;&#x2F;jsonapi.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;jsonapi.org&#x2F;</a>\n[2] <a href="https:&#x2F;&#x2F;swagger.io&#x2F;specification&#x2F;" rel="nofollow">https:&#x2F;&#x2F;swagger.io&#x2F;specification&#x2F;</a>\n[3] <a href="https:&#x2F;&#x2F;json-schema.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;json-schema.org&#x2F;</a>')
