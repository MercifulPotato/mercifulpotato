This provides the following benefits over using Hibernate&#x2F;JPA:<p>1. You craft the queries yourself and so there is no intermediate ORM language which you need to use.<p>2. It parameterises the queries so that you don&#x27;t have to append strings to get to the final query.<p><a href="https:&#x2F;&#x2F;dzone.com&#x2F;tutorials&#x2F;java&#x2F;spring&#x2F;spring-simple-jdbc-template.html" rel="nofollow">https:&#x2F;&#x2F;dzone.com&#x2F;tutorials&#x2F;java&#x2F;spring&#x2F;spring-simple-jdbc-t...</a><p>&lt;code&gt;<p>@Override\n public void insertForum(Forum forum) {<p><pre><code>  String query = &quot;INSERT INTO FORUMS (FORUM_ID, FORUM_NAME, FORUM_DESC) VALUES (?,?,?)&quot;;\n\n  jdbcTemplate.update(query, forum.getForumId(), forum\n    .getForumName(), forum.getForumDesc());\n\n }\n</code></pre>\n@Override\npublic Forum selectForum(int forumId) {<p><pre><code>  String query = &quot;SELECT * FROM FORUMS WHERE FORUM_ID=?&quot;;\n\n  return jdbcTemplate.queryForObject(query, new ParameterizedRowMapper&lt;Forum&gt;() {\n\n     @Override\n\n     public Forum mapRow(ResultSet resultSet, int rowNum) throws SQLException {\n\n      return new Forum(resultSet.getInt(&quot;FORUM_ID&quot;), resultSet.getString(&quot;FORUM_NAME&quot;), \n\n                           resultSet.getString(&quot;FORUM_DESC&quot;));\n\n     }\n\n    }, forumId);\n\n }</code></pre>\n&lt;&#x2F;code&gt;<p>3. It fetches only the columns that you need it to fetch and across multiple tables (if need be) without fetching the &quot;relationships&quot; between tables unnecessarily.<p>The example given below shows how to use RowMappers and to fetch only the columns you need:<p>&lt;code&gt;\n String query = &quot;SELECT FORUM_ID, FORUM_NAME, FORUM_DESC FROM FORUMS WHERE FORUM_ID=?&quot;;<p>return (Forum) jdbcTemplate.queryForObject(query, new Object[] { Integer.valueOf(forumId) },<p><pre><code>      new RowMapper() {\n\n         public Object mapRow(ResultSet resultSet, int rowNum) throws SQLException {\n\n                    return new Forum(resultSet.getInt(&quot;FORUM_ID&quot;), resultSet.getString(&quot;FORUM_NAME&quot;), \n                            resultSet.getString(&quot;FORUM_DESC&quot;));\n\n                }\n\n            });</code></pre>')
