Item(by='self_awareness', descendants=None, kids=None, score=None, time=1603366498, title=None, item_type='comment', url=None, parent=24845654, text='It&#x27;s easy to do on the beginning until you want to add a library that has a set of custom perl scripts to do the actual build.<p>Then you want to add a library that compiles fine on Fedora, but fails on ArchLinux, because they&#x27;ve moved some header file from `&#x2F;usr&#x2F;include` to `&#x2F;usr&#x2F;include&#x2F;subdir`.<p>Then you want to add another library, but you want to statically link it, only to find out that the package manager installs a version of the library that allows only dynamic linkage.<p>Then you want to add Windows support, and the whole thing detonates because Visual Studio uses completely different compilation switches than what you&#x27;d expect from a compiler in a unix world.<p>Next there are 1000 other problems, including people who insist on using raw Makefiles because they think they know better.<p>By the way, the fact that C doesn&#x27;t use rich signatures is a problem, not a feature. C ABI can also be changed by using different compilation options, but the user won&#x27;t even know it until the app explodes (crashes). At least C++ will emit a linker error in such case (sometimes, other times it will explode as well).')
