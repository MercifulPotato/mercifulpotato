Item(by='codebje', descendants=None, kids=[24750533], score=None, time=1602457282, title=None, item_type='comment', url=None, parent=24749658, text='I mean that source code comes with costs, often substantial, but has no direct benefits.<p>It&#x27;s easy for us as developers to think that source code is valuable - but this leads to problems like never removing code &quot;in case it&#x27;s needed&quot;, or with an in-house dev team developing systems you could get off the shelf.<p>If code is an asset, then it makes a lot more sense to write stuff yourself: you not only get the artefact, you also get the source.<p>If it&#x27;s a liability, then it makes a lot more sense to let someone else bear the costs of that liability, especially if they have economies of scale, except where you can&#x27;t get your desired outcome other than writing code.<p>This is of course technically incorrect. It&#x27;s perhaps more accurate to say that source code requires upkeep and is expensive to maintain. That tends to draw less interest and discussion, because it&#x27;s &quot;obvious.&quot; Except as an industry we&#x27;re overall pretty lousy at paying the required upkeep on code.')
