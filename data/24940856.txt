Item(by='jamessb', descendants=None, kids=[24941855], score=None, time=1604053804, title=None, item_type='comment', url=None, parent=24939733, text='&gt;  Looks like this library generates static graphs.<p>It can create interactive charts [0].<p>&gt; Is there any specific reason to use this, as opposed to say, Apache Echarts<p>I&#x27;ve never used it, so my initial impressions may be mistaken, but ECharts looks much less declarative.<p>It&#x27;s interesting to compare the specifications for a bubble chart in both systems [1, 2].<p>The ECharts example [1] first specifies a <i>chart-type</i> (&quot;scatter&quot;), which seems to be hard-coded to use the first two elements of each entry in the data array as the x and y positions. This is then customised by writing JavaScript <i>functions</i> to set the symbol size and color.<p>In contrast, the Vega-Lite example [2] defines the chart fully declaratively - you first set the <i>mark</i> type to circle, then specify the data encoding which defines how each variable maps to each attribute. This mapping is properly declarative - it isn&#x27;t just a manually-defined function.<p>If you had a multidimensional dataset and wanted to change which variables you want to plot, it looks like you&#x27;d need to reshape the data array if you were using ECharts, whereas you could just change the &quot;field&quot; attributes in the encoding part of a Vega-Lite specification. This makes Vega-Lite more convenient for exploratory data analysis.<p>The way Vega-Lite represents these encodings is convenient - a recently created library by Krist Wongsuphasawat tries to expose a similar interface to other visualisation components [3].<p>[0]: <a href="https:&#x2F;&#x2F;vega.github.io&#x2F;vega-lite&#x2F;examples&#x2F;#interactive" rel="nofollow">https:&#x2F;&#x2F;vega.github.io&#x2F;vega-lite&#x2F;examples&#x2F;#interactive</a><p>[1]: <a href="https:&#x2F;&#x2F;echarts.apache.org&#x2F;examples&#x2F;en&#x2F;editor.html?c=bubble-gradient" rel="nofollow">https:&#x2F;&#x2F;echarts.apache.org&#x2F;examples&#x2F;en&#x2F;editor.html?c=bubble-...</a><p>[2]: <a href="https:&#x2F;&#x2F;vega.github.io&#x2F;vega-lite&#x2F;examples&#x2F;circle_natural_disasters.html" rel="nofollow">https:&#x2F;&#x2F;vega.github.io&#x2F;vega-lite&#x2F;examples&#x2F;circle_natural_dis...</a><p>[3]: <a href="https:&#x2F;&#x2F;encodable.vercel.app&#x2F;" rel="nofollow">https:&#x2F;&#x2F;encodable.vercel.app&#x2F;</a>')
