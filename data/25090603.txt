Item(by='gibs0ns', descendants=None, kids=[25091529], score=None, time=1605333765, title=None, item_type='comment', url=None, parent=25075296, text='I discovered Babylon last year iirc and was impressed by demos, docs, and support by Microsoft so I figured I wanted to build something worthwhile in it to try it out.<p>I decided to start developing a 3rd-person MMORPG using Babylon v4.0 as a side project, however after investing far too much time and getting it to a decent point, I came to question why I was bothering with a strictly web-based engine for a rather complex and extensive game.\nIf I was going to go to the effort of creating this game, why not use some kind of game engine that can then export it to multiple platforms including web?\nSo since then I&#x27;ve halted dev on the babylon version of the game, and begun porting to the Godot game engine, which can export to web among other platforms.<p>With all that said, I still enjoyed my experience using Babylon, it was just the wrong tool for my chosen project.<p>Babylon is a great choice for a lot of web based 3d content, but if you&#x27;re intending to build any kind of significant game I&#x27;d recommend using a proper game engine from the start as it will enable you more flexibility in the long term.<p>If however you do require a character controller for Babylon, I used babylonjs-charactercontroller [1] for my project and found it dynamic enough to work for most situations.<p>[1] <a href="https:&#x2F;&#x2F;www.npmjs.com&#x2F;package&#x2F;babylonjs-charactercontroller" rel="nofollow">https:&#x2F;&#x2F;www.npmjs.com&#x2F;package&#x2F;babylonjs-charactercontroller</a>')
