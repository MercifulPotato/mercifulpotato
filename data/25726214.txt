I felt it strongly at the time too that Android didn&#x27;t need anti-Malware [0] but I don&#x27;t anymore.<p>Some 5 years ago, I switched to Firefox Mobile out of annoyance because Chrome refused to block all those popups random websites would show with some prompting an app install.<p>Google has since made a <i>lot</i> of improvements including tightening up which apps can install other apps (not a blanket permission anymore), running the <i>Potentially Harmful Applications</i> program, narrowing down fingerprinting (still some way to go as evident by TFA). Google is even locking the code up that runs <i>outside</i> of Android in <i>crosvm</i>s [1]. I&#x27;m positive, things will improve [2] even if slowly because Android, at this point, is the most widely distributed OS and they can&#x27;t move as fast anymore without hurting developers and users.<p>Google did implement what they call <i>AppOps</i> (2013) which paved for tremendous amount of user control over app permissions. They removed <i>AppOps</i> citing that it was never meant to be used by end-users but by <i>AOSP</i> and app developers [3].<p>Fortunately, if Android is rooted, one can use <i>AppOps</i> [4]; but then rooting exposes one to an incredible amount.<p>Besides, there are LittleSnitch-esque firewalls for both root and non-root devices [5].<p>[0] <a href="https:&#x2F;&#x2F;arstechnica.com&#x2F;information-technology&#x2F;2011&#x2F;11&#x2F;mobile-security-vendors-we-are-not-charlatans-and-scammers&#x2F;" rel="nofollow">https:&#x2F;&#x2F;arstechnica.com&#x2F;information-technology&#x2F;2011&#x2F;11&#x2F;mobil...</a><p>[1] <a href="https:&#x2F;&#x2F;youtu.be&#x2F;edqJSzsDRxk" rel="nofollow">https:&#x2F;&#x2F;youtu.be&#x2F;edqJSzsDRxk</a><p>[2] <a href="https:&#x2F;&#x2F;android-developers.googleblog.com&#x2F;search&#x2F;label&#x2F;Security" rel="nofollow">https:&#x2F;&#x2F;android-developers.googleblog.com&#x2F;search&#x2F;label&#x2F;Secur...</a><p>[3] <a href="https:&#x2F;&#x2F;www.zdnet.com&#x2F;article&#x2F;google-removes-awesome-but-unintended-privacy-controls-in-android-4-4-2&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.zdnet.com&#x2F;article&#x2F;google-removes-awesome-but-uni...</a><p>[4] <a href="https:&#x2F;&#x2F;github.com&#x2F;MuntashirAkon&#x2F;AppManager" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;MuntashirAkon&#x2F;AppManager</a><p>[5] <a href="https:&#x2F;&#x2F;awesomeopensource.com&#x2F;projects&#x2F;android&#x2F;firewall" rel="nofollow">https:&#x2F;&#x2F;awesomeopensource.com&#x2F;projects&#x2F;android&#x2F;firewall</a>')
