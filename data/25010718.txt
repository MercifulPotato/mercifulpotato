Item(by='jitl', descendants=None, kids=[25011656, 25010755, 25011744, 25010754, 25010800], score=None, time=1604693682, title=None, item_type='comment', url=None, parent=25008308, text='SQLite has a system that abstracts various low-level storage operations called VFS - this is the layer that allows SQLite to be portable across different operating systems and environments: <a href="https:&#x2F;&#x2F;www.sqlite.org&#x2F;vfs.html" rel="nofollow">https:&#x2F;&#x2F;www.sqlite.org&#x2F;vfs.html</a><p>Recently I’ve wondered how possible it would be to implement a SQLite VFS on top of IndexedDB - and, would such a VFS be competitive in speed to using IndexedDB directly? Or, would it be equivalent to use Emscripten’s existing POSIX-ish filesystem backed by IndexedDB?<p>An IndexedDB VFS would allow sql.js to durably persist data in the browser.')
