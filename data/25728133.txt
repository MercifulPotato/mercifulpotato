Item(by='dgb23', descendants=None, kids=[25729470], score=None, time=1610368953, title=None, item_type='comment', url=None, parent=25727433, text='The claims in the article sound weak, because they communicate in an informal and natural way.<p>&quot;Haskell&#x27;s type system is more expressive than X and Y&quot; is a strong claim and can be proven by showing that X and Y need to compose run-time workarounds for a given property that can be checked statically in Haskell.<p>&quot;Functional Programming reduces the surface area for Bugs&quot; is a strong claim and can be proven by showing that a single mutable reference strictly introduces a set of possible bugs that were not possible before and that these bugs cannot be checked in language X.<p>It is kind of annoying that these discussions often seem superficial, cultural or partisan, when in fact they could be much more rigorous.<p>Now, If we assume or find these claims to be true we can finally proceed with the real discussion: What are the costs and benefits of these properties in a given setting?')
