Item(by='estebank', descendants=None, kids=None, score=None, time=1606773787, title=None, item_type='comment', url=None, parent=25257492, text='Those subpar diagnostics are <i>a bug</i>. The underlying reasons are that<p>- The async&#x2F;await feature <i>is an MVP</i><p>- async&#x2F;await desugars to code you can write yourself that leverages a bunch of relatively advanced features, namely trait bounds and associated types<p>- It also leverages a nightly only feature (generators)<p>- Async programming is inherently harder :)<p>- We&#x27;ve had less time to see how people <i>misuse</i> the features to see <i>where</i> they land in order to clean up those cases and make the errors as user friendly as they can be<p>Put all of those together and the experience of writing async code in Rust is similar to the experience of writing regular Rust code maybe two or three years ago. When you encounter things like this, please file a ticket at <a href="https:&#x2F;&#x2F;github.com&#x2F;rust-lang&#x2F;rust&#x2F;issues" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;rust-lang&#x2F;rust&#x2F;issues</a> so that we can fix them.')
