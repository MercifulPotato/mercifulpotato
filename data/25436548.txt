Item(by='neilparikh', descendants=None, kids=None, score=None, time=1608073338, title=None, item_type='comment', url=None, parent=25435402, text='There&#x27;s a real reason to do something similar if you&#x27;re developing for a memory constrained devices with a API someone else will use.<p>If you have objects your API functions return that API consumers will use, you want to make sure the objects don&#x27;t suddenly grow in size, since that would eat into the memory that the consumers expect to use. So, you&#x27;d pad those objects with unused fields, and then if you need to add a new field, you can just use that padding area.<p>Something similar applies if you&#x27;re sending objects over the wire, since you may not want to increase the size of the message later.')
