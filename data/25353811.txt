Item(by='isbvhodnvemrwvn', descendants=None, kids=None, score=None, time=1607474936, title=None, item_type='comment', url=None, parent=25351477, text='It&#x27;s not strictly speaking serializable, what they described is a lost update anomaly, to avoid which repeatable read or snapshot isolation is sufficient.<p>Serializable (or serializable snapshot isolation) is stronger, it doesn&#x27;t allow anomalies such as write skew, but also a lot more expensive since you need to keep track of changes in rows matched by predicates to avoid phantom reads (as compared to just keeping track of the rows returned by the query with predicates in this particular transaction).<p>Also worth noting that some DBs such as Oracle actually lie about implementing serializable (in this case they only offer snapshot isolation), so it&#x27;s worth keeping that in mind and use locks if necessary.')
