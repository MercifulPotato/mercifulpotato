Item(by='jlgaddis', descendants=None, kids=None, score=None, time=1607995515, title=None, item_type='comment', url=None, parent=25414881, text='It seems you missed an opportunity (BTW, is that what they call &quot;content marketing&quot;?).<p>Had you left your VM running a little while longer, you might&#x27;ve noticed something else in your Netflow data: yet another service calling home to Canonical twice a day.<p>I wish I were joking about this but... the (likely) &quot;primary&quot; reason for this is so that they can <i>show ads to users</i> when they log in -- ads for things like TV shows or software for other operating systems or, sometimes, just a &quot;fun fact&quot;.<p>I can&#x27;t really say it&#x27;s the <i>only</i> reason, however. With all of the system information they&#x27;re getting off of each host, there&#x27;s now several different things they can use it for and, of course, that provides them with even <i>more</i> motivation to keep doing it!<p>This isn&#x27;t a new thing, though. In fact, people have been expressing their &quot;unhappiness&quot; about it to Canonical for a few years now [0,1,2].<p>Unfortunately:<p>- it&#x27;s still installed by default<p>- it&#x27;s still enabled by default<p>- they&#x27;ve actually <i>increased</i> the amount of system info that they&#x27;re collecting [3]<p>- it&#x27;s installed as part of the &quot;base-files&quot; package -- an &quot;essential&quot; package which <i>cannot be removed!</i><p>- &quot;it&#x27;s a &#x27;feature&#x27;, not a bug&quot; (so we absolutely should <i>not</i> expect it to get removed)<p>- apparently, a 42 line MOTD [4] is perfectly acceptable<p>I suppose there&#x27;s one &quot;positive&quot; thing I can say about it: at least they&#x27;re using TLS to encrypt the data over the wire. :&#x2F;<p>--<p>If you want to disable this spyware on your Ubuntu hosts, change &quot;ENABLED&quot; to &quot;0&quot; in &#x2F;etc&#x2F;default&#x2F;motd-news:<p><pre><code>  $ sudo sed -i -e &#x27;&#x2F;^ENABLED=1$&#x2F;s&#x2F;1&#x2F;0&#x2F;&#x27; &#x2F;etc&#x2F;default&#x2F;motd-news\n</code></pre>\nWhen executed, &#x2F;etc&#x2F;update-motd.d&#x2F;50-motd-news checks if &quot;ENABLED&quot; is set to &quot;1&quot;. If not, it exits without calling home.<p>50-motd-news gets launched by motd-news.service which itself is activated -- twice per day -- by motd-news.timer. Since I like to be thorough, I recommend running a few more commands (as it&#x27;s impossible to know that Canonical won&#x27;t change any of this in the future!):<p><pre><code>  $ sudo systemctl mask --now motd-news.service\n  $ sudo systemctl mask --now motd-news.timer\n  $ sudo systemctl clean --what=state motd-news.timer\n  $ sudo rm -f &#x2F;var&#x2F;cache&#x2F;motd-news\n</code></pre>\nThat should do it. If you&#x27;re rather not have to worry about something like this in the future, you may want to consider replacing Ubuntu Linux with Debian GNU&#x2F;Linux.<p>--<p>[0]: <a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=14662088" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=14662088</a><p>[1]: <a href="https:&#x2F;&#x2F;old.reddit.com&#x2F;r&#x2F;linux&#x2F;comments&#x2F;98ctgu&#x2F;ubuntu_server_including_ads_in_the_terminal&#x2F;" rel="nofollow">https:&#x2F;&#x2F;old.reddit.com&#x2F;r&#x2F;linux&#x2F;comments&#x2F;98ctgu&#x2F;ubuntu_server...</a><p>[2]: <a href="https:&#x2F;&#x2F;dev.to&#x2F;fivnex&#x2F;open-letter-to-canonical-ltd-a1k" rel="nofollow">https:&#x2F;&#x2F;dev.to&#x2F;fivnex&#x2F;open-letter-to-canonical-ltd-a1k</a><p>[3]: <a href="https:&#x2F;&#x2F;bugs.launchpad.net&#x2F;ubuntu&#x2F;+source&#x2F;base-files&#x2F;+bug&#x2F;1840946" rel="nofollow">https:&#x2F;&#x2F;bugs.launchpad.net&#x2F;ubuntu&#x2F;+source&#x2F;base-files&#x2F;+bug&#x2F;18...</a><p>[4]: <a href="https:&#x2F;&#x2F;joe.schafer.dev&#x2F;chatty-ubuntu-motd&#x2F;" rel="nofollow">https:&#x2F;&#x2F;joe.schafer.dev&#x2F;chatty-ubuntu-motd&#x2F;</a>')
