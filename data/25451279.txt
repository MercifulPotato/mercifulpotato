Item(by='jlgaddis', descendants=None, kids=None, score=None, time=1608169203, title=None, item_type='comment', url=None, parent=25450046, text='There&#x27;s a few ways to try to &quot;workaround&quot; this issue, typically involving multiple &quot;Host&quot; entries in ~&#x2F;.ssh&#x2F;config.<p>I haven&#x27;t actually tried it yet myself, but I think my preference would be to set the &quot;IdentityFile&quot; option to an environment variable (i.e., &quot;${SSH_IDENTITYFILE}&quot;) in my ~&#x2F;.ssh&#x2F;config file, within an explicit &quot;Host&quot; definition for &quot;github.com&quot;).<p>In my shell initialization files, I&#x27;d set $SSH_IDENTITYFILE to either &quot;%d&#x2F;.ssh&#x2F;id_ed25519_github_personal&quot; or &quot;%d&#x2F;.ssh&#x2F;id_ed25519_github_work&quot; (for example), depending on which I wanted to be the default.<p>When I wanted to use the other account, all I would need to do would be change which of the keys $SSH_IDENTITYFILE pointed to.<p>--<p>Pro-tip: you&#x27;ll also want to set &quot;ControlPath none&quot; and &quot;IdentitiesOnly yes&quot; for &quot;Host github.com&quot;.')
