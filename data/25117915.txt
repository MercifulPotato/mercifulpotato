Item(by='jazzyjackson', descendants=None, kids=None, score=None, time=1605565905, title=None, item_type='comment', url=None, parent=25115083, text='It&#x27;s not really an operator, it&#x27;s part of a pattern-matching syntax. &quot;_&quot; is called Blank and matches anything. Putting a name in front of it just makes it available by name -- like referring to capture groups in a regex.<p>So when you call a function on some argument or list of arguments, the interpreter looks up all the function definitions and finds one with a pattern that matches.<p><pre><code>  Double[x_] := 2 * x\n</code></pre>\nWill work for Double[3] but not for Double[3, 2], the latter doesn&#x27;t match any patterns, and the interpreter doesn&#x27;t know what to do with it. You would have to define another function with the same name to handle other patterns, like overloading in C++<p>You can further constrain the pattern to types like this:<p><pre><code>  Double[x_Integer] := 2 * x\n</code></pre>\nAnd now the pattern only matches integers. \nMore in the manual, &quot;Patterns&quot;[0] and &quot;More about Patterns&quot;[1]<p>[0] <a href="https:&#x2F;&#x2F;www.wolfram.com&#x2F;language&#x2F;elementary-introduction&#x2F;2nd-ed&#x2F;32-patterns.html" rel="nofollow">https:&#x2F;&#x2F;www.wolfram.com&#x2F;language&#x2F;elementary-introduction&#x2F;2nd...</a><p>[1] <a href="https:&#x2F;&#x2F;www.wolfram.com&#x2F;language&#x2F;elementary-introduction&#x2F;2nd-ed&#x2F;41-more-about-patterns.html" rel="nofollow">https:&#x2F;&#x2F;www.wolfram.com&#x2F;language&#x2F;elementary-introduction&#x2F;2nd...</a>')
