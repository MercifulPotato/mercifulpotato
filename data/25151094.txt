Item(by='steveklabnik', descendants=None, kids=[25151940], score=None, time=1605803306, title=None, item_type='comment', url=None, parent=25151037, text='Some things want an array of a specific size. If you have a function like <a href="https:&#x2F;&#x2F;doc.rust-lang.org&#x2F;stable&#x2F;std&#x2F;primitive.u16.html#method.from_be_bytes" rel="nofollow">https:&#x2F;&#x2F;doc.rust-lang.org&#x2F;stable&#x2F;std&#x2F;primitive.u16.html#meth...</a><p><pre><code>  pub const fn from_be_bytes(bytes: [u8; 2]) -&gt; u16\n</code></pre>\nyou can&#x27;t pass it a slice. This would let you pass a vector to this function.<p>Now, because this wasn&#x27;t possible previously, it means many things take a slice, and then check that the length is the length they expect, because that ends up being easier for folks. Stuff like this helps it be able to be done properly.<p>(This specific function is one example that is done in this style, and it&#x27;s a pain. I have real world code that looks like<p><pre><code>  let foo = u16::from_be_bytes([some_slice[0], some_slice[1]]);\n</code></pre>\nThis gets even worse with say, u128::from_be_bytes.)')
