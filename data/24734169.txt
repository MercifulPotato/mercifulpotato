Item(by='jhallenworld', descendants=None, kids=[24735278, 24735454, 24734956, 24734498], score=None, time=1602275794, title=None, item_type='comment', url=None, parent=24730713, text='I don&#x27;t like either syntax.  Wikipedia has this example:<p>QUEL:<p><pre><code>    range of E is EMPLOYEE\n    retrieve into W\n    (COMP = E.Salary &#x2F; (E.Age - 18))\n    where E.Name = &quot;Jones&quot;\n</code></pre>\nSQL:<p><pre><code>    select (e.salary &#x2F; (e.age - 18)) as comp\n    from employee as e\n    where e.name = &quot;Jones&quot;\n</code></pre>\nI would prefer an operator syntax that directly mimics relational algebra.  Something like:<p><pre><code>    w = employee(name == &quot;Jones&quot;)[comp = salary &#x2F; (age - 18)]\n</code></pre>\nSo () is &quot;where&quot;, [] is &quot;project&quot; (choose or create columns) and you can use * for join and + for union.  The result is a table with a column named comp.')
