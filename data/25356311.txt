Item(by='nyanpasu64', descendants=None, kids=None, score=None, time=1607499945, title=None, item_type='comment', url=None, parent=25347241, text='Qt Widgets does <i>not</i> use native widgets on Win32 (I heard they&#x27;re called &quot;windows&quot; there). It only spawns a single native window, and performs all input handling itself. Qt 5 has a mode (<a href="https:&#x2F;&#x2F;doc.qt.io&#x2F;qt-5&#x2F;qwidget.html#native-widgets-vs-alien-widgets" rel="nofollow">https:&#x2F;&#x2F;doc.qt.io&#x2F;qt-5&#x2F;qwidget.html#native-widgets-vs-alien-...</a>) which uses native Win32 widgets, but they&#x27;re more like blank canvases drawn by Qt, than actual Win32 push buttons, and still look slightly off. And it reduces performance.')
