Item(by='LukeEF', descendants=None, kids=[24735855, 24735007], score=None, time=1602280298, title=None, item_type='comment', url=None, parent=24732943, text='TerminusDB (co founder here) was partially inspired by the git scraping approach to the revision control for data problem. We built a database that gives you all of the functionality of git, but in a database so you can query and with a commit graph etc. Has git semantics for clone, fork, rebase, merge and the other major functions. We store &amp; share deltas and use succinct data structures that allow us to pass around in-memory DBs that you can then query in place. We&#x27;re relatively new, but happy to see versioning data wherever it might land. Open source forever so really trying to be git for data.
