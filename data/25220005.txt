Google definitely uses third party code but it is outstanding how much is proprietary Google owned and authored. For this code you don&#x27;t release new versions and update, instead you evolve APIs gradually (as the other commenters have explained).<p>For third party dependencies there are two main options.<p>1. Do it all at once. This is good for relatively small numbers of users or small API changes.<p>2. Introduce the new version, migrate users, then delete the old version.<p>Note that for third party libraries Google does maintain a fairly strict one version policy so 2 is always a temporary measure.<p><a href="https:&#x2F;&#x2F;opensource.google&#x2F;docs&#x2F;thirdparty&#x2F;oneversion&#x2F;" rel="nofollow">https:&#x2F;&#x2F;opensource.google&#x2F;docs&#x2F;thirdparty&#x2F;oneversion&#x2F;</a>')
