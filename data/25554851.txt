Item(by='MaxBarraclough', descendants=None, kids=None, score=None, time=1609108371, title=None, item_type='comment', url=None, parent=25554018, text='Interesting idea, pity about the name. There&#x27;s a vaguely similar effort with a similarly poor choice of name, <i>Clean C</i>, which is guaranteed to compile as both C and C++ with the same behaviour (provided there&#x27;s no undefined behaviour or platform-specific behaviour). [0][1]<p>&gt; Exception handling is the only C++ language feature which requires significant support from a complex runtime system, and it&#x27;s the only C++ feature that has a runtime cost even if you don&#x27;t use it<p>I don&#x27;t think that&#x27;s true. I believe that if you enable RTTI you pay for it even if you don&#x27;t use it, which is why it&#x27;s banned in the LLVM codebase. [2]<p>&gt; Don&#x27;t use C++ runtime wrapper for C runtime includes (&lt;cstdio&gt;, &lt;cmath&gt;, etc.), use C runtime instead (&lt;stdio.h&gt;, &lt;math.h&gt;, etc.)<p>I believe this is still supported, but is officially deprecated, for what that&#x27;s worth.
