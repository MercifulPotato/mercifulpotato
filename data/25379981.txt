Item(by='emeraldpanda', descendants=None, kids=[25380217], score=None, time=1607641282, title=None, item_type='comment', url=None, parent=25379831, text='The building blocks for location based AR are pretty much all there:<p>- Azure Spatial Anchors (iOS, Android, HoloLens, ROS) [1]<p>- Google Cloud Anchors (iOS, Android) [2]<p>- Apple Geo Anchors (iOS &amp; not all locations so not ready for use yet) [3]<p>Integration with these SDKs is pretty easy, and Azure Spatial Anchors and Google Cloud Anchors are production-ready. Minecraft Earth for example used Azure Spatial Anchors for shared multiplayer AR experiences. Infopop is nice, but doesn&#x27;t have multiplayer or long-term persistence, so not really useful as-is. But wouldn&#x27;t be surprised if it eventually uses one of these SDKs to provide better multiplayer + shared relocalization. You could probably throw together a rough version of Infopop in a week if sufficiently motivated, ARKit does a lot of the heavy lifting there.<p>[1]: (<a href="https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;azure&#x2F;spatial-anchors&#x2F;overview" rel="nofollow">https:&#x2F;&#x2F;docs.microsoft.com&#x2F;en-us&#x2F;azure&#x2F;spatial-anchors&#x2F;overv...</a>)<p>[2]: (<a href="https:&#x2F;&#x2F;developers.google.com&#x2F;ar&#x2F;develop&#x2F;java&#x2F;cloud-anchors&#x2F;overview-android" rel="nofollow">https:&#x2F;&#x2F;developers.google.com&#x2F;ar&#x2F;develop&#x2F;java&#x2F;cloud-anchors&#x2F;...</a>)<p>[3]: (<a href="https:&#x2F;&#x2F;developer.apple.com&#x2F;documentation&#x2F;arkit&#x2F;argeoanchor" rel="nofollow">https:&#x2F;&#x2F;developer.apple.com&#x2F;documentation&#x2F;arkit&#x2F;argeoanchor</a>)')
