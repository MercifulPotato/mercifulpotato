Item(by='ThePhysicist', descendants=None, kids=[24966946, 24967364, 24966425], score=None, time=1604311556, title=None, item_type='comment', url=None, parent=24965671, text='In most cases, encrypting sensitive information like e-mail addresses with a memory-resident key (e.g. injected using tools like Vault) in the application layer is a better strategy, at least if you need asynchronous access to that information (e.g. to send out weekly update e-mails). Most of the data leaks in the past were caused by compromised or misconfigured databases, not by compromised application server code.<p>Also, within the EU I need to be able to proactively reach my users (e.g. to notify them about a data loss), so only storing hashes of e-mail addresses and hoping users will log in so that I can send them an e-mail won&#x27;t work.')
