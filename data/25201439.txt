I like FreeBSD, but I wouldn&#x27;t really recommend this particular cheatsheet.  Why?  Some examples from the top (and if the top of the cheatsheet is problematic, it&#x27;s not a useful quick reference IMO):<p>1. mount_msdosfs: The &#x27;ad&#x27; name is largely historical; you&#x27;ll see &#x27;ada&#x27; and &#x27;da&#x27; these days (the latter for USB drives)[1].<p>2. ISO9660 is also basically historical at this point; UDF is more likely.  And it&#x27;s useful to split out mdconfig (memory disk config) as its own command -- it&#x27;s useful in a lot of contexts, and separating it might make that more obvious to someone reading.<p>3. Linux procfs and Linux procfs?  This is confusing and doesn&#x27;t help users looking to run Linux programs (a lot more work is needed; see [2]).  Creating &#x2F;proc because it doesn&#x27;t exist is odd; the directory should exist in FreeBSD installations and images, regardless of whether procfs is mounted[3].<p>Get what you will from the document; if it helps you, great.<p>[1]: <a href="https:&#x2F;&#x2F;lists.freebsd.org&#x2F;pipermail&#x2F;freebsd-questions&#x2F;2016-September&#x2F;273636.html" rel="nofollow">https:&#x2F;&#x2F;lists.freebsd.org&#x2F;pipermail&#x2F;freebsd-questions&#x2F;2016-S...</a><p>[2]: <a href="https:&#x2F;&#x2F;wiki.freebsd.org&#x2F;LinuxJails" rel="nofollow">https:&#x2F;&#x2F;wiki.freebsd.org&#x2F;LinuxJails</a> , previously: <a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24971456" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24971456</a><p>[3]: <a href="https:&#x2F;&#x2F;github.com&#x2F;freebsd&#x2F;freebsd&#x2F;blob&#x2F;master&#x2F;etc&#x2F;mtree&#x2F;BSD.root.dist#L116" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;freebsd&#x2F;freebsd&#x2F;blob&#x2F;master&#x2F;etc&#x2F;mtree&#x2F;BSD...</a>')
