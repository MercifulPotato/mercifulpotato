Item(by='1vuio0pswjnm7', descendants=None, kids=None, score=None, time=1604979821, title=None, item_type='comment', url=None, parent=25034456, text='To use a single third party utility like dig that would retrieve entries from both &#x2F;etc&#x2F;hosts and from zone files, one could use a program like pdns_recursor with the &quot;--export-etc-hosts=on&quot; option.<p><a href="https:&#x2F;&#x2F;doc.powerdns.com&#x2F;recursor&#x2F;manpages&#x2F;pdns_recursor.1.html" rel="nofollow">https:&#x2F;&#x2F;doc.powerdns.com&#x2F;recursor&#x2F;manpages&#x2F;pdns_recursor.1.h...</a><p>Assuming for example, one had HOSTS entries<p><pre><code>   127.0.0.1 localhost\n   93.184.216.34  example.com \n</code></pre>\npdns_recursor is listening on 127.0.0.1 and &#x2F;etc&#x2F;resolv.conf contains namserver 127.0.0.1, then<p><pre><code>   dig ns example.com \n</code></pre>\nshould return &quot;localhost&quot; as the NS RR for example.com so we know the A RR came from &#x2F;etc&#x2F;hosts not a.iana-servers.net<p>Additionally, adding the &quot;--trace=on&quot; option will output debugging info that will tell whether the answer came from &quot;local auth storage&quot;, e.g., &#x2F;etc&#x2F;hosts')
