Item(by='jackweirdy', descendants=None, kids=None, score=None, time=1606046065, title=None, item_type='comment', url=None, parent=25174668, text='Probably the best recommendation I can give is the certbot cloudflare plugin docs:<p><a href="https:&#x2F;&#x2F;certbot-dns-cloudflare.readthedocs.io&#x2F;en&#x2F;stable&#x2F;" rel="nofollow">https:&#x2F;&#x2F;certbot-dns-cloudflare.readthedocs.io&#x2F;en&#x2F;stable&#x2F;</a><p>I use this in my own setup. When letsencrypt provides a DNS challenge, the cloudflare plugin will complete it. The only requirements are<p>(a) owning a domain\n(b) having cloudflare control it\n(c) somewhere to run the certbot renewal cron job<p>If you don&#x27;t like b, you can swap out cloudflare for any of the other DNS providers. I chose it because it&#x27;s free.<p>For devices which can&#x27;t run certbot (e.g. a managed network switch) I run certbot on a raspberry pi and use a deployment hook to deploy it. The code for deploying to a tp link switch is open source:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;NotBobTheBuilder&#x2F;cert-deploy-t1600g-28ps" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;NotBobTheBuilder&#x2F;cert-deploy-t1600g-28ps</a>')
