Item(by='theamk', descendants=None, kids=[25986586, 25988318, 25987440, 25986505, 25992610, 25988355, 25987769, 25986932], score=None, time=1612154831, title=None, item_type='comment', url=None, parent=25983759, text='Reminder about alternatives:<p>For the less extreme case, when you want mostly one distro, but also a few packages from another one, there is “alien” [0] which converts between different package formats. It sometimes needs help with system integration or dependencies, but occasionally “just works”<p>For a more extreme case, there is a schroot (or a docker container) with &#x2F;home, audio and X mapped in. Works surprisingly well for desktop apps, like those embedded IDEs which require a specific Linux distribution you don’t want to run. You do have manage desktop integration (menu items, file associations) yourself though.<p>Those solutions are hard to set up than bedrock, but they also do not have special filesystem magic, so I bet they are much easier to debug.<p>[0] <a href="https:&#x2F;&#x2F;manpages.debian.org&#x2F;unstable&#x2F;alien&#x2F;alien.1p.en.html" rel="nofollow">https:&#x2F;&#x2F;manpages.debian.org&#x2F;unstable&#x2F;alien&#x2F;alien.1p.en.html</a>')
