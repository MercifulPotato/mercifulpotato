Item(by='commoner', descendants=None, kids=None, score=None, time=1611704130, title=None, item_type='comment', url=None, parent=25921487, text='The following Linux distributions support different scaling factors on different displays by default: Pop!_OS, Ubuntu (and GNOME-based derivatives), Linux Mint. Arch Linux, Manjaro, and all distributions using GNOME + Wayland can also enable mixed scaling with a quick setting change.<p>Pop!_OS (developed by System76) created its own HiDPI daemon to handle HiDPI and LoDPI displays on X11 at the same time:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;pop-os&#x2F;hidpi-daemon" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;pop-os&#x2F;hidpi-daemon</a><p><a href="https:&#x2F;&#x2F;blog.system76.com&#x2F;post&#x2F;174414833678&#x2F;all-about-the-hidpi-daemon" rel="nofollow">https:&#x2F;&#x2F;blog.system76.com&#x2F;post&#x2F;174414833678&#x2F;all-about-the-hi...</a><p>It is preinstalled on all System76 computers and enabled by default.<p>Ubuntu&#x27;s fork of the Mutter display manager (used by its fork of GNOME) includes a patch to handle different display resolutions for HiDPI and LoDPI displays on X11:<p><a href="https:&#x2F;&#x2F;bugs.launchpad.net&#x2F;ubuntu&#x2F;+source&#x2F;mutter&#x2F;+bug&#x2F;1820850" rel="nofollow">https:&#x2F;&#x2F;bugs.launchpad.net&#x2F;ubuntu&#x2F;+source&#x2F;mutter&#x2F;+bug&#x2F;182085...</a><p>Ubuntu and all of its GNOME-based derivatives include this patch, unless it is specifically excluded by the maintainers.<p>Linux Mint implemented fractional display scaling, with different settings for each display, in Cinnamon 4.6:<p><a href="https:&#x2F;&#x2F;blog.linuxmint.com&#x2F;?p=3858" rel="nofollow">https:&#x2F;&#x2F;blog.linuxmint.com&#x2F;?p=3858</a><p>Arch Linux and Manjaro users can also choose Cinnamon as the desktop environment for the same features.<p>If you are using a GNOME on X11 on Manjaro, you can install the mutter-x11-scaling package to replace Mutter with a version that includes Ubuntu&#x27;s changes:<p><a href="https:&#x2F;&#x2F;gitlab.manjaro.org&#x2F;packages&#x2F;extra&#x2F;mutter-x11-scaling&#x2F;-&#x2F;tree&#x2F;master" rel="nofollow">https:&#x2F;&#x2F;gitlab.manjaro.org&#x2F;packages&#x2F;extra&#x2F;mutter-x11-scaling...</a><p><a href="https:&#x2F;&#x2F;github.com&#x2F;puxplaying&#x2F;mutter-x11-scaling" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;puxplaying&#x2F;mutter-x11-scaling</a><p>Finally, if you are using GNOME on Wayland, mixed scaling is already supported. To enable fractional scaling, activate the &quot;scale-monitor-framebuffer&quot; setting:<p><a href="https:&#x2F;&#x2F;wiki.archlinux.org&#x2F;index.php&#x2F;HiDPI#GNOME" rel="nofollow">https:&#x2F;&#x2F;wiki.archlinux.org&#x2F;index.php&#x2F;HiDPI#GNOME</a><p>On Wayland, scaled applications that do not use GTK 3+ or Qt 5+ may appear blurry. This affects all Electron applications. X11 does not have the same issue, but Wayland is generally smoother and more stable than X11.')
