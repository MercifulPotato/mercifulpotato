Item(by='heroprotagonist', descendants=None, kids=None, score=None, time=1609232793, title=None, item_type='comment', url=None, parent=25565527, text='I moved to splitting my bashrc into multiple files and having my main bashrc source them from a ~&#x2F;.bashrcd directory.<p>At heart it&#x27;s a short snippet that just checks for existence and sources each file in the directory:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;targaryen&#x2F;bashrcd&#x2F;blob&#x2F;master&#x2F;install&#x2F;install_brcd.sh#L46" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;targaryen&#x2F;bashrcd&#x2F;blob&#x2F;master&#x2F;install&#x2F;ins...</a><p>I added aliases to list&#x2F;edit&#x2F;remove entries from the .bashrcd directory and resource it.  And a script I can call with a one-liner to edit bashrc on a new machine to add the sourcing and the helper aliases.<p>It&#x27;ll load alphabetically so I can prefix entries with a number to specify load order (defaulting to 0100 so I don&#x27;t need to specify this in the commands unless I explicitly changed them).<p>So the end result is that I can quickly edit or create a new bashrc entry by running &#x27;ebrc entryname&#x27;.  This opens ~&#x2F;.bashrcd&#x2F;0100--entryname in vi, and when it&#x27;s saved it&#x27;ll re-source so the add&#x2F;change takes effect immediately.<p>Or &#x27;lbrc&#x27; to list contents of the directory, or &#x27;rbrc entryname&#x27; to remove ~&#x2F;.bashrcd&#x2F;0100--entryname<p>It&#x27;s fairly simplistic but takes away most of the cognitive load of managing a complex bashrc.')
