By default, CORS dictates that cross-origin requests should not be allowed.<p>But sometimes you want that to be possible, so we have headers available to is where we can signal which domains are allowed to access the origin when on another origin.<p>In the case of the GitHub API, they (GitHub) are setting these headers to allow any origin to access the GitHub API from any other origin, that&#x27;s why your browser doesn&#x27;t throw a security error. Check out the various &quot;access-control-*&quot; headers the GitHub API returns as response headers when you use it.<p>It has nothing to do with custom domains or GitHub Pages, but all to do with CORS and associated headers. You can learn more about CORS here: <a href="https:&#x2F;&#x2F;developer.mozilla.org&#x2F;en-US&#x2F;docs&#x2F;Web&#x2F;HTTP&#x2F;CORS" rel="nofollow">https:&#x2F;&#x2F;developer.mozilla.org&#x2F;en-US&#x2F;docs&#x2F;Web&#x2F;HTTP&#x2F;CORS</a>')
