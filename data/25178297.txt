Item(by='flohofwoe', descendants=None, kids=[25178494], score=None, time=1606059719, title=None, item_type='comment', url=None, parent=25177990, text='A &#x27;-&gt;&#x27; is always a runtime indirection involving an extra memory access, while a &#x27;.&#x27; is always resolved into a single offset at compile time.<p>E.g. a:<p>int x = a-&gt;b-&gt;c-&gt;d;<p>means there&#x27;s 3 memory accesses, while<p>int x = a.b.c.d;<p>means there&#x27;s one memory access for the whole expression.<p>Also consider this:<p>int x = a-&gt;b.c-&gt;d;<p>I can immediately see where pointer indirections are happening.<p>...unless you&#x27;re in C++ of course which messed up this simple rule when references were added to the language.')
