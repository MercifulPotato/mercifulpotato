Item(by='saila', descendants=None, kids=None, score=None, time=1609449490, title=None, item_type='comment', url=None, parent=25596311, text='I think it&#x27;s better to create a <i>virtualenv</i> for each project and install tox and other project dependencies there.<p>In addition, I&#x27;m not sure tox-pyenv is required. I use tox with multiple pyenv-installed Python versions and have never used it.<p>Base:<p><pre><code>    - install homebrew\n    - brew install openssl readline sqlite3 xz zlib # recommended\n    - brew install pyenv\n    - pyenv install &lt;version&gt;  # for one or more versions\n</code></pre>\nProject:<p><pre><code>    - create&#x2F;clone project directory\n    - cd project directory\n    - pyenv local x.y.z x.y.z ...  # include each version the project supports\n    - python3 -m venv .venv   # create virtualenv for project using one of the pyenv versions\n    - install the project into the virtualenv\n    - install any additional dependencies into this virtualenv too, if necessary\n</code></pre>\nEdit: I&#x27;d also add that if you do need a tool to be installed globally, it&#x27;s better to use pipx or pip with --user. It&#x27;s generally not a good idea to install things into a system-level Python.<p><a href="https:&#x2F;&#x2F;pipxproject.github.io&#x2F;pipx&#x2F;" rel="nofollow">https:&#x2F;&#x2F;pipxproject.github.io&#x2F;pipx&#x2F;</a>')
