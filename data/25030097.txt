Item(by='MaxBarraclough', descendants=None, kids=None, score=None, time=1604878401, title=None, item_type='comment', url=None, parent=25028632, text='The point of the AGPL is to require sharing of modifications to the software when that software is used in a service. If you&#x27;re using an <i>unmodified</i> binary, then you haven&#x27;t made any changes to the software. You won&#x27;t be obligated to share  changes  if the changes don&#x27;t exist, so you could use the unmodified binary internally or externally.<p>The AGPLv3 has a lot to say about patents, which is enough to scare off many companies from software that uses the AGPLv3, but I believe it won&#x27;t be triggered if you never make any changes in the first place (and are therefore presumably not a &#x27;contributor&#x27;).<p>AGPLv3: <a href="https:&#x2F;&#x2F;www.gnu.org&#x2F;licenses&#x2F;agpl-3.0.en.html" rel="nofollow">https:&#x2F;&#x2F;www.gnu.org&#x2F;licenses&#x2F;agpl-3.0.en.html</a><p>See also <a href="https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Affero_General_Public_License" rel="nofollow">https:&#x2F;&#x2F;en.wikipedia.org&#x2F;wiki&#x2F;Affero_General_Public_License</a> , <a href="https:&#x2F;&#x2F;softwareengineering.stackexchange.com&#x2F;questions&#x2F;107883&#x2F;agpl-what-you-can-do-and-what-you-cant#comment202259_107931" rel="nofollow">https:&#x2F;&#x2F;softwareengineering.stackexchange.com&#x2F;questions&#x2F;1078...</a>')
