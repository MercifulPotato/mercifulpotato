Item(by='jeremyis', descendants=None, kids=None, score=None, time=1605116350, title=None, item_type='comment', url=None, parent=25060532, text='If you try to fetch values and there is no connection nor ached values, the client won&#x27;t be able to hit Config.ly&#x27;s servers.  I think it&#x27;s a good idea to implement Config.ly with graceful degradation if you are concerned about this case (e.g. &quot;configly.get(key) || &quot;default_value&quot;); though it should only happen in this specific use case.<p>If you lose internet while using it, the values could be cached and it will work well.<p>We have a graceful degradation feature on the roadmap; something like hard-coding fallback values in case of offline issues like this. Something like:<p>configly.init(API_KEY, { \n  default_values: {\n    price: &#x27;$1,000,000&#x27;,\n    upgrades: [&#x27;AC&#x27;, &#x27;Hot rod red&#x27;],\n  }\n)<p>What do you think?')
