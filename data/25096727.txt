Item(by='acemarke', descendants=None, kids=[25097093, 25097329, 25098216], score=None, time=1605397449, title=None, item_type='comment', url=None, parent=25095060, text='Hi, I&#x27;m a Redux maintainer.<p>If you haven&#x27;t looked at Redux recently, you should look again.  &quot;Modern Redux&quot; code is drastically simpler than the Redux code you&#x27;ve probably seen before.  In particular, our official Redux Toolkit package [0] and the React-Redux hooks API [1] make it much simpler to write Redux code, by auto-generating action creators, writing reducers as object lookup tables instead of switch statements, letting you write &quot;mutating&quot; immutable updates in reducers, and providing good defaults out of the box and utilities that handle common use cases.<p>FWIW, we&#x27;ve just completely rewritten the official tutorials in our docs to show how much simpler &quot;modern Redux&quot; code is, as well as better explain the fundamentals and why many of these patterns exist:<p>- The &quot;Redux Essentials&quot; tutorial [2] teaches &quot;how to use Redux, the right way&quot;, by building a real-world app using Redux Toolkit<p>- The &quot;Redux Fundamentals&quot; tutorial [3] teaches &quot;how Redux works, from the bottom up&quot;, by showing how to write Redux code by hand and why standard usage patterns exist, and how Redux Toolkit simplifies those patterns<p>In addition, we&#x27;ve added a &quot;Style Guide&quot; docs page [4] that lists our recommended patterns and best practices, which help you write Redux logic that is simpler, easier to understand, and easier to maintain.  That includes things like &quot;write Redux logic as single file &#x27;slices&#x27;&quot;, &quot;put as much logic as possible in reducers&quot;, &quot;model actions as &#x27;events&#x27;, not &#x27;setters&#x27;&quot;, and so on.<p>We&#x27;ve gotten a lot of great feedback telling us that RTK and the docs improvements have really made it a lot easier for people to learn and use Redux effectively.<p>[0] <a href="https:&#x2F;&#x2F;redux-toolkit.js.org" rel="nofollow">https:&#x2F;&#x2F;redux-toolkit.js.org</a><p>[1] <a href="https:&#x2F;&#x2F;react-redux.js.org&#x2F;api&#x2F;hooks" rel="nofollow">https:&#x2F;&#x2F;react-redux.js.org&#x2F;api&#x2F;hooks</a><p>[2] <a href="https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;essentials&#x2F;part-1-overview-concepts" rel="nofollow">https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;essentials&#x2F;part-1-overview-co...</a><p>[3] <a href="https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;fundamentals&#x2F;part-1-overview" rel="nofollow">https:&#x2F;&#x2F;redux.js.org&#x2F;tutorials&#x2F;fundamentals&#x2F;part-1-overview</a><p>[4] <a href="https:&#x2F;&#x2F;redux.js.org&#x2F;style-guide&#x2F;style-guide" rel="nofollow">https:&#x2F;&#x2F;redux.js.org&#x2F;style-guide&#x2F;style-guide</a>')
