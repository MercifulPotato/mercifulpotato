Item(by='susam', descendants=None, kids=[25621987, 25621922, 25621889, 25621892], score=None, time=1609682855, title=None, item_type='comment', url=None, parent=25621519, text='There is an easy fix that prevents an issue like this: Configure the web-server to accept HTTP connections only for specific hostnames. Here is how the config would look like:<p><pre><code>  sudo mkdir &#x2F;etc&#x2F;nginx&#x2F;ssl\n  sudo openssl req -x509 -newkey rsa:4096 -nodes -keyout &#x2F;etc&#x2F;nginx&#x2F;ssl&#x2F;key.pem -out &#x2F;etc&#x2F;nginx&#x2F;ssl&#x2F;cert.pem -days 365 -subj &quot;&#x2F;CN=localhost&quot;\n  echo &quot;server {\n      listen 80 default_server;\n      listen [::]:80 default_server;\n      listen 443 ssl default_server;\n      listen [::]:443 ssl default_server;\n      ssl_certificate &#x2F;etc&#x2F;nginx&#x2F;ssl&#x2F;cert.pem;\n      ssl_certificate_key &#x2F;etc&#x2F;nginx&#x2F;ssl&#x2F;key.pem;\n      return 444;\n  }&quot; &gt; &#x2F;etc&#x2F;nginx&#x2F;sites-enabled&#x2F;default</code></pre>')
