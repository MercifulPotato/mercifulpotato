Item(by='varl', descendants=None, kids=[25070829, 25071233, 25072775, 25074289, 25072918, 25071028, 25072891], score=None, time=1605191745, title=None, item_type='comment', url=None, parent=25069350, text='I found this user management strategy somewhere, and it&#x27;s been working great for me:<p><pre><code>  git config --global --unset user.name\n  git config --global --unset user.email\n  git config --global --unset user.signingkey\n\n  git config --global user.useConfigOnly true\n\n  git config --global user.&lt;id&gt;.name &quot;&lt;name&gt;&quot;\n  git config --global user.&lt;id&gt;.email &quot;&lt;email&gt;&quot;\n\n  git config --global alias.identity &#x27;! git config user.name &quot;$(git config user.$1.name)&quot;; git config user.email &quot;$(git config user.$1.email)&quot;; :&#x27;\n</code></pre>\nSo given that I have created two users, e.g. personal and work I run:<p><pre><code>  git identity work\n</code></pre>\nin repos that need the work name&#x2F;e-mail, and<p><pre><code>  git identity personal\n</code></pre>\nin the ones that are private.')
