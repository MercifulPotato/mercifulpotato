Item(by='alexashka', descendants=None, kids=[25158597, 25158516], score=None, time=1605861484, title=None, item_type='comment', url=None, parent=25158363, text='It&#x27;s a joke compared to the ruby on rails or node ecosystem.<p>It is about ecosystems, not languages.<p>Swift&#x27;s ecosystem is Apple products. If you&#x27;re not doing that, don&#x27;t use Swift.<p>You <i>will</i> have to learn and use multiple languages and multiple ecosystems regardless because each ecosystem insists on using its own language and tools. So just use the most popular one for each - it will be the most polished with the fewest number of gotchas.<p>At this point, the clear winners for each ecosystem are Kotlin, Swift, Typescript, C# and C++ if you&#x27;re doing games.<p>If you write a todo app on each platform, you&#x27;ll realize you&#x27;re using the same set of language features with different syntax - that should lessen the anxiety of learning a new language. Once you&#x27;re free and nimble, you&#x27;d have to be nuts to choose anything but the right tool for the job or when in doubt, the most popular.')
