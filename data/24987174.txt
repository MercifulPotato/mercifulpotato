Item(by='dtech', descendants=None, kids=None, score=None, time=1604476831, title=None, item_type='comment', url=None, parent=24987011, text='I&#x27;d say that holds about as much value as criticizing C-style languages for allowing all these variants<p><pre><code>    if(x) true\n    if(x)\n      true\n    if(x) (true)\n    if(x) {true}\n    if(x) {\n      true\n    }\n    if(x)\n    {\n      true\n    }\n</code></pre>\netc. In practice you use a linter to enforce a style and it&#x27;s not a problem.<p>For the curious. The combinations follow out of fairly simple rules:<p><pre><code>    () and {} are interchangeable for expressions - altough {} can contain multiple statements and () only an expression.\n    x op y is equivalent to x.op(y)\n    Type abscriptions - x : Type - are optional and will be inferred if possible\n    { case ... } is the pattern match construct and works similarly to a function with 1 parameter.</code></pre>')
