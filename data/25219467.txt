Item(by='dessant', descendants=None, kids=None, score=None, time=1606394296, title=None, item_type='comment', url=None, parent=25219235, text='One of my worst decisions as a developer was to build upon a component framework maintained by Google.<p>Material Components for the web is constantly introducing breaking changes that make building upon the framework a nightmare. Lately they&#x27;ve even set this expectation in the project description. The problem is that the components are rather buggy, so you must regularly update them to have upstream bugs fixed.<p>&gt; Material Components Web tends to release breaking changes on a monthly basis, but follows semver so you can control when you incorporate them. We typically follow a 2-week release schedule which includes one major release per month with breaking changes, and intermediate patch releases with bug fixes.<p><a href="https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-web" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-w...</a><p>The development team&#x27;s decisions have been questionable [1][2][3], so consider this a warning in case you are tempted to use the project.<p>[1] <a href="https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-web&#x2F;issues&#x2F;4799#issuecomment-551537015" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-w...</a><p>[2] <a href="https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-web&#x2F;issues&#x2F;2865" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-w...</a><p>[3] <a href="https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-web&#x2F;issues&#x2F;2639" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;material-components&#x2F;material-components-w...</a>')
