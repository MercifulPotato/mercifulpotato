Item(by='ignoramous', descendants=None, kids=[25884851, 25886280, 25885155], score=None, time=1611423077, title=None, item_type='comment', url=None, parent=25883253, text='The blog post doesn&#x27;t mention it but Firecracker was originally based on <i>crosvm</i> [0] built at Google by the ChromeOS team for a WSL-like Linux sandbox on top of ChromeOS running debian-buster containers viz. <i>penguin</i> (afaik) in a gentoo-based VM viz. <i>termina</i> [1]. <i>crosvm</i> inturn is part of a much bigger <i>crostini</i> project [2], which I find to be super fascinating, as it supports UI workloads (over Wayland and X).<p>If you&#x27;re using ChromeOS to run any Linux app, you&#x27;re using <i>crostini</i> which launches those apps in <i>crosvm</i> managed sandboxes (Container inside a VM) in seconds. In not so distant future, it looks like Android will sandbox platform workloads (running outside the Android framework?) managed by <i>crosvm</i>, to considerably improve security [3].<p>I looked for but couldn&#x27;t find ChromeOS GCP instances.
