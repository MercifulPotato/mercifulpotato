Item(by='twic', descendants=None, kids=None, score=None, time=1612114903, title=None, item_type='comment', url=None, parent=25979123, text='Unfortunately, there is tooling, and a subculture that builds tooling, that uses configuration files in the same directory as the source code.<p>Say you&#x27;re deploying with cool-deploy-tool. cool-deploy-tool finds the password for the server by looking for a file called &quot;Coolfile&quot; in the project root. You have to create that file to use it. Now you have a footgun lying around.<p>I associate this stuff with the Ruby community, but it&#x27;s spread far and wide, notably to the Node and Go communities, which i think have large Ruby emigrant populations.<p>As a prototypal example, consider dotenv, which began in Ruby:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;bkeepers&#x2F;dotenv" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;bkeepers&#x2F;dotenv</a><p>And has travelled to Node:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;motdotla&#x2F;dotenv" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;motdotla&#x2F;dotenv</a><p>And Go:<p><a href="https:&#x2F;&#x2F;github.com&#x2F;joho&#x2F;godotenv" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;joho&#x2F;godotenv</a>')
