Item(by='sergeykish', descendants=None, kids=None, score=None, time=1607176922, title=None, item_type='comment', url=None, parent=25305680, text='Unfortunately<p>&gt; If the local() function is provided, specifying a font name to look for on the user&#x27;s computer, and the user agent finds a match, that local font is used. Otherwise, the font resource specified using the url() function is downloaded and used.<p><a href="https:&#x2F;&#x2F;developer.mozilla.org&#x2F;en-US&#x2F;docs&#x2F;Web&#x2F;CSS&#x2F;@font-face" rel="nofollow">https:&#x2F;&#x2F;developer.mozilla.org&#x2F;en-US&#x2F;docs&#x2F;Web&#x2F;CSS&#x2F;@font-face</a><p>And this is not the case for Google Fonts anymore, for example<p><a href="https:&#x2F;&#x2F;fonts.googleapis.com&#x2F;css2?family=Noto%20Sans" rel="nofollow">https:&#x2F;&#x2F;fonts.googleapis.com&#x2F;css2?family=Noto%20Sans</a><p>I&#x27;ve created issue<p><a href="https:&#x2F;&#x2F;github.com&#x2F;google&#x2F;fonts&#x2F;issues&#x2F;2855" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;google&#x2F;fonts&#x2F;issues&#x2F;2855</a><p>----<p>&gt;&gt; Any correctly-coded website will try a local source first, which should help a lot for poor connections.<p>&gt; Then Google Fonts is not correctly coded, as it&#x27;ll use remote font only. And myriads of websites reusing Google Fonts snippets. To try local source first, you have to explicitly ask for it in @font-face&#x2F;src which Google Fonts does not do.<p>&gt; So installing font into your OS won&#x27;t help you with any website using Google Fonts service.<p><a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=25301830" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=25301830</a>')
