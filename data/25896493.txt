Item(by='robbiejs', descendants=None, kids=[25896996], score=None, time=1611525009, title=None, item_type='comment', url=None, parent=25894656, text='Data grids &#x2F; tables &#x2F; spreadsheets are different beasts. Always ask yourself what kind of table your SaaS app or website needs:<p>1) A read-only table for presentation purposes, with pagination and sorting options.<p>2) A read-only data grid that supports high-frequency updates&#x2F;redraws, for things like Stock or Bitcoin trading.<p>3) A data grid to edit JSON&#x2F;Javascript data, with user-friendly spreadsheet-like controls.<p>4) A spreadsheet-product, with features like formulas, merged cells, HTML (images) in cells, pivot tables, etc.<p>They have overlapping features. Most will have sorting, some will have undo&#x2F;redo, some will have formulas, etc. If a table product has many features, it will probably suffer performance. Some libs are 1MB (minified!)<p>I have written an (unfinished and biased) article that compares a few data table products, for anyone that&#x27;s interested: <a href="https:&#x2F;&#x2F;www.datagridxl.com&#x2F;compare" rel="nofollow">https:&#x2F;&#x2F;www.datagridxl.com&#x2F;compare</a>.')
