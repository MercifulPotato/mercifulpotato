Item(by='mumblemumble', descendants=None, kids=[24871119, 24871085, 24872779], score=None, time=1603468264, title=None, item_type='comment', url=None, parent=24870166, text='As a consumer, I loved the phone. As a developer, I was afraid to touch it. Dumping the old Windows Mobile platform for Windows Phone was understandable, but the compatibility break between Windows Phone 7 and Windows Phone 8 was really off-putting.<p>Though I think the biggest problem there is that it didn&#x27;t happen in a vacuum. It was one more event in a fairly long list of Microsoft technologies and APIs that were shipped to much fanfare, and then abandoned shortly after release. It made it hard to feel comfortable trying <i>any</i> Microsoft tech that wasn&#x27;t at least a fewy years old. Including mobile APIs. Which, given what happened with Windows Phone 8, did not turn out to be an overabundance of caution.<p>The problem hurt more than Windows Phone, too. A lot of people got sick of having the rug repeatedly yanked out from under them, and started developing on non-Microsoft tech stacks. Which then removed a lot of the need to run Windows Server, and things sort of snowballed.<p>Despite all his infamous ranting about the importance of keeping developers happy early on, the Ballmer years turned out to be the era when Microsoft perfected the art of alienating developers.')
