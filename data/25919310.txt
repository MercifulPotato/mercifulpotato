Item(by='Liquid_Fire', descendants=None, kids=None, score=None, time=1611685089, title=None, item_type='comment', url=None, parent=25918873, text='I always assumed this was related to the multiprocess architecture. If you&#x27;ve updated Firefox in the background (e.g. via your package manager), and the new version is not API-compatible, it will not be able to create a content process for the new tab, hence the message.<p>Obviously this is only a problem on Linux, because on other platforms Firefox&#x27;s own autoupdater will only apply the update when Firefox is restarting.')
