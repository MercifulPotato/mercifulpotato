Item(by='yencabulator', descendants=None, kids=[25174426], score=None, time=1606010164, title=None, item_type='comment', url=None, parent=25171572, text='For what it&#x27;s worth, `man fdatasync` on linux says (emphasis mine)<p><pre><code>  fdatasync() is similar to fsync(), but does not flush modified metadata *unless that meta‐\n  data  is  needed*  in  order to allow a subsequent data retrieval to be correctly handled.\n  For example, changes to st_atime or st_mtime (respectively, time of last access and  time\n  of  last  modification; see inode(7)) do not require flushing because they are not neces‐\n  sary for a subsequent data read to be handled correctly.  On the other hand, a change  to\n  the file size (*st_size, as made by say ftruncate(2)*), *would require a metadata flush*.</code></pre>')
