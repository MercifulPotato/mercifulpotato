Item(by='dspillett', descendants=None, kids=[25070856], score=None, time=1605193482, title=None, item_type='comment', url=None, parent=25070329, text='You can&#x27;t specifically access the cached version, but the OS will (assuming the file isn&#x27;t too large) remember the blocks read from disk in the cache for a while (until the memory is needed by something else) so that next time the file is ready those blocks do not need to be read from disk. This can make a huge difference on traditional drives.<p>&#x2F;dev&#x2F;null <i>is</i> the abyss, but the cat command will read all the blocks comprising the file from the filesystem before sending them to that abyss.<p>You can see the effect like so:<p><pre><code>  dspillett@swann2:&#x2F;tmp$ time cat a.bigish.file &gt; &#x2F;dev&#x2F;null\n  real    0m3.830s\n  user    0m0.000s\n  sys     0m0.468s\n  dspillett@swann2:&#x2F;tmp$ time cat a.bigish.file &gt; &#x2F;dev&#x2F;null\n  real    0m0.165s\n  user    0m0.004s\n  sys     0m0.164s\n  dspillett@swann2:&#x2F;tmp$ time cat a.bigish.file &gt; &#x2F;dev&#x2F;null\n  real    0m0.148s\n  user    0m0.000s\n  sys     0m0.144s\n</code></pre>\n(if &quot;a.bigish.file&quot; has only just been created, it will likely already be in RAM, run<p><pre><code>  sync; echo 3 &gt; &#x2F;proc&#x2F;sys&#x2F;vm&#x2F;drop_caches\n</code></pre>\nas root or other privileged user to clear the not-currently-active cache &amp; buffers, or chose an older file) (note though that this clears all cache machine-wide, not just cached information from your current session)')
