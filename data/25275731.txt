Item(by='Cthulhu_', descendants=None, kids=None, score=None, time=1606912024, title=None, item_type='comment', url=None, parent=25274678, text='Lambda is not specifically made for long and slow tasks; AWS Lambda specifically has a maximum execution time of 15 minutes (and you have to explicitly configure it to do so, see <a href="https:&#x2F;&#x2F;aws.amazon.com&#x2F;about-aws&#x2F;whats-new&#x2F;2018&#x2F;10&#x2F;aws-lambda-supports-functions-that-can-run-up-to-15-minutes&#x2F;#:~:text=You%20can%20now%20configure%20your,Lambda%20function%20was%205%20minutes" rel="nofollow">https:&#x2F;&#x2F;aws.amazon.com&#x2F;about-aws&#x2F;whats-new&#x2F;2018&#x2F;10&#x2F;aws-lambd...</a>.).<p>For longer tasks, spinning up an EC2 or Beanstalk instance is probably the way to go.<p>As for what to use it for, we used it in our application (deployed to Netlify which uses Lambda under the hood) where lambdas operated like a &#x27;proxy&#x27; to various 3rd party API suppliers (Commercetools, Adyen, some age verification service), and those too would use a lambda function to ping back at us (e.g. when payment was confirmed). Worked pretty well, although in retrospect I would&#x27;ve preferred a &#x27;normal&#x27;, monolithic server to do the same thing.')
