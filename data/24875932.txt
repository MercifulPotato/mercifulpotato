Item(by='chubot', descendants=None, kids=[24876053], score=None, time=1603500250, title=None, item_type='comment', url=None, parent=24874990, text='(author here) I do use Python.  In fact I wrote something like 30K lines of Python for Oil.<p>The question isn&#x27;t shell OR Python.<p>My shell scripts call Python scripts, many of white I wrote myself.  That is working as intended.<p>They also call C programs, C++ programs and R programs.  And put JavaScript programs in various places.<p>----<p>I guess this is a huge misconception about shell that I have to write a blog post about.<p><a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24083764" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24083764</a><p><i>I&#x27;m often asked this about Oil [1]: Why do you want to write programs in shell?</i><p><i>That&#x27;s not the idea of shell. The idea is that I write programs in Python, JavaScript, R, and C++ regularly, and about 10 different DSLs (SQL, HTML, etc.) And I work on systems written by others, consisting of even more languages.</i><p><i>I need a language to glue them together. A language to express build automation and describe deployed systems. Most big systems consist of more than one language.</i><p><i>Shell is the best language for that, but it&#x27;s also old and crufty, with poor implementations.</i><p><i>When you program in shell, gcc, git, pip, npm, markdown, rsync, diff, perf, strace, etc. are part of your &quot;standard library&quot;.</i><p>-----<p>If you want some concrete examples, look in the Oil repo.  There are dozens of shell scripts that invoke custom tools in Python, R, and C++.<p><a href="https:&#x2F;&#x2F;github.com&#x2F;oilshell&#x2F;oil&#x2F;tree&#x2F;master&#x2F;benchmarks" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;oilshell&#x2F;oil&#x2F;tree&#x2F;master&#x2F;benchmarks</a><p>For example, to generate this report on how much we speed up Python by translating it to C++: <a href="https:&#x2F;&#x2F;www.oilshell.org&#x2F;release&#x2F;0.8.3&#x2F;benchmarks.wwz&#x2F;mycpp-examples&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.oilshell.org&#x2F;release&#x2F;0.8.3&#x2F;benchmarks.wwz&#x2F;mycpp-...</a><p>The tables are manipulated by R code, and shell&#x2F;Python&#x2F;CommonMark generates the HTML.<p>-----<p>Another example is that the release page is essentially a huge shell script: <a href="https:&#x2F;&#x2F;www.oilshell.org&#x2F;release&#x2F;0.8.3&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.oilshell.org&#x2F;release&#x2F;0.8.3&#x2F;</a><p>This page and all the linked pages are generated by: <a href="https:&#x2F;&#x2F;github.com&#x2F;oilshell&#x2F;oil&#x2F;blob&#x2F;master&#x2F;devtools&#x2F;release.sh" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;oilshell&#x2F;oil&#x2F;blob&#x2F;master&#x2F;devtools&#x2F;release...</a><p>------<p>If you don&#x27;t understand why shell should call Python and C, I recommend reading <i>The Art of Unix Programming</i><p><a href="http:&#x2F;&#x2F;www.catb.org&#x2F;~esr&#x2F;writings&#x2F;taoup&#x2F;html&#x2F;" rel="nofollow">http:&#x2F;&#x2F;www.catb.org&#x2F;~esr&#x2F;writings&#x2F;taoup&#x2F;html&#x2F;</a><p><a href="https:&#x2F;&#x2F;www.amazon.com&#x2F;UNIX-Programming-Addison-Wesley-Professional-Computng&#x2F;dp&#x2F;0131429019" rel="nofollow">https:&#x2F;&#x2F;www.amazon.com&#x2F;UNIX-Programming-Addison-Wesley-Profe...</a>')
