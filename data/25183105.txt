Item(by='matvore', descendants=None, kids=None, score=None, time=1606099221, title=None, item_type='comment', url=None, parent=25181261, text='This is used by the Ganesha project (userspace NFS server). Look for the symbol &quot;container_of&quot; and usages of it in <a href="https:&#x2F;&#x2F;github.com&#x2F;nfs-ganesha&#x2F;nfs-ganesha&#x2F;" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;nfs-ganesha&#x2F;nfs-ganesha&#x2F;</a> (disclaimer: I&#x27;m a minor contributor).<p>The way it&#x27;s used is that Ganesha supports defining of alternate filesystem backends and serving them as NFS shares. Handles to objects (e.g. files) would exist as pointers which live inside the struct of the backend&#x27;s handle struct. i.e.:<p><pre><code>  struct my_file_data {\n    struct ganesha_file_data {\n      &#x2F;&#x2F; generic data\n    };\n\n    &#x2F;&#x2F; data specific to my module\n  };\n</code></pre>\nThe &quot;my&quot; module would take pointers to ganesha_file_data when the NFS core code calls it. The &quot;my&quot; module then uses container_of to convert ganesha_file_data ptr to my_file_data ptr.')
