Item(by='DoofusOfDeath', descendants=None, kids=None, score=None, time=1603725795, title=None, item_type='comment', url=None, parent=24892400, text='I&#x27;m generally not a fan, but header-only libraries <i>do</i> make it easier to ensure that specific compiler&#x2F;linker flags are consistent across your code <i>and</i> the library&#x27;s code.<p>Some examples of when this is helpful:<p>- using gcc&#x27;s &#x2F; clang&#x27;s sanitizer frameworks<p>- tweaking builds for deep-dive optimization and&#x2F;or debugging<p>- letting the compiler target a specific hardware architecture, e.g. `-mavx512f`.<p>I guess this highlights a limitation in typical Linux systems: a given library can be built with many different configuration options, but there&#x27;s no (widely known?) way to have all those variants installed at the same time.<p>I think fixing this would require more than just changing cmake, apt&#x2F;dpkg, or the C++ standard. A clean solution might also require changes to the ELF standard and&#x2F;or Linux&#x27;s compilers, linkers, dynamic loaders, and debuggers.')
