Item(by='jokethrowaway', descendants=None, kids=None, score=None, time=1610537214, title=None, item_type='comment', url=None, parent=25757787, text='If you&#x27;re talking about having periodic jobs to run you can use heroku&#x27;s scheduler plugin and start a job.\nIt will use a different dyno from the one running your api.<p>You will end up paying more, but you won&#x27;t have problems with resources sharing.<p>In a traditional VPS environment you would have needed to have a machine beefy enough to run both api and jobs or spin up a new machine.\nCron, Jenkins or another task runner could have achieved the same.<p>If you&#x27;re talking about having long running requests initiated by users, a common pattern is to have a queue:<p>- User send a request<p>- API add request to a queue<p>- n workers wait for a queue message and process it once it arrives<p>Another simpler approach is to just spin another instance of the API and just forward long running requests to the new instance. Depending on the type of job, its dependencies, its frequency you may run into some problems that you wouldn&#x27;t have with the queue based approach')
