Adding async nature to a technical problem (webhooks + API as you suggest) immediately makes a otherwise sync problem way harder and error prone.<p>&gt; I bet GitHub covers the vast majority of their target users.<p>Sure, but if they added Git, they would reach everyone using GitHub + everyone else who is using Git.<p>With powerful primitives you can build powerful abstractions, Git is one of the examples of this.
