They added it a while back.<p><pre><code>  [things-to-capture](params) -&gt; return-type { function-body; }\n</code></pre>\nThe return type is optional [0] since it can be detected automatically. The things-to-capture have a couple variants:<p><pre><code>  [&amp;foo]...\n</code></pre>\nCaptures variable foo by reference.<p><pre><code>  [foo]...\n</code></pre>\nMakes a copy of foo.<p>You can also capture &quot;everything&quot; using [&amp;] or [=]. Handy if you don&#x27;t want to make an explicit list of captured variables. You can also mix it:<p><pre><code>  [&amp;foo, bar]...\n</code></pre>\nCaptures foo by reference and a copy of bar.<p><a href="https:&#x2F;&#x2F;en.cppreference.com&#x2F;w&#x2F;cpp&#x2F;language&#x2F;lambda" rel="nofollow">https:&#x2F;&#x2F;en.cppreference.com&#x2F;w&#x2F;cpp&#x2F;language&#x2F;lambda</a><p>[0] Additionally, lambdas may not have a return value. Like this one:<p><pre><code>  int n = 0;\n  auto counter = [&amp;n] () { n++; }\n  counter();\n  &#x2F;&#x2F; now n == 1</code></pre>')
