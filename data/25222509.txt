Regardless if you write almost everything yourself or use a full featured framework, composer is always good to learn.<p>And even when you are a follower of KISS there always things that you don&#x27;t care enough about to write yourself, boring things like logging frameworks (Monolog <a href="https:&#x2F;&#x2F;seldaek.github.io&#x2F;monolog&#x2F;" rel="nofollow">https:&#x2F;&#x2F;seldaek.github.io&#x2F;monolog&#x2F;</a>) or testing frameworks (PHPUnit <a href="https:&#x2F;&#x2F;phpunit.de&#x2F;" rel="nofollow">https:&#x2F;&#x2F;phpunit.de&#x2F;</a>). And composer can also help you set up class autoloading, a must for modern PHP.<p>Here you can find packages to download with composer \n<a href="https:&#x2F;&#x2F;packagist.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;packagist.org&#x2F;</a><p>Next I would look into different frameworks, not necessary to use them but to learn about them and from them. The 2 biggest today is Symfony ( (<a href="https:&#x2F;&#x2F;symfony.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;symfony.com&#x2F;</a>) and Laravel (<a href="https:&#x2F;&#x2F;laravel.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;laravel.com&#x2F;</a>). But there are others like Slim (<a href="https:&#x2F;&#x2F;www.slimframework.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.slimframework.com&#x2F;</a>), CakePHP (<a href="https:&#x2F;&#x2F;cakephp.org&#x2F;" rel="nofollow">https:&#x2F;&#x2F;cakephp.org&#x2F;</a>), Zend (<a href="https:&#x2F;&#x2F;framework.zend.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;framework.zend.com&#x2F;</a>) and many more. Try a few and get the feeling how they do things and if you like them.<p>For view rendering you can do that with PHP out of the box (just remember to escape output!) or you can use a template engine like the more advanced Twig (<a href="https:&#x2F;&#x2F;twig.symfony.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;twig.symfony.com&#x2F;</a>) or the simpler Mustache (<a href="https:&#x2F;&#x2F;github.com&#x2F;bobthecow&#x2F;mustache.php" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;bobthecow&#x2F;mustache.php</a>). Plates (<a href="http:&#x2F;&#x2F;platesphp.com&#x2F;" rel="nofollow">http:&#x2F;&#x2F;platesphp.com&#x2F;</a>) uses native PHP templates but helps you create reusable layouts.
