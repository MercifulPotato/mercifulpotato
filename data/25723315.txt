Item(by='dane-pgp', descendants=None, kids=None, score=None, time=1610330430, title=None, item_type='comment', url=None, parent=25722076, text='Impressive work, but it leaves open the inevitable question of whether the software that is distributed actually implements the protocol as modelled in the proof.  With reproducible builds and a trustworthy build environment, we can hopefully assume that the binary matches the source code, but there isn&#x27;t yet a proof that the source code doesn&#x27;t have some unmodelled edge cases (as in the 2017 Key Reinstallation AttaCKs[0] on WPA2).<p>About this threat, the paper says:<p>&quot;Concerning implementation flaws, our formalisation can be used as a reference for the correct implementation of MTProto 2.0 clients (and servers). Tools like Spi2Java or FS2PV can be useful to this end.&quot;<p>[0] <a href="https:&#x2F;&#x2F;www.krackattacks.com&#x2F;" rel="nofollow">https:&#x2F;&#x2F;www.krackattacks.com&#x2F;</a>')
