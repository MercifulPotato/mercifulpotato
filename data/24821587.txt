Item(by='Myrmornis', descendants=None, kids=None, score=None, time=1603059709, title=None, item_type='comment', url=None, parent=24821513, text='Over the last year I&#x27;ve been writing a replacement for the preview-latex functionality in auctex. Coincidentally, I just posted a Show HN on it:<p><a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24821474" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=24821474</a><p><a href="https:&#x2F;&#x2F;github.com&#x2F;dandavison&#x2F;xenops" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;dandavison&#x2F;xenops</a><p>Xenops is intended to be used with auctex. The main feature that Xenops brings is that math, tables, and TikZ are rendered automatically, as soon as the cursor leaves the element. Rendering is asynchronous (using emacs-aio), so it doesn&#x27;t interrupt user actions. It allows you to work on a \\begin{equation&#x2F;align&#x2F;table&#x2F;TikZ} block with a preview of the rendered content in view.<p>Another advantage is that it supports SVG images, so preview images are perfectly crisp when viewed on a high resolution display.<p>A video demo of Xenops is here: <a href="https:&#x2F;&#x2F;www.youtube.com&#x2F;watch?v=GNS-ZFVaFM4" rel="nofollow">https:&#x2F;&#x2F;www.youtube.com&#x2F;watch?v=GNS-ZFVaFM4</a>')
