Item(by='masklinn', descendants=None, kids=[25683871], score=None, time=1610107965, title=None, item_type='comment', url=None, parent=25683713, text='There are lots of optimisations which have ultimately observable side-effects e.g. in most JS implementations the closed-over environment is optimised to just the free variables (and there&#x27;s no closure at all if there&#x27;s nothing being closed over statically), this is extremely noticeable if you drop into a debugger in the inner function because you will not be able to access any variable which wasn&#x27;t closed over. Likewise, if the compiler decides to reuse part of the stackframe because it sees two variables are only used consecutively, you won&#x27;t be able to access the first variable despite it still purportedly being &quot;in scope&quot;.')
