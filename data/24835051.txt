Item(by='JdeBP', descendants=None, kids=[24840917], score=None, time=1603182427, title=None, item_type='comment', url=None, parent=24830611, text='As the headlined page is about portable shell script coding, note that the -n option to the echo utility is not portable.  You should be using printf here, to be in the spirit of the headlined page.  (And you should also be quoting variable assignments for safety.)<p>There&#x27;s a long history to this, which starts with the fact that adding any option support at all to the original echo breaks stuff.  The result over the years has been quite a mess.  The current <i>Single Unix Specification</i> has an note stating outright that one should simply not use -n or escape sequences at all with echo if one desires portability, and also stating that one should use printf instead.<p>* <a href="https:&#x2F;&#x2F;unix.stackexchange.com&#x2F;q&#x2F;65803&#x2F;5132" rel="nofollow">https:&#x2F;&#x2F;unix.stackexchange.com&#x2F;q&#x2F;65803&#x2F;5132</a><p>* <a href="https:&#x2F;&#x2F;pubs.opengroup.org&#x2F;onlinepubs&#x2F;9699919799&#x2F;utilities&#x2F;echo.html" rel="nofollow">https:&#x2F;&#x2F;pubs.opengroup.org&#x2F;onlinepubs&#x2F;9699919799&#x2F;utilities&#x2F;e...</a>')
