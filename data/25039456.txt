Item(by='todd8', descendants=None, kids=None, score=None, time=1604953591, title=None, item_type='comment', url=None, parent=25038191, text='Advanced garbage collectors have short pause times. In [1], the author cites Twitter statistics showing approximately two 0.5ms &quot;pause&quot; times per GC cycle for the go language garbage collector, and during the GC cycle only 25% of the CPU was used by the GC. It doesn&#x27;t seem like this is going to cause frame-rates to dip.<p>To be fair, every application is going to differ, and perhaps games need tighter control of memory than a high performance GC is capable of, but automatic GC does make programming a lot easier.<p>[1] <a href="https:&#x2F;&#x2F;blog.golang.org&#x2F;ismmkeynote" rel="nofollow">https:&#x2F;&#x2F;blog.golang.org&#x2F;ismmkeynote</a>')
