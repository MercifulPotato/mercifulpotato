Item(by='flohofwoe', descendants=None, kids=None, score=None, time=1603458061, title=None, item_type='comment', url=None, parent=24860304, text='The main benefit is that the GPU-side code can better integrate with the CPU-side code, and that you don&#x27;t need separate tools and a complex build process integration.<p>For instance it&#x27;s possible to share data structure between the CPU and GPU side directly from the same source code, or you can run the same code on the CPU or GPU for whatever reason (debugging, unit testing etc...).<p>The Metal API uses C++ (with some extensions) as shader language, which IMHO is complete overkill too, but one nice side effect is that you can put shared data structures into headers, and then include the same shared headers on the CPU and GPU side.<p>Ideally it should be possible to mix GPU and CPU code in the same source file, and only (for instance) annotate GPU functions with an attribute which causes it to be compiled as shader code.')
