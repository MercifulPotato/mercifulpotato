For the reasons on why creating Deno I recommend  &quot;10 Things I Regret About Node&quot; by Deno&#x27;s author [1]<p>Deno is different than Node in several aspects; most notably:<p>- Deno supports only ES modules, there&#x27;s no built-in support for CommonJS modules<p>- Deno&#x27;s APIs are all promised based<p>- Deno does not use NPM, instead it can pull code from any URL, much like browsers do<p>- Deno has built-in permission system that by default runs your code in full sandbox allowing to opt-in into breaking out of sandbox (eg. to read a file from disk)<p>- As you&#x27;ve mention Deno can run .ts files without explicit build step<p>- Deno comes with a full toolchain in a single binary (formatter, linter, test runner, bundler, doc generator)<p>[1] <a href="https:&#x2F;&#x2F;www.youtube.com&#x2F;watch?v=M3BM9TB-8yA" rel="nofollow">https:&#x2F;&#x2F;www.youtube.com&#x2F;watch?v=M3BM9TB-8yA</a>')
