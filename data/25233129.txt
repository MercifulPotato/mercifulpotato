Item(by='jlokier', descendants=None, kids=None, score=None, time=1606516175, title=None, item_type='comment', url=None, parent=25232855, text='It&#x27;s probably doable because VMware did something very similar about 20 years ago, back before x86 had virtualisation hardware.<p>That was x86 on x86, but it took a similar level of sophistication then as with cross-architecture &quot;software virtualisation&quot; now to run all 32-bit Windows software, as well as all 16-bit software, other OSes such as Linux, BSD and MS-DOS and Netware, etc.<p>Considering all the things around like self-modifying code, generated code, thunks etc in Windows and other OSes and applications, VMware did a great job of scanning all code for entry points, intercepting every path that couldn&#x27;t be executed directly, patching what needed to be patched.<p>I expect they can revive that approach with cross-architecture translation now, and I would be very surprised if VMware and Parallels are not working on exactly this for x86-on-ARM VMs right now.')
