Item(by='rfoo', descendants=None, kids=None, score=None, time=1610717335, title=None, item_type='comment', url=None, parent=25789175, text='Sure, but there are no corruption at all.<p>The bug is, if you write garbage data into a file &quot;name&quot; similar to those holding filesystem internal metadata, the filesystem driver detects this and mistakenly thinks the file could be the real metadata, so it does a consistency check on it, and because it fails consistency check, it sets a &quot;volume needs fsck&quot; flag. Windows (after 18H2) then show an alert. The alert is extremely confusing (it literally says &quot;disk corruption&quot;) to the user, hence all the buzz.')
