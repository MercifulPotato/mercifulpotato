Item(by='dane-pgp', descendants=None, kids=None, score=None, time=1607047997, title=None, item_type='comment', url=None, parent=25296837, text='The video seems to end with an open question about how to trust a JavaScript client to actually be doing the E2EE that you think it is, but this is not as intractable a problem as it seems.<p>As long as you can pin a web app the first time you access it, you get at least TOFU security (which is roughly equivalent to downloading a binary app that hasn&#x27;t been reproducibly compiled from independently audited source code).<p>One way to achieve this is to use the SRI bookmarklet trick[0], and in fact it lets you pin a version of a web app with a specific hash, which can be provided out-of-band by a contact who you trust to vouch for the security of that version.<p>The biggest drawback with this method, though, is that the Data URI appears in the browser&#x27;s location bar instead of a trusted domain.  Fortunately there exists an experimental standard that could solve that problem, namely Cryptographic Hyperlinks[1].<p>[0] <a href="https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=17776456" rel="nofollow">https:&#x2F;&#x2F;news.ycombinator.com&#x2F;item?id=17776456</a><p>[1] <a href="https:&#x2F;&#x2F;w3c-ccg.github.io&#x2F;hashlink&#x2F;" rel="nofollow">https:&#x2F;&#x2F;w3c-ccg.github.io&#x2F;hashlink&#x2F;</a>')
