Item(by='pdkl95', descendants=None, kids=None, score=None, time=1606627297, title=None, item_type='comment', url=None, parent=25242311, text='Source routing also gives malicious hosts the ability to influence packet routing. IPv4 already tried this with the LSRR&#x2F;SSRR[1] options. The LSRR option was useful to malicious hosts for many purposes, such as[2]:<p><pre><code>   o  Bypass firewall rules.\n\n   o  Reach otherwise unreachable internet systems.\n\n   o  Establish TCP connections in a stealthy way.\n\n   o  Learn about the topology of a network.\n\n   o  Perform bandwidth-exhaustion attacks.\n</code></pre>\nDue to these &quot;well-known security implications&quot;[2], RFC 7126 recommends[3]:<p><pre><code>    Routers, security gateways, and firewalls SHOULD implement an option-\n    specific configuration knob to select whether packets with this\n    option are dropped, packets with this IP option are forwarded as if\n    they did not contain this IP option, or packets with this option are\n    processed and forwarded as per [RFC0791].  The default setting for\n    this knob SHOULD be &quot;drop&quot;, and the default setting MUST be\n    documented.\n</code></pre>\n[1] &quot;{Loose&#x2F;Strict} Source and Record Route&quot; <a href="http:&#x2F;&#x2F;www.networksorcery.com&#x2F;enp&#x2F;protocol&#x2F;ip&#x2F;option003.htm" rel="nofollow">http:&#x2F;&#x2F;www.networksorcery.com&#x2F;enp&#x2F;protocol&#x2F;ip&#x2F;option003.htm</a><p>[2] <a href="https:&#x2F;&#x2F;tools.ietf.org&#x2F;html&#x2F;rfc7126#section-4.3.3" rel="nofollow">https:&#x2F;&#x2F;tools.ietf.org&#x2F;html&#x2F;rfc7126#section-4.3.3</a><p>[3] <a href="https:&#x2F;&#x2F;tools.ietf.org&#x2F;html&#x2F;rfc7126#section-4.3.5" rel="nofollow">https:&#x2F;&#x2F;tools.ietf.org&#x2F;html&#x2F;rfc7126#section-4.3.5</a>')
