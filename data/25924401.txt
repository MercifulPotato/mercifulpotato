Item(by='networked', descendants=None, kids=None, score=None, time=1611722610, title=None, item_type='comment', url=None, parent=25921218, text='1. <i>at</i> runs jobs at a given time.   <i>batch</i> runs jobs &quot;when system load levels permit&quot; [1].  <i>nq</i> runs jobs in sequence with no regard to the system&#x27;s load average.<p>2. <i>at</i> and <i>batch</i> have 52 built-in queues: a-z and A-Z.  Any directory can be a queue for <i>nq</i>.<p>3. You can follow the output of an <i>nq</i> queue <i>tail</i>-style with <i>fq</i>.<p>4. The syntax is different.  <i>at</i> and <i>batch</i> take whole scripts from the standard input or a file; <i>nq</i> takes a single command as its command line arguments.<p>5. <i>nq</i> doesn&#x27;t rely on a daemon.  It&#x27;s an admirably simple design.  Jobs are just flock()-ed output files in a directory.<p>[1] See <a href="https:&#x2F;&#x2F;www.freebsd.org&#x2F;cgi&#x2F;man.cgi?query=at&amp;sektion=1&amp;n=1" rel="nofollow">https:&#x2F;&#x2F;www.freebsd.org&#x2F;cgi&#x2F;man.cgi?query=at&amp;sektion=1&amp;n=1</a>.')
