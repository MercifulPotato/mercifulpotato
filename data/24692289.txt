Item(by='noworriesnate', descendants=None, kids=[24692606], score=None, time=1601933617, title=None, item_type='comment', url=None, parent=24692061, text='There are a couple of types of distributed applications though, right?<p>There&#x27;s the traditional client&#x2F;server scenario where you distribute the load horizontally across many servers. If most of the server computation is stateless, and all the state is stored in the database, then you can use either the FaaS or the actor model.<p>Then there&#x27;s the client&#x2F;server scenario where the server computation is stateful, in which case you shouldn&#x27;t use FaaS. You could still use actors though. Isn&#x27;t this where Orleans sits?<p>Then there&#x27;s the peer to peer architecture where all computers run the same code, whether they&#x27;re in the cloud or on an end user&#x27;s laptop or phone. Does Orleans make sense in this case?')
