Item(by='yodsanklai', descendants=None, kids=None, score=None, time=1608239764, title=None, item_type='comment', url=None, parent=25458230, text='Great article.<p>The first thing that I was told about git was that commits are snapshots. It is enough to understand the simple workflow: pull, commit -a, push.<p>What confused me was everything that involves applying diffs (git stash apply, git rebase, git cherry-pick).<p>The secret is that you need to think both in terms of snapshots, or diffs, depending on the context.<p>Some git tutorials show you a few &quot;magic&quot; commands to get you started but it&#x27;s best to really understand the git model as soon as possible. It&#x27;s not that hard, and it&#x27;s here to stay. We may still be using git in 30 years, just like we&#x27;re still using bash and vim.')
