Item(by='wtetzner', descendants=None, kids=None, score=None, time=1604715337, title=None, item_type='comment', url=None, parent=25009772, text='In a scenario where you&#x27;re just rebinding, you can&#x27;t do this, for example (pseudo code):<p><pre><code>    foo = 12\n    for x in xs {\n      foo = x + foo\n    }\n</code></pre>\nIt would <i>always</i> create a new foo binding in the for loop with value (12 + x);<p>I think using Lisp syntax might help clarify, because it makes the binding scope explicit:<p><pre><code>    (let [foo 15]\n      (println foo) ; prints 15\n      (let [foo 12]\n        ; prints 12\n        (println foo)\n      ; prints 15\n      (println foo)))\n</code></pre>\n&gt; unlike a value which is actually a thing sitting in a memory slot in your vm.<p>A mutable variable <i>is</i> a memory slot that can be mutated, though.')
