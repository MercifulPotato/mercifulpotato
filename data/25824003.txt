Item(by='PragmaticPulp', descendants=None, kids=[25824096], score=None, time=1610990338, title=None, item_type='comment', url=None, parent=25823790, text='Generally, no.<p>GitHub access shouldn&#x27;t imply all-or-nothing access to the code. The only reason to give someone GitHub access is if they&#x27;re working with something stored in Git repositories. Most designers don&#x27;t store anything in Git, so no need.<p>&gt; Access to the latest build of an application<p>Generally, letting non-devs grab random nightly builds isn&#x27;t as helpful as it sounds. If designers need a new build, they should ask for one from the team and get something that has been minimally vetted.<p>Handing out nightly builds outside of dev teams leads to a slew of unexpected problems, including a lot of complaints about known issues or a sentiment that the code is &quot;buggy&quot; because people don&#x27;t understand what a nightly (or per-commit) build means to the dev team.')
