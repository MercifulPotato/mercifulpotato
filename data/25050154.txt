For me, PHP&#x27;s docs were an amazing on-ramp to becoming a developer &quot;I just need a function that does something like...&quot;. A lot of the language design is annoyingly inconsistent and hard to use, but even in 2005 there were clear explanations on how to make it work.<p>As an experienced engineer, if I&#x27;m handed a well-designed but poorly-documented API, I can usually guess the intent and make it work, at the cost of working more slowly. But a junior engineer will be completely stymied by insufficient docs, regardless of how good the software design is.<p>I guess, in summary: good docs can make up for bad code. But good code can&#x27;t make up for bad docs.')
