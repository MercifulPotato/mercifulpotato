Item(by='com2kid', descendants=None, kids=None, score=None, time=1607379835, title=None, item_type='comment', url=None, parent=25336204, text='It adds overhead on initial write, saves massive overhead every day after that.<p>There is JS code I deal with once a month where the manipulation of types is so complex I probably burn ~30 minutes every time I have to touch it. If that was was transformed into TS (which is going to happen eventually) that&#x27;d be 30 minutes saved per month, on this one particular flow of data.<p>I&#x27;ve done refactorings that were only possible because TS existed.<p>A lot of JS unit tests consist of &quot;ensure these fields exist on this object after it has been called by these functions.&quot;<p>Typescript removes the need for those tests.<p>And it removes the need to update those tests every time the code changes (just change the declarations appropriately!). And it removes the need to run those tests on every commit.<p>That said, the overall code&#x2F;compile&#x2F;run time savings is possibly not in TS&#x27;s favor due to how slow the compiler is. :&#x2F;')
